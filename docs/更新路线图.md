# XianTao 指令系统更新路线图

## 当前概况

10 个 Listener → 7 个 Handler，总计 48 条触发模式

---

## P0 — 必须修复/添加

### 1. [BUG] 灵兽指令双重注册

`灵兽出战`、`灵兽召回`、`灵兽恢复` 同时在 **FudiHandle** 和 **BeastHandle** 注册了 `@Filter`，且路由到不同的 Handler（FudiCommandHandler vs BeastCommandHandler），导致双发。

**解决**：删除 FudiHandle 中三条重复 listener，`灵兽XXX` 统一由 BeastHandle 专管。

涉及文件：
- `handle/onebotv11/FudiHandle.java` — 删除第 176-207 行三个方法
- `handle/command/FudiCommandHandler.java` — 删除 handleDeployBeast / handleUndeployBeast / handleRecoverBeast

### 2. [缺失] 帮助系统 `帮助 {{command}}`

文字 MUD 无帮助不可玩。需添加帮助指令，按模块列出可用命令及用法。

**新增**：
- `handle/onebotv11/HelpHandle.java` — 新 Listener
- `handle/command/HelpCommandHandler.java` — 新 Handler
- 可返回系统内所有指令列表及示例

## P1 — 核心体验缺失

### 3. [缺失] 丢弃物品 `丢弃 {{item}}`

背包物品无法丢弃，空间无法释放。

**新增**：`handle/command/` 中新增或复用现有 handler，调用 `StackableItemService` 或 `InventoryService` 实现扣减。

### 4. [缺失] 查看/观察 `查看 {{target}}`

无法查看物品详情、地图描述、他人信息。

**新增**：`查看 {{item}}` 查看物品属性，`查看 {{nickname}}` 查看他人状态（受限于可见信息）。

## 指令精简 — 可优化项

### A. 背包子命令合并

```
背包             ← 汇总（保留）
背包 种子        ← 独立 @Filter
背包 装备        ← 独立 @Filter
背包 兽卵        ← 独立 @Filter
```

**建议**：合并为一条 `背包 {{category}}`，category 可选（种子/装备/兽卵/丹药/材料/全部），缺省显示汇总。可减少 3 个 @Filter + 3 个 handler 方法。

### B. 丹方入口统一

`丹方列表`（列表）与 `丹方 {{name}}`（详情）共存，但 `丹方` 无参时不触发任何操作。

**建议**：`丹方`（无参）= 列表，`丹方 {{name}}` = 详情，省去 `丹方列表` 的独立 @Filter。

### C. 自动/手动炼丹歧义

`炼 {{recipeName}}` 与 `炼 {{herbInput}}` 的 @Filter 模式完全相同，无法区分。

**建议**：
- 手动：`炼 {{recipeName}}`（手动配药）
- 自动：`炼方 {{herbInput}}`（根据丹方）

### D. 法决语义调整

`法决` = 已装载列表，`法决列表` = 所有已学。命名不直观。

**建议**：
- `法决` = 已装载（保留）
- `法决 全部` / `法决 已学` = 已学列表（替代 `法决列表`）

### E. 装备/卸下不对称

`装备 {{itemName}}` 按物品名/编号，`卸下 {{slotName}}` 按部位名。用户可能混淆。

**建议**：`卸下` 额外支持物品编号（`卸下 1`），与 `装备` 操作习惯对齐。
