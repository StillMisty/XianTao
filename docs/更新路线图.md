# 更新路线图

---

## Phase 1 — 战斗系统核心重构（优先级：高）

**目标：** 建立可扩展的战斗框架

| 任务 | 说明 | 文件 |
|------|------|------|
| 重构战斗引擎 | 创建 `CombatEngine` 接口，支持不同战斗场景 | `service/CombatService.java` |
| 实现 `BattleContext` | 封装战斗状态和配置 | `domain/monster/BattleContext.java` |
| 动态遇怪间隔 | 组合因素计算：地图难度、玩家等级、装备评分 | `service/CombatService.java` |
| 完善战斗日志 | 记录遇敌场次、击杀数、战败次数等统计 | `domain/monster/vo/BattleResultVO.java` |

**预计工期：** 1-2周

---

## Phase 2 — 技能效果系统完善（优先级：中）

**目标：** 实现所有技能效果类型

| 任务 | 说明 | 文件 |
|------|------|------|
| Buff/Debuff 系统 | 创建 `Buff` 实体和 `BuffType` 枚举 | `domain/monster/Buff.java` |
| 破甲效果 | 降低目标防御力，持续3-5回合 | `service/CombatService.java` |
| 减速效果 | 降低目标速度，持续2-3回合 | `service/CombatService.java` |
| 持续伤害 | 每回合造成固定伤害，持续3-5回合 | `service/CombatService.java` |
| 控制效果 | 眩晕/冰冻/沉默，跳过目标行动 | `service/CombatService.java` |
| 治疗效果 | 恢复目标生命值 | `service/CombatService.java` |
| 增益效果 | 提升目标属性（攻击/防御/速度） | `service/CombatService.java` |

**预计工期：** 1周

---

## Phase 3 — 灵兽AI行为系统（优先级：中）

**目标：** 让灵兽有智能的战斗行为

| 任务 | 说明 | 文件 |
|------|------|------|
| 创建 `BeastAI` 接口 | 定义行为决策方法 | `domain/beast/BeastAI.java` |
| 实现配合玩家策略 | 优先攻击对玩家威胁最大的敌人 | `service/CombatService.java` |
| 启用灵兽技能 | 修改 `BeastCombatant.getSkills()` 返回实际技能 | `service/CombatService.java` |
| 灵兽行动日志 | 记录灵兽的行动和决策 | `domain/monster/vo/BattleResultVO.java` |

**预计工期：** 1周

---

## Phase 4 — 历练系统优化（优先级：中）

**目标：** 完善历练体验

| 任务 | 说明 | 文件 |
|------|------|------|
| 等级衰减机制 | 高于地图等级10级后开始衰减，每级衰减5% | `service/TrainingService.java` |
| 精简输出格式 | 统计数据为主，文字MUD风格 | `handle/command/MapCommandHandler.java` |
| 高光战斗识别 | 识别势均力敌、回合数多、稀有触发的战斗 | `service/CombatService.java` |
| LLM美化渲染 | 每次历练最多1场高光战斗用LLM渲染 | `handle/command/MapCommandHandler.java` |
| 历练预览功能 | 预计收益、遇怪概率等 | `service/TrainingService.java` |

**预计工期：** 3-5天

---

## Phase 5 — LLM 叙事（独立后续）

| 文件 | 说明 |
|------|------|
| `handle/command/CombatCommandHandler.java` | 引入 LLM 将 `CombatResultVO` 美化渲染为修仙小说风格叙述文本 |
| `CombatResultVO` | 携带足够的战斗上下文（法器名、怪物名、法决名、伤害值）供 LLM 发挥 |

---

## 实施顺序依赖

```
Phase 1 ─── 战斗系统核心重构（基础框架）
    │
    ├── Phase 2 ─── 技能效果系统（依赖 Phase 1 的战斗框架）
    │
    ├── Phase 3 ─── 灵兽AI行为（依赖 Phase 1 的战斗框架）
    │
    └── Phase 4 ─── 历练系统优化（依赖 Phase 1-3 的功能）
            │
            └── Phase 5 ─── LLM 叙事（依赖 Phase 1-4 的数据输出）
```
