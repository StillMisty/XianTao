# 更新路线图

基于 2026-05-05 游玩体验审查 + 2026-05-06 全面代码审查结论整理，按优先级排序。

---

## P0 — CRITICAL（运行时故障 / 数据损坏）

### PillResistance 实体缺少 @Id 注解

- **问题：** `domain/pill/entity/PillResistance.java:11` — 无 `@Id`，`PillResistanceRepositoryImpl.incrementCount()` 调用 `mapper.update(pr)` 时 MyBatis-Flex 无法生成 WHERE 条件，会更新**全表所有行**。DB 有复合主键 `(user_id, template_id)` 但实体未映射。
- **修复：** 添加 `@Id` 注解（或添加代理 `id` 字段 `@Id(keyType = KeyType.Auto)`）。

### PillResistanceRepositoryImpl 竞态条件

- **问题：** `infrastructure/repository/PillResistanceRepositoryImpl.java:24-35` — read-check-act 模式，并发下两个请求同时查到 `isEmpty==true` → 同时 insert → 重复键冲突。
- **修复：** 改用 PostgreSQL `INSERT ... ON CONFLICT (user_id, template_id) DO UPDATE SET count = xt_pill_resistance.count + 1` 原子 upsert。

### SpiritEmotionTools 不安全 ScopedValue.get()

- **问题：** `service/ai/SpiritEmotionTools.java:26,51` — 直接调用 `UserContext.CURRENT_USER.get()` 未先检查 `isBound()`，ScopedValue 未绑定时抛出 `NoSuchElementException`。
- **修复：** 改用 `UserContext.getCurrentUserId()`（返回 null 不抛异常），或先调用 `UserContext.CURRENT_USER.isBound()`。

## P1 — HIGH（严重缺陷 / 可能数据不一致）

### BountyService 多处缺少 @Transactional

- **问题：**
  - `BountyService.java:129-177` `startBounty()` — 连续执行 `userBountyRepo.save` + `user.setStatus` + `userStateService.save`，无事务包裹
  - `BountyService.java:278-297` `abandonBounty()` — 同上
- **修复：** 添加 `@Transactional`。

### 缺失 @EnumValue 的枚举（MyBatis-Flex 映射失败）

| 枚举 | DB 列 | 文件 |
|------|-------|------|
| `WeaponType` | `Equipment.weaponType` | `domain/item/enums/WeaponType.java` |

### AuthenticatedAspect 丢失原始错误码

### BountyRewardListTypeHandler 使用不兼容 Jackson 版本

## P2 — MEDIUM（设计 / 可维护性）

### BeastService 上帝类（1077 行）

- **问题：** `service/BeastService.java` — 承担 8+ 职责：孵化、释放、进化、突破、部署/召回、治疗、产能收集、技能解锁、经验、变异。
- **重构：** 拆分为 `BeastBreedingService`、`BeastCombatService`、`BeastProductionService`、`BeastSkillService`。

### FudiService ↔ FarmService/BeastService 循环耦合

- **问题：** `FudiService.java:361-375` 调用 `farmService.updateGrowthProgress()`、`beastService.updateBeastProduction()` 等多个**包私有**方法，而 Farm/Beast 又依赖 FudiHelper。三者难以独立测试和变更。
- **重构：** 提取 `FudiCellCoordinator` 作为协调层，或用事件解耦。

### Monster 域 10 个 @Component Bean 误放 domain 包

- **问题：** `domain/monster/` 中 `DefaultCombatEngine`、`DamageCalculator`、`EncounterCalculator`、`HighlightBattleDetector` 等都是 `@Component` Spring Bean，属于服务层而非领域层。
- **重构：** 移动到 `service/combat/`。

### Command Handler 中包含业务逻辑（违反 AGENTS.md）

- **问题：**
  - `FudiCommandHandler.handleBuild()` — 调用 `CellType.fromChineseName()` 并捕获异常
  - `MapCommandHandler.handleStartBounty()` — `Long.parseLong()` + 异常处理
  - `FudiCommandHandler.handleCollect()` — 条件分支和字符串拼接
- **修复：** 输入解析和类型映射移入 Service 层。

### SpiritChatService 绕过 Repository 直接调用 Mapper

- **问题：** `service/ai/SpiritChatService.java:69` — `spiritFormMapper.selectOneById()` 直接调用 Mapper，其余服务全部走 Repository → Mapper。
- **修复：** 通过 `SpiritFormRepository` 访问。

### SpiritChatService 复制 FarmService 逻辑

- **问题：** `service/ai/SpiritChatService.java:260-284` — `calculateGrowthProgress()` 和 `getCropName()` 从 FarmService 复制粘贴。
- **修复：** 调用 FarmService 公共方法，或提取共享工具。

### SpiritPromptTemplates 负好感度处理错误

- **问题：** `service/ai/SpiritPromptTemplates.java:19` — `spiritAffection / 200` 整数除法，负数（如 -100）得 0，落入默认分支而不触发更冷淡对待。
- **修复：** 添加负值分支处理。

### PillRecipeService.learnRecipe() 返回 null

- **问题：** `service/PillRecipeService.java:89-118` — 多处失败返回 null，public 方法将 null 包裹为 `ServiceResult.Success<>(null)`，导致调用方拿到空数据误判成功。
- **修复：** 失败时 throw 或返回 `ServiceResult.Failure`。

### BeastService 返回 ServiceResult\<Object\>（类型不安全）

- **问题：** `service/BeastService.java:113,121` — public API 用 raw `Object` 作泛型参数。
- **修复：** 定义 sealed VO 接口涵盖 `ActionResultVO | BatchCountVO`，用 `ServiceResult<BeastResultVO>`。
