# 更新路线图

基于 2026-05-05 游玩体验审查结论整理，按优先级排序。

---

## P0 — 严重（角色卡死 / 数据损坏）

### 历练死亡卡死 + DYING 无恢复机制

- **问题：** 历练战败后状态变为 DYING，但 `endTraining()` 要求 status == EXERCISING，玩家无法结束历练也无法做任何事，角色永久废掉。`UserStateService.resolveState()` 也不处理 DYING 恢复。
- **修复：**
  1. `TrainingService.endTraining()`：如果状态是 DYING，仍然结算历练，奖励为 0，将状态重置为 IDLE
  2. `TrainingService` 历练中战败时：直接结算训练（奖励 0）并恢复 IDLE，不抛给玩家处理
  3. `UserStateService.resolveState()`：新增 DYING → IDLE 的超时恢复（如 30 分钟后自动恢复，或固定 HP 恢复到 20%）

### PvP 防守方绕过状态解析

切磋，双方都以满状态进行，PVP后的状态也不必写入数据库，相当于只是模拟。

### 福地收获后删除地块行

- **问题：** `FudiService.collectAll()` 对一次性作物调用 `deleteById` 直接删库行，多次收获后玩家地块逐渐归零。
- **修复：**
  1. 改为 `cell.setCellType(EMPTY); cell.setConfig(null); cell.setPlantedAt(null);` 原地重置
  2. 多年生作物在最后一次收获后同样重置为 EMPTY

### 悬赏兽卵奖励 ID/名称错位

- **问题：** `BountyService.java:343-345` 两次独立调用 `rng.nextInt()`，ID 和名称可能来自不同模板。
- **修复：** 提取为 `int idx = rng.nextInt(eggs.size())`，ID 和名称用同一个索引。

## P1 — 主要（体验严重受损）

### 查看玩家面板攻击/防御硬编码为 0

- **问题：** `PlayerViewService.java:59-60` 写死 `attack=0, defense=0`。
- **修复：** 通过 `CharacterStatusService` 或直接计算有效攻击/防御填真实值。

### 错误提示格式不统一

- **问题：** FudiCommandHandler 用 `"❌ "` 前缀，其他 Handler 裸返回。
- **修复：** 统一格式化工具类，所有 Failure 消息输出 `❌ <消息>`，Success 不加前缀。
