## Phase 3 — 法决系统

### 3.1 DB Migration

**文件**: `V2.2.0__create_spell.sql`

```
新建 xt_spell 表
  ├── effect_type       DAMAGE/ARMOR_BREAK/SLOW/EXECUTE/LIFESTEAL/MULTI_HIT/DOT
  ├── binding_type      NONE/WEAPON_TYPE/WEAPON_CATEGORY/ELEMENT/WEAPON_AND_ELEMENT
  ├── bind_weapon_type  可空
  ├── bind_category     可空
  ├── bind_element      可空（预留）
  ├── cooldown_seconds
  ├── damage_formula    "wis*3 + 20"
  ├── effect_params     JSONB
  ├── require_level / require_wis / require_spell_id
```

**文件**: `V2.2.1__create_user_spell.sql`

```
新建 xt_user_spell 表
  ├── user_id      FK → xt_user(id)
  ├── spell_id     FK → xt_spell(id)
  ├── is_slotted   BOOLEAN
  ├── slot_index   INT（1-5）
  ├── UNIQUE(user_id, spell_id)
```

## 实施顺序依赖

```
Phase 1 ────────────────────────── 先行，是后面所有系统的基础
  │
  ├─ Phase 2 ─── 依赖 Phase 1 的 EquipmentTemplate + WeaponType
  │
  ├─ Phase 3 ─── 依赖 Phase 1 的 WeaponType/Category
  │             依赖 Phase 2 的 CombatService（法决在战斗里触发）
  │
  └─ Phase 4 ─── 依赖 Phase 2+3 的 CombatResultVO 完整数据
```
