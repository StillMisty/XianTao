# 地灵好感度与精力系统设计

> 基于 2026-04-29 讨论结果沉淀

---

## 1. 核心概念

- **好感度 (spiritAffection)**：地灵对玩家的情感值，连续数值，无等级/分段概念。上限 **1000**。
- **精力 (spiritEnergy)**：地灵的活力值，每次服务消耗，随时间自然恢复。上限随劫数成长。
- **形态 (SpiritForm)**：地灵的外观形象（如小狐妖、春秋蝉等），创建时随机分配，不可更改。
- **MBTI人格**：地灵的性格类型，创建时玩家选择，不可更改。

---

## 2. 好感度系统

### 2.1 上限

```
好感度上限 = 1000
```

好感度不会超过此上限。

### 2.2 获取途径

| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫胜利 | +5 | 每次渡劫成功固定增加 |
| 赠送地灵所爱礼物 | +10~50 | 物品 tags 命中 `liked_tags`，幅度取决于 rarity |
| 赠送中性礼物 | +1~3 | tags 既不命中喜欢也不命中讨厌，出于礼貌 |
| 地灵日常对话 | +1 | 每次 LLM 对话有概率增加（低频，每日有上限） |

### 2.3 失去途径

| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫失败 | -clearCount | 按被毁地块数扣除 |
| 赠送地灵讨厌的礼物 | -5~20 | 物品 tags 命中 `disliked_tags`，幅度取决于 rarity |
| 玩家说了冒犯的话 | -1~5 | LLM 通过 `reportPlayerOffense` 工具上报，severity=1~5 |

### 2.4 作用

| 作用 | 公式 |
|------|------|
| **精力消耗减免** | `实际消耗 = 基础消耗 × (1 - min(0.5, affection / 1000))`，满好感最多减半 |
| **灵兽进化成功率加成** | `min(15, affection / 7)`，保持现有 |
| **灵兽品质突破成功率加成** | `min(20, affection / 5)`，保持现有 |
| **LLM对话语气** | Prompt 注入好感度描述，高好感亲密/低好感冷淡 |
| **渡劫怜悯** | `affection ≥ 800` 且 `defense ≥ attack × 80%` 时触发（见第5节） |

## 3. 精力系统

### 3.1 上限公式

```
精力上限 = 100 + 劫数 × 20
```

### 3.2 恢复机制（懒更新）

- **触发时机**：任何使用地灵的操作（对话、查看状态、福地操作等）
- **恢复量**：`精力上限 × 5% × 距上次使用小时数`
- **上限**：最多恢复到 `精力上限`
- **特点**：约 20 小时从 0 回满；比例恢复使高劫数玩家单位时间恢复绝对值更高

### 3.3 消耗范围

通过 `@ConsumeSpiritEnergy` 注解在 Service 内部方法上，由 AOP 切面统一处理。

| 操作 | 基础消耗 | 备注 |
|------|---------|------|
| LLM 对话 | 5 | `SpiritChatService.chatWithSpirit()` |
| 种植 | 3 | `FudiService.plantCrop()` |
| 收获（单次） | 5 | `FudiService.harvestCrop()` |
| 批量收获 | 5 × 数量 | `FudiService.harvestAllCrops()` |
| 建造 | 3 | `FudiService.buildCell()` |
| 拆除 | 3 | `FudiService.removeCell()` |
| 孵化 | 10 | `FudiService.hatchBeast()` |
| 进化/突破 | 8 | `FudiService.evolveBeastTier()` / `breakthroughBeastQuality()` |
| 收取产出 | 3 | `FudiService.collectProduce()` / `collectAllProduce()` |

### 3.4 AOP 切面逻辑

```java
@ConsumeSpiritEnergy(5)  // 标注在内部方法上
```

**SpiritEnergyAspect 执行流程：**
1. 从方法参数提取 `userId`（约定第一个 Long 类型参数）
2. 查询 Fudi
3. **懒恢复**：计算 `(当前时间 - lastEnergyUpdate)` 的小时数 × 上限 × 5%
4. **好感减免**：`实际消耗 = 基础消耗 × (1 - min(0.5, affection / 1000))`
5. 精力不足 → 允许执行但消耗到 0，触发 `FATIGUED` 情绪
6. 更新 `lastEnergyUpdate`，保存 Fudi

## 4. 地灵形态系统

### 4.1 分配方式

- 创建福地时 **随机分配**，不可更改
- 形态要符合世界观设定，可有些梗

### 4.2 存储结构

形态数据存于 `xt_spirit_form` 表，每个形态预定义所爱/所厌的 tag：

```sql
CREATE TABLE xt_spirit_form (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE,
    description TEXT NOT NULL,
    liked_tags JSONB NOT NULL DEFAULT '[]',
    disliked_tags JSONB NOT NULL DEFAULT '[]'
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | VARCHAR(50) UNIQUE | 形态名 |
| `description` | TEXT | 梗向描述 |
| `liked_tags` | JSONB | 该形态喜欢的物品 tag 池，如 `["wine","art","book","prestige","shiny"]` |
| `disliked_tags` | JSONB | 该形态讨厌的物品 tag 池，如 `["common","ore","forge_base","bone","slime"]` |

Fudi 实体通过 `form` 字段关联形态名。

### 4.3 喜好生成（随机抽取）

创建福地时自动触发，无需 LLM 介入：

```
1. 根据随机分配的形态名，查询 xt_spirit_form 获取 liked_tags 和 disliked_tags 池
2. 从 liked_tags 池中随机抽取 3~5 个，作为该地灵的 likes
3. 从 disliked_tags 池中随机抽取 2~3 个，作为该地灵的 dislikes
4. 存入 Fudi.spiritConfig JSONB（运行时直接读 JSONB，无需联表）
```

这种设计的好处：
- **零 LLM 依赖**：创建福地瞬间完成
- **批量更新友好**：新增物品 tag 后，只需更新 `xt_spirit_form` 的 tag 池，新创建的福地自然包含新 tag
- **个性化**：同一种形态的不同地灵，因随机抽取而可能有不同的喜好组合

```json
{
  "form": "fox",
  "form_name": "小狐妖",
  "liked_tags": ["wine", "book", "shiny"],
  "disliked_tags": ["ore", "slime"]
}
```

### 4.4 送礼判定

玩家通过 `#地灵送礼 [物品名]` 或 LLM 调用 `giveGift` 工具送礼时，从 Fudi.spiritConfig 获取 `liked_tags` / `disliked_tags` 进行匹配：

| 匹配 | 好感变化 | LLM 回应 |
|------|---------|---------|
| tags 命中 `liked_tags` | +10~50（依 rarity 递增） | 开心、惊喜 |
| tags 命中 `disliked_tags` | -5~20 | 嫌弃、生气 |
| 均未命中 | +1~3（礼貌收下） | 平淡感谢 |

## 5. 渡劫怜悯机制

### 5.1 触发条件

```
affection ≥ 800（上限 1000 的 80%）
AND defense ≥ attack × 80%
AND defense < attack（确实会失败）
```

### 5.2 效果

| 项目 | 变化 |
|------|------|
| 天劫结果 | **视为胜利**（劫数+1、连胜+1、灵气上限增长） |
| 好感度 | 不加好感（正常渡劫胜利给 +5，怜悯模式不给） |
| 精力 | **归零** |
| 情绪 | → `FATIGUED` |

### 5.3 LLM 叙事美化

渡劫怜悯触发后不直接返回普通文本，而是将结构化结果传给 **LLM** 生成叙事文本：

```
输入：地灵形态、情绪、精力归零、劫数+1、attack/defense 数值
输出："小狐妖燃烧灵体，九尾化作光幕挡住了天雷……她虚弱地倒在你怀里"
```

## 6. LLM 冒犯上报

在 `SpiritTools` 中添加工具，让 LLM 在自然对话中判断玩家是否说了冒犯的话：

```java
@Tool(description = "当玩家说了不礼貌、冒犯、或让地灵不开心的话时调用此工具")
void reportPlayerOffense(
    @ToolParam(description = "玩家说了什么") String reason,
    @ToolParam(description = "冒犯程度 1~5") int severity
)
```

- 由 LLM 自主判断是否触发，无需硬编码关键词
- severity 影响好感扣除量（-severity）
