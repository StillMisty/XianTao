# 地图系统设计文档

## 1. 模块概述

地图系统是仙道游戏的世界架构基础，负责管理地理位置、区域连接、特产掉落和旅行事件等核心功能。

### 1.1 核心职责

- 地图节点管理（名称、描述、等级要求）
- 地图连通性（相邻关系、旅行耗时）
- 特产掉落池（加权随机掉落）
- 旅行事件系统（D20判定、事件触发）
- 地图访问权限控制（等级限制）

## 2. 数据模型设计

### 2.1 地图节点 (xt_map_node)

#### 2.1.1 表结构说明

| 字段名                 | 类型           | 说明              |
|---------------------|--------------|-----------------|
| id                  | BIGSERIAL    | 地图ID（主键）        |
| name                | VARCHAR(128) | 地图名称            |
| description         | TEXT         | 地图描述            |
| map_type            | VARCHAR(32)  | 地图类型（主城、野外、秘境等） |

| specialties         | JSONB        | 历练掉落池/特产（加权列表）  |

| create_time         | TIMESTAMP    | 创建时间            |
| update_time         | TIMESTAMP    | 更新时间            |

#### 2.1.2 JSONB字段结构

**neighbors（相邻地图）**：

```json
{
  "黑金主城": 5,
  "枯骨林": 10,
  "迷雾沼泽": 15
}
```

- Key：相邻地图名称
- Value：旅行耗时（分钟）

**specialties（特产掉落池）**：

```json
[
  {
    "name": "毒龙草",
    "templateId": 101,
    "weight": 30
  },
  {
    "name": "铁矿石",
    "templateId": 202,
    "weight": 50
  },
  {
    "name": "灵泉",
    "templateId": 303,
    "weight": 20
  }
]
```

- name：物品名称
- templateId：物品模板ID
- weight：掉落权重（数值越大越容易掉落）

**travel_events（旅行事件权重）**：

```json
[
  {
    "eventType": "ambush",
    "weight": 40
  },
  {
    "eventType": "find_treasure",
    "weight": 10
  },
  {
    "eventType": "weather",
    "weight": 20
  }
]
```

- eventType：事件类型（ambush/find_treasure/weather/safe_passage）
- weight：触发权重

## 3. 核心业务规则

### 3.1 地图连通性

#### 3.1.1 有向图结构

地图系统本质上是一个**有向加权图**：

- **节点**：地图（MapNode）
- **边**：相邻关系（带权重=旅行耗时）
- **方向性**：A→B的旅行耗时可能与B→A不同

#### 3.1.2 相邻判定

```java
boolean isAdjacentTo(String targetMapName) {
    return neighbors.containsKey(targetMapName);
}
```

**示例**：

- 黑金主城的neighbors包含"枯骨林": 10
- 则从黑金主城可以前往枯骨林，耗时10分钟
- 若枯骨林的neighbors不包含"黑金主城"，则无法反向通行

### 3.2 地图访问权限

#### 3.2.1 等级限制检查

**设计意图**：

- 防止低等级玩家误入高等级危险区域
- 引导玩家按等级梯度探索世界

**示例**：

- 枯骨林等级要求：10级
- 玩家等级：8级 → 无法进入
- 玩家等级：10级 → 可以进入

### 3.3 旅行耗时查询

#### 3.3.1 获取到指定地图的旅行时间

**返回值**：

- 存在相邻关系：返回旅行耗时（分钟）
- 不存在相邻关系：返回null

## 4. 特产掉落系统

### 4.1 加权随机算法

#### 4.1.1 算法原理

根据物品的weight属性进行加权随机抽取，权重越高掉落概率越大。

**计算步骤**：

1. 计算总权重：`totalWeight = sum(all weights)`
2. 生成随机数：`random = [0, totalWeight)`
3. 遍历物品列表，累加权重
4. 当`random < currentWeight`时，选中该物品

#### 4.1.2 掉落概率计算

假设掉落池配置：

```json
[
  {
    "name": "毒龙草",
    "weight": 30
  },
  {
    "name": "铁矿石",
    "weight": 50
  },
  {
    "name": "灵泉",
    "weight": 20
  }
]
```

总权重 = 30 + 50 + 20 = 100

**掉落概率**：

- 毒龙草：30/100 = 30%
- 铁矿石：50/100 = 50%
- 灵泉：20/100 = 20%

### 4.2 历练掉落应用

**掉落次数计算**：

```
dropChances = (历练分钟数 / 10) × 效率倍率
效率倍率 = 1.0 + 敏捷 × 0.01
```

**示例**：

- 历练60分钟，敏捷20点
- 效率倍率 = 1.0 + 20×0.01 = 1.2
- 掉落次数 = (60/10) × 1.2 = 7.2 → 7次

## 5. 旅行事件系统

### 5.1 D20骰点判定

#### 5.1.1 事件触发规则

到达目的地时触发D20（20面骰）判定：

**判定逻辑**：

- D20 ≤ 10：触发随机事件（50%概率）
- D20 > 10：安全通行（50%概率）

#### 5.1.2 事件类型选择

当事件触发时，根据地图的travel_events配置进行加权随机选择。

**示例配置**：

```json
[
  {
    "eventType": "ambush",
    "weight": 40
  },
  {
    "eventType": "find_treasure",
    "weight": 10
  },
  {
    "eventType": "weather",
    "weight": 20
  }
]
```

总权重 = 40 + 10 + 20 = 70

**事件概率**：

- 伏击(ambush)：40/70 ≈ 57.1%
- 发现宝藏(find_treasure)：10/70 ≈ 14.3%
- 恶劣天气(weather)：20/70 ≈ 28.6%

## 6. 旅行系统设计

### 6.1 移动指令

玩家通过 `#前往` 指令在不同地图间移动，支持两种模式：

- **自动模式**：`#前往 <目标地图名称>` — 优先使用体力快速移动（`5 × 旅行耗时分钟`体力），体力不足时自动降级为真实时间模式
- **等待模式**：`#前往 <目标地图名称> 等待` — 强制使用真实时间模式，无论体力是否充足

### 6.2 双模式旅行

#### 6.2.1 真实时间模式

**流程**：

1. 检查玩家状态（必须为IDLE）
2. 验证地图连通性和等级要求
3. 设置玩家状态为RUNNING
4. 记录旅行开始时间和目的地
5. 返回预计到达时间

**到达后**：

1. 玩家手动触发完成旅行
2. 检查是否已到达（当前时间 ≥ 开始时间 + 旅行耗时）
3. 触发D20事件判定
4. 更新玩家位置为目的地
5. 恢复玩家状态为IDLE

**优势**：

- 增强沉浸感，模拟真实旅行过程
- 鼓励玩家长期在线规划行程

#### 6.2.2 体力跳过模式

**流程**：

1. 检查玩家状态（必须为IDLE）
2. 验证地图连通性和等级要求
3. 计算体力消耗：`旅行耗时(分钟) × 5`
4. 检查体力是否足够
5. 扣除体力
6. 立即触发D20事件判定
7. 更新玩家位置为目的地
8. 保持玩家状态为IDLE

**优势**：

- 快速移动，适合紧急场景
- 消耗资源（体力）换取时间

### 6.3 旅行约束

#### 6.3.1 前置条件

- 玩家状态必须为IDLE（空闲）
- 目标地图必须与当前地图相邻
- 玩家等级必须满足目标地图要求

#### 6.3.2 状态限制

- 旅行中（RUNNING状态）无法执行其他操作（历练、探索等）
- 必须先完成旅行才能进行其他活动

## 7. 地图类型设计

### 7.1 地图分类

| 类型         | 说明 | 特点                |
|------------|----|-------------------|
| CITY       | 主城 | 安全区域，无旅行事件，NPC聚集  |
| WILDERNESS | 野外 | 普通区域，中等风险，常见材料    |
| DUNGEON    | 秘境 | 高风险高回报，稀有材料，高等级要求 |
| SPECIAL    | 特殊 | 剧情地图、活动地图等        |

### 7.2 地图设计原则

#### 7.2.1 等级梯度

- 新手村：0级
- 初级野外：5-10级
- 中级区域：15-30级
- 高级秘境：40+级

#### 7.2.2 连通性设计

- 每个地图至少有1个相邻地图（避免孤立）
- 主城作为交通枢纽，连接多个区域
- 高等级地图通过多条路径可达（增加策略性）

#### 7.2.3 特产差异化

- 不同地图拥有独特的特产掉落池
- 鼓励玩家探索多个区域收集资源
- 高等级地图产出稀有材料

## 附录A：核心公式汇总

### A.1 旅行耗时

- 从地图A到地图B：`A.neighbors[B的名称]`（分钟）

### A.2 体力消耗（体力模式）

- 体力消耗：`旅行耗时(分钟) × 5`

### A.3 掉落概率

- 物品掉落概率：`物品权重 / 总权重`

### A.4 事件触发

- 事件触发概率：`D20 ≤ 10`（50%）
- 特定事件概率：`事件权重 / 总权重`
