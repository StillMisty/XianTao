# 物品与装备系统

## 1. 物品模板 (xt_item_template)

### 1.1 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 物品名称（UNIQUE） |
| `type` | VARCHAR(32) | 物品类型 |
| `properties` | JSONB | 类型特有属性（种子/丹药/材料等） |
| `tags` | JSONB | 物品标签 (GIN) |
| `max_stack` | INT | 最大堆叠 |
| `description` | TEXT | 描述 |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

**注意**: `rarity` 列已移除。稀有度在装备实例化时加权随机决定，详见 §3。

## 2. 装备模板 (xt_equipment_template)

### 2.1 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `template_id` | BIGINT | FK → xt_item_template(id) |
| `slot` | VARCHAR(32) | 装备部位 (WEAPON/ARMOR/ACCESSORY) |
| `weapon_type` | VARCHAR(32) | 法器子类型，可空（护甲/饰品为 null） |
| `category` | VARCHAR(16) | 法器大类：刀兵/长兵/远兵/奇兵（护甲/饰品为 null） |
| `equip_level` | INT | 装备等级 |
| `base_attack` | INT | 基础攻击 |
| `base_defense` | INT | 基础防御 |
| `base_str` | INT | 基础力量加成 |
| `base_con` | INT | 基础体质加成 |
| `base_agi` | INT | 基础敏捷加成 |
| `base_wis` | INT | 基础智慧加成 |
| `attack_speed` | DECIMAL(3,1) | 攻速（法器专属，如 1.2 剑快 / 0.7 斧慢） |
| `attack_range` | VARCHAR(16) | 近战/远程（法器专属） |
| `drop_weight` | JSONB | 稀有度掉落权重 |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 2.2 drop_weight JSONB

```json
{"BROKEN": 60, "COMMON": 25, "RARE": 10, "EPIC": 4, "LEGENDARY": 1}
```

key 为 `Rarity.code`，value 为权重（整数）。

### 2.3 装备部位 (EquipmentSlot)

| 枚举值 | 中文名 | 说明 |
|------|------|------|
| WEAPON | 法器 | 主要攻击源，有 weapon_type + category |
| ARMOR | 护甲 | 主要防御源，无 weapon_type |
| ACCESSORY | 饰品 | 属性辅助，无 weapon_type |

> `WEAPON` 中文名已从"武器"改为"法器"，所有显示文本统一。

### 3.3 克制规则

```
法器克制怪物 → 玩家伤害 ×1.5，怪物伤害 ×0.8
法器被怪物克制 → 玩家伤害 ×0.8，怪物伤害 ×1.5
无克制关系 → 倍率 1.0
```

## 4. 装备实例化 —— 稀有度加权随机

### 4.1 流程

装备获取时统一走 `ItemService.createEquipment(templateId)`：

```
① 获取 EquipmentTemplate.drop_weight
② 加权随机 roll 出 Rarity（BROKEN/COMMON/RARE/EPIC/LEGENDARY）
③ 在 Rarity 的 qualityMultiplier 范围内随机取一个值
④ 在 Rarity 的 affixCount 范围内随机词条数，从词条池中随机选取
⑤ 从 Rarity 的 prefixes 列表中随机选前缀 + 模板名 = 装备名
⑥ Equipment.create(...) 持久化
```

### 4.2 稀有度体系 (Rarity)

| 稀有度 | 品质系数 | 词条数 | 前缀示例 |
|------|---------|--------|---------|
| BROKEN | 0.80–0.99 | 0 | 破烂的、生锈的 |
| COMMON | 1.00–1.15 | 1 | 普通的、结实的 |
| RARE | 1.16–1.35 | 1–2 | 锋利的、精工的 |
| EPIC | 1.36–1.60 | 2–3 | 卓越的、附魔的 |
| LEGENDARY | 1.61–2.00 | 3–4 | 传说的、完美的 |

### 4.3 属性计算公式

```
攻击 = 基础攻击 + floor(基础攻击 × qualityMultiplier) + forgeLevel × 5
防御 = 基础防御 + floor(基础防御 × qualityMultiplier) + forgeLevel × 5
总属性 = 基础属性 + 属性词条加成
```

## 5. 装备实例 (xt_equipment)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | 持有者 |
| `template_id` | BIGINT | 模板ID |
| `weapon_type` | VARCHAR(32) | 法器子类型（可空） |
| `name` | VARCHAR(128) | 实际名称（含品质前缀） |
| `slot` | VARCHAR(32) | 部位 |
| `rarity` | VARCHAR(32) | 稀有度 |
| `quality_multiplier` | DOUBLE | 品质乘数 |
| `affixes` | JSONB | 随机词条 (GIN) |
| `stat_bonus` | JSONB | 属性加成 (GIN) |
| `attack_bonus` | INT | 攻击加成 |
| `defense_bonus` | INT | 防御加成 |
| `forge_level` | INT | 锻造等级（默认 0） |
| `equipped` | BOOLEAN | 已装备 |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 5.1 词条 (AffixType)

| 词条 | 效果 |
|------|------|
| STRENGTH | 力量 +X |
| CONSTITUTION | 体质 +X |
| AGILITY | 敏捷 +X |
| WISDOM | 智慧 +X |
| LIFE_STEAL | 伤害的 X% 转化为 HP |
| TREASURE_HUNT | 寻宝率 +X% |

## 8. 背包管理

- 初始容量：50 格
- 装备：每件独立实例，不堆叠
- 材料/消耗品：按模板合并，受 `max_stack` 限制
- 添加物品时自动堆叠，背包满时返回失败
- 历练结算掉落物**自动进背包**，无需手动拾取
