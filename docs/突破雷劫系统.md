# 突破雷劫系统

## 1. 设计原则

- **大境界突破 = 战斗**：跨大境界突破和渡劫期内每一级，触发雷劫战斗
- **小境界突破 = RNG**：同一大境界内升级，保持原有概率骰子逻辑
- **复用战斗引擎**：雷劫 Boss 使用 `CombatService.simulate()`，与历练/秘境/PVP 共用同一引擎
- **丹药/护道转化为 Boss 削弱**：突破辅助丹药和护道者的 RNG 加成，在战斗场景中转化为雷劫 Boss 的属性削弱

---

## 2. 突破流程

### 2.1 小境界突破（不变）

```
经验达标 → RNG骰子（基础率 + 保底 + 护道 + 丹药）→ 成功/失败
```

### 2.2 跨大境界 + 渡劫期（新增）

```
经验达标 → 扣除经验 → 随机雷劫类型 → 生成雷劫Boss → 战斗 → 胜=突破 / 败=失败
```

## 3. 境界改名：每级独立命名

`CultivationRealm` 从「阶段制」（初窥/凝气/化液/圆满）改为「每层独立命名」：

| 大境界 | 层数 | 命名示例 |
|--------|------|---------|
| 炼气期 | 1-10 | 启灵/引气/炼息/凝旋/.../圆满 |
| 筑基期 | 11-20 | 开光/辟府/筑台/.../圆满 |
| ... | ... | ... |
| 渡劫期 | 111+ | 一劫/二劫/.../九劫/飞升劫/第N劫 |

每级 `realmDisplay()` 唯一，如 `"炼气期 · 启灵"`、`"渡劫期 · 三劫"`。

## 4. 雷劫类型（TribulationType）

共 7 种雷劫，按境界解锁 + 概率加权：

| 类型 | 难度 | 最低解锁 | 权重 | 技能效果 |
|------|------|---------|------|---------|
| 三清雷劫 | ×1.0 | 筑基 | 55% | 无 |
| 紫霄神雷 | ×1.3 | 筑基 | 22% | DAMAGE "attack*1.8" + EXECUTE(0.3) |
| 太乙青雷 | ×1.3 | 金丹 | 12% | DOT(0.15, 3回合, 3层) |
| 九霄金雷 | ×1.6 | 化神 | 7% | ARMOR_BREAK(0.3, 3回合) |
| 玄冥黑雷 | ×1.6 | 炼虚 | 3% | FREEZE(2回合) |
| 灭世神雷 | ×2.0 | 大乘 | 0.9% | AOE_DAMAGE "attack*1.3" |
| 九色神雷 | ×3.0 | 渡劫 | 0.1% | DAMAGE + DOT + ARMOR_BREAK + FREEZE |

境界解锁规则：
- 突破到筑基(1)~金丹(2)：仅 三清 + 紫霄
- 金丹(3)~元婴(4)：三清 + 紫霄 + 太乙
- 化神(5)~炼虚(6)：+ 九霄 + 玄冥
- 合体(7)~大乘(8)：+ 灭世
- 渡劫期每一级：全部 7 种

## 5. Boss 数值公式

基于玩家队伍属性缩放：

```
hpRate  = (1.5 + targetRealmOrdinal × 0.5) × type.difficultyMultiplier
atkRate = (0.8 + targetRealmOrdinal × 0.3) × type.difficultyMultiplier
defRate = (0.6 + targetRealmOrdinal × 0.25) × type.difficultyMultiplier
spdRate = (0.7 + targetRealmOrdinal × 0.2) × type.difficultyMultiplier
```

削弱因子（独立相乘）：

```
combinedReduction = (1 - bossReduction)       // 丹药+护道 (0~0.5)
                  × (1 - pityReduction)        // 保底 (0~0.5, 每次失败 +5%)
                  × max(0.1, 1 - tribulationResist)  // 雷劫抗性buff
```

渡劫期每级额外难度：`levelMultiplier = 0.5 + tribulationLevel × 0.5`

## 6. 丹药体系

### 6.1 突破辅助丹（原有，仅小境界突破有效）

筑基丹/结丹丹/碎丹丹/化婴丹/炼神丹/化神丹/融虚丹/大乘丹：按原 RNG 逻辑加成成功率。

### 6.2 雷劫抗性丹（新增）

| 丹药 | 类型 | 效果 | 品级 |
|------|------|------|------|
| 避雷丹 | tribulation_resist +15% | 雷劫 Boss 伤害降低 15% | basic |
| 天劫丹（修正） | tribulation_resist +25% | 从 defense buff 改为雷劫抗性 | epic |
| 化劫丹 | tribulation_resist +40% | 雷劫 Boss 伤害降低 40% | rare |
| 渡厄金丹 | tribulation_resist +60% | 雷劫 Boss 伤害降低 60% | legendary |
| 招雷散 | tribulation_resist -40% | Boss 难度 ×1.4，战胜额外 +50% 修为 | rare |

### 6.3 Buff 类型扩展

`PlayerBuffType` 新增 `TRIBULATION_RESIST("tribulation_resist", "雷劫抗性")`。DB CHECK 约束同步更新。

## 7. 战后处理

- **战胜**：等级提升、跨大境界时全属性 +20% + 灵石奖励、渡劫期每级 +5% 全属性、清除突破 buff 和护道关系
- **战败**：经验已扣除、失败计数 +1、清除突破 buff 和护道关系、HP 保持战斗后状态（可能残血/濒死）
- **灵兽**：复用 `CombatService.applyCombatHpToBeasts()`——死亡则取消部署 + 恢复计时，存活则自愈
- **LLM 叙事**：战斗结束后生成 AI 战斗过程描述，从战斗日志中提取回合数、HP 变化、技能使用、击杀/重创等关键事件，送入 ChatClient 生成仙侠古风叙事文本
