# 地图系统设计文档 (节点连线制)

地图系统的核心目标是**划分资源产出区域**，并通过"旅行事件"提供碎片化的游戏乐趣，绝不使用XY坐标走格子。

## 1. 地图节点数据结构 (Map Node)

每个地图是一个独立节点，连接着其他几个节点。

| 字段              | 描述       | 举例                                                        |
|:----------------|:---------|:----------------------------------------------------------|
| **Name**        | 地图名称     | `幽暗沼泽`                                                    |
| **Type**        | 区域类型     | `SAFE_TOWN` (主城/安全区)、`AFK_ZONE` (挂机区)、`HIDDEN_ZONE` (隐藏区) |
| **Level**       | 推荐等级阈值   | `level >= 5`                                              |
| **Neighbors**   | 相邻节点及耗时  | `{"黑金主城": 5分钟, "枯骨林": 10分钟}`                              |
| **Specialties** | 挂机掉落池/特产 | `毒龙草 (30%), 铁矿石 (50%), 史莱姆粘液`                             |
| **Events**      | 旅行事件表权重  | `遇袭(40%), 捡漏(10%), 毒雾天气(50%)`                             |

## 2. 核心移动机制

玩家只能在**相邻节点**间移动。

### 2.1 移动指令

* **自动模式**：`#前往 幽暗沼泽` - 优先使用体力（5体力/分钟），体力不足时使用真实时间
* **等待模式**：`#前往 幽暗沼泽 等待` - 强制使用真实时间

### 2.2 执行流程

1. 系统检查当前地图是否与"幽暗沼泽"相邻。
2. 检查玩家等级是否满足地图要求。
3. 检查玩家体力是否足够：
   - **体力足够**：消耗 `5 * travelTimeMinutes` 体力，立即完成移动
   - **体力不足**：设置玩家状态为 `RUNNING`，记录 `travelStartTime` 和 `travelDestinationId`
4. **暗骰判定 (D20)** 触发路途事件（1-10触发事件，11-20安全通行）。
5. 到达目的地，更新玩家 `locationId`，输出到达提示。

## 3. 历练与探索

### 3.1 历练系统 (`#历练`)

按照地图的 `Specialties` 掉落池，结合玩家的属性计算奖励：

| 属性           | 作用             |
|:-------------|:---------------|
| **敏捷 (AGI)** | 影响产出效率（奖励倍率）   |
| **体质 (CON)** | 影响挂机时长（体力消耗速度） |

**懒加载评估**：

- 不使用定时任务，玩家调用 `#历练` 后 `#历练停止` 时才计算收益
- 基于 `currentTime - afkStartTime` 计算时长
- 一次性发放奖励并更新 `afkStartTime`

### 3.2 探索系统 (`#探索`)

受**智慧 (WIS)** 和**地图等级**影响的主动技能：

- 消耗一定时间/体力进行单次深度搜寻
- 高智慧可能搜出该地图的隐藏配方、特殊灵蛋
- 地图等级越高，探索难度越大（智慧要求越高）
- 使用 Spring AI + LLM 生成美化的探索描述

**有效智慧计算**：

```
effectiveWisdom = wisdom - (mapLevel - 1) × 5
```

- 地图等级每高1级，探索难度增加5点
- 最低有效智慧为1

**探索奖励表**（基于有效智慧）：

| 有效 WIS 范围 | 可能获得             |
|:----------|:-----------------|
| 1-20      | 基础材料             |
| 21-50     | 稀有材料             |
| 51-80     | 稀有材料 + 小概率配方     |
| 81+       | 稀有材料 + 配方 + 特殊灵蛋 |

**经验值计算**：

```
经验 = 10 + (WIS × 2) + (mapLevel × 5)
```

- 地图等级越高，获得的经验越多

## 6. 技术实现要点

### 6.1 数据库设计

- **xt_map_node** 表：存储地图节点信息，使用 JSONB 存储 neighbors、specialties、events
- **xt_map_connection** 表：显式存储地图连接关系
- **xt_user** 表扩展：添加 `travel_start_time` 和 `travel_destination_id` 字段

### 6.2 架构模式

- 遵循 DDD 架构，domain 层不依赖 infrastructure 层
- 使用 MyBatis-Flex 进行 ORM 操作
- JSONB 字段使用 GIN 索引优化查询

### 6.3 Spring AI 集成

- 使用 `@Bean` + `@Description` 定义 LLM function
- `generateExplorationDescription(String mapName, String eventType, List<String> foundItems)`
- LLM 仅输出描述文本，奖励由 Java 后端添加

### 6.4 并发处理

- 所有 service 方法兼容 Virtual Threads
- 避免 ThreadLocal 阻塞操作
