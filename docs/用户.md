# 用户系统设计

## 1. 角色实体 (xt_user)

### 1.1 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 角色ID (PK) |
| `nickname` | VARCHAR(64) | 道号 (UNIQUE) |
| `level` | INT | 等级 (默认1) |
| `exp` | BIGINT | 经验值 |
| `coins` | BIGINT | 铜币 |
| `spirit_stones` | BIGINT | 灵石 |
| `stat_str/con/agi/wis` | INT | 四维属性 (默认各5) |
| `free_stat_points` | INT | 可用属性点 |
| `hp_current` | INT | 当前生命值 |
| `stamina_current` | INT | 当前体力值 |
| `status` | VARCHAR(32) | 当前状态 |
| `location_id` | BIGINT | 所在地图ID |
| `training_start_time` | TIMESTAMP | 历练开始时间 |
| `breakthrough_fail_count` | INT | 突破失败次数 |
| `last_reset_points_time` | TIMESTAMP | 上次洗点时间 |
| `last_stamina_update_time` | TIMESTAMP | 上次体力更新时间 |
| `travel_start_time` | TIMESTAMP | 旅行开始时间 |
| `travel_destination_id` | BIGINT | 旅行目的地 |
| `extra_data` | JSONB | 扩展字段 |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 1.2 索引

- `idx_xt_user_level` (level DESC) — 等级榜
- `idx_xt_user_coins` (coins DESC) — 财富榜
- `idx_xt_user_spirit_stones` (spirit_stones DESC) — 灵石榜
- `uq_nickname` — 道号唯一

## 3. 等级与经验

### 3.1 升级公式

```
升级所需经验 = 100 × level²
经验存储上限 = 升级所需经验 × 3
```

**示例**: 10级升11级需 `100 × 10² = 10000` 经验，最多存储 `30000`。

### 3.2 升级效果

1. 扣除升级所需经验
2. level += 1
3. free_stat_points += 1
4. 生命值回满
5. breakthrough_fail_count = 0
6. 清除所有被护道关系

## 4. 四维属性

| 属性 | 影响 |
|------|------|
| 力量 (STR) | 攻击力、锻造成功率 |
| 体质 (CON) | HP上限、体力上限、防御 |
| 敏捷 (AGI) | 历练效率倍率 |
| 智慧 (WIS) | 经验加成、探索掉宝率 |

### 4.1 属性公式

```
HP上限    = 100 + CON × 20
体力上限  = 100 + CON × 5
效率倍率  = 1.0 + AGI × 0.01
有效智慧  = max(1, WIS - (地图等级 - 1) × 5)
```

### 4.2 属性分配

每次升级获得 1 点自由属性点，可一次性分配多点到任意属性。

## 5. 突破系统

### 5.1 成功率公式

```
基础概率 = 100 / (1 + (level / 50)^4.5)
单次补偿 = 3 + 17 / (1 + (level / 50)²)
总成功率 = min(100, 基础概率 + 失败次数 × 单次补偿)
```

### 5.2 突破效果

**成功**: level+1, exp 扣除, 失败计数归零, HP 回满, 清除被护道关系
**失败**: 扣除修为, 失败计数+1, 清除被护道关系

### 5.3 护道加成

```
单人加成 = 5% + (护道者等级 - 被护道者等级) × 1%
总加成上限 = 20% (仅同地点护道者生效)
```

## 6. 洗点系统

- 所有属性重置为 5，返还已分配点数至自由属性池
- 冷却时间：72 小时

## 7. 护道系统 (xt_dao_protection)

| 字段 | 说明 |
|------|------|
| `protector_id` | 护道者 |
| `protege_id` | 被护道者 |
| UNIQUE(protector_id, protege_id) | 防重复 |

### 7.1 约束

- 护道者等级 ≥ 被护道者等级
- 护道者最多同时为 3 人护道
- 被护道者突破后自动清除所有护道关系

## 8. 体力系统

### 8.1 恢复速率

| 状态 | 速率 |
|------|------|
| IDLE (空闲) | 每5分钟1点 |
| MEDITATING (打坐) | 每分钟2点 |
| 其他 | 不恢复 |

### 8.2 消耗

| 操作 | 消耗 |
|------|------|
| 历练 | 不消耗体力 |
| 探索 | 固定20点 |
| 旅行(体力模式) | 5点/分钟 |
| 打坐 | 恢复体力 |

### 8.3 懒加载机制

系统不在后台定时更新体力，而是在玩家查询或操作时即时计算：

```
if (status == MEDITATING) 恢复量 = 分钟差 × 2
else if (status == IDLE)  恢复量 = 分钟差 / 5 (整除)
else                      恢复量 = 0
实际恢复量 = min(恢复量, 体力上限 - 当前体力)
```
