# 用户系统设计

## 1. 角色实体 (xt_user)

### 1.1 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 角色ID (PK) |
| `nickname` | VARCHAR(64) | 道号 (UNIQUE) |
| `level` | INT | 等级 (默认1) |
| `exp` | BIGINT | 经验值 |
| `coins` | BIGINT | 铜币 |
| `spirit_stones` | BIGINT | 灵石 |
| `stat_str/con/agi/wis` | INT | 四维属性（由等级决定，默认 5） |
| `hp_current` | INT | 当前生命值 |
| `status` | VARCHAR(32) | 当前状态 |
| `location_id` | BIGINT | 所在地图ID |
| `training_start_time` | TIMESTAMP | 历练开始时间 |
| `breakthrough_fail_count` | INT | 突破失败次数 |
| `travel_start_time` | TIMESTAMP | 旅行开始时间 |
| `travel_destination_id` | BIGINT | 旅行目的地 |
| `extra_data` | JSONB | 扩展字段 |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 1.2 已移除字段

| 字段 | 原因 |
|------|------|
| `free_stat_points` | 取消自由加点，属性由等级决定 |
| `last_reset_points_time` | 取消洗点系统 |
| `stamina_current` / `last_stamina_update_time` | 已废弃的体力系统 |

### 1.3 索引

- `idx_xt_user_level` (level DESC) — 等级榜
- `idx_xt_user_coins` (coins DESC) — 财富榜
- `idx_xt_user_spirit_stones` (spirit_stones DESC) — 灵石榜
- `uq_nickname` — 道号唯一

## 2. 跨平台授权 (xt_user_auth)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | FK → xt_user(id) CASCADE |
| `platform` | VARCHAR(32) | 平台类型 |
| `platform_open_id` | VARCHAR(128) | 平台唯一ID |
| `bind_time` | TIMESTAMP | 绑定时间 |

**约束**: UNIQUE(platform, platform_open_id)，同一平台账号只能绑定一个角色。

## 3. 等级与经验

### 3.1 升级公式

```
升级所需经验 = 100 × level²
经验存储上限 = 升级所需经验 × 5
```

### 3.2 突破效果

**成功**:
- level += 1
- 扣除升级所需经验
- 全属性 += 1（力量/体质/敏捷/智慧各+1）
- 失败计数归零
- HP 回满
- 清除所有被护道关系

**失败**:
- 扣除修为
- 失败计数 += 1
- 清除所有被护道关系

## 4. 四维属性

### 4.1 基础值

```
基础属性 = 4 + level

等级 1: 全属性 = 5
等级 50: 全属性 = 54
每升一级：全属性 +1
```

### 4.2 属性公式

| 属性 | 影响 |
|------|------|
| 力量 (STR) | 法器普攻伤害 `attack = str × 2 + equipAttack` |
| 体质 (CON) | HP上限 `100 + con × 20`，防御值 `defense = con + equipDefense` |
| 敏捷 (AGI) | 战斗效率（增加攻击轮次） |
| 智慧 (WIS) | 法决伤害 `wis × scale + base` |

### 4.3 差异化来源

玩家间的属性差异**不来自加点**，而来自：
- **装备**：每件装备有不同的 `base_str/con/agi/wis` 加成 + 随机词条
- **法器类型**：不同法器对怪物有不同克制倍率
- **法决**：智慧高 → 法决伤害高

## 5. 突破系统

### 5.1 成功率公式

```
基础概率 = 100 / (1 + (level / 50)^4.5)
单次补偿 = 3 + 17 / (1 + (level / 50)²)
总成功率 = min(100, 基础概率 + 失败次数 × 单次补偿)
```

### 5.2 护道加成

```
单人加成 = 5% + (护道者等级 - 被护道者等级) × 1%
总加成上限 = 20%（仅同地点护道者生效）
```

## 6. 护道系统 (xt_dao_protection)

| 字段 | 说明 |
|------|------|
| `protector_id` | 护道者 |
| `protege_id` | 被护道者 |
| UNIQUE(protector_id, protege_id) | 防重复 |

### 6.1 约束

- 护道者等级 ≥ 被护道者等级
- 护道者最多同时为 3 人护道
- 被护道者突破后自动清除所有护道关系

## 7. 用户状态 (UserStatus)

| 状态 | 说明 |
|------|------|
| IDLE | 空闲 |
| EXERCISING | 历练中 |
| RUNNING | 旅途中 |
| BOUNTY | 悬赏任务中 |
| DYING | 濒死（HP=0，无法行动） |

### 7.1 濒死 (DYING)

触发条件：自动战斗中小于或等于 0

效果：
- 无法进行任何行动（历练/旅行/悬赏）
- 可使用丹药恢复 HP，HP > 0 后状态回到 IDLE
- 或等待自然恢复

### 7.2 HP 自然恢复

```
IDLE 状态：每分钟 +1 HP，直到满血
其他状态：不恢复
```

规则与提升的懒加载机制一致——状态查询/操作时即时计算。
