# ADR-0001: 保留MyBatis-Flex Model继承

## 状态

已接受

## 背景

在架构审查中发现，领域实体（如`User`）继承了`com.mybatisflex.core.activerecord.Model`，导致领域层直接依赖基础设施层。这违反了传统的DDD分层原则，即领域层不应依赖基础设施层。

## 决策

**保留**当前的Model继承模式，不进行重构。

## 理由

1. **技术栈确定性**：项目明确不会更换MyBatis-Flex框架和PostgreSQL数据库
2. **开发效率**：Model继承提供了便捷的Active Record模式，简化了CRUD操作
3. **框架深度集成**：MyBatis-Flex的Model提供了丰富的查询构建器和类型安全功能
4. **团队熟悉度**：当前模式已经形成，团队已适应这种开发方式

## 后果

### 正面
- 保持开发效率和代码简洁性
- 避免大规模重构带来的风险和成本
- 继续享受MyBatis-Flex框架的便利功能

### 负面
- 领域层与基础设施层耦合
- 单元测试需要依赖数据库（或使用内存数据库）
- 未来如果确实需要更换框架，重构成本较高

## 相关决策

- 架构审查应避免重新提出"移除Model继承"的建议
- 单元测试策略应考虑使用Testcontainers或内存数据库
