# 地灵 MBTI 人格化对话系统

## 1. 系统概述

地灵是福地的人格化 AI 精灵，通过 Spring AI 2.x 的 Function Calling 机制实现自然语言驱动的福地管理。**所有福地操作统一使用
Function Calling 模式**，LLM 自主判断是纯聊天还是调用工具。

### 核心流程

```
玩家: "@地灵 在1号地块种灵芝"
  ↓
FudiHandle → FudiCommandHandler → SpiritChatService.chatWithSpirit()
  ├─ 设置用户上下文 (UserContext)
  ├─ 懒恢复地灵精力
  ├─ 构建统一系统 Prompt (福地状态 + 网格详情 + 形态 + MBTI 语气风格 + 好感度)
  ├─ 调用 LLM (含 tools 参数)
  ├─ Spring AI 自动执行 SpiritTools（含精力消耗）
  └─ LLM 根据执行结果生成人格化回复
```

## 3. 地灵形态系统 (SpiritForm)

地灵拥有随机分配的形态（100+ 种），存储在 `xt_spirit_form` 表中。

### 随机分配

创建福地时，系统从 `xt_spirit_form` 表中随机抽取一个形态分配给该福地，不可变更。

### 喜好生成（LLM 驱动）

创建福地时，系统自动执行以下流程生成地灵的个人喜好：

```
1. 从 ItemTemplate 表 DISTINCT 查询所有 tags
2. 将 tags 列表 + 地灵形态描述 + MBTI 人格 发送给 LLM
3. LLM 返回 liked_tags 和 disliked_tags（各 3-5 个）
4. 存入 spiritConfig JSONB
```

地灵喜好基于物品的 `tags` 字段（已在 ItemTemplate 中预定义），而非硬编码物品名称。

## 4. 好感度系统

### 4.1 基础属性

- **上限**: 固定 1000
- **初始值**: 0

### 4.2 增减途径

**增加：**
| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫胜利 | +5 | 每渡过一次天劫 |
| 赠送所爱礼物 | +10~50 | 取决于物品 rarity 与 liked_tags 匹配度 |
| 日常对话 | +1 | 每日上限 5 次，防刷 |

**减少：**
| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫失败 | -clearCount | 被毁地块数 |
| 赠送讨厌礼物 | -5~20 | 匹配 disliked_tags |
| 地灵对话中冒犯 | -1~5 | LLM 通过 reportPlayerOffense 工具上报 |

### 4.3 作用范围

| 作用                     | 公式                                                   |
|------------------------|------------------------------------------------------|
| 精力消耗减免                 | `实际消耗 = baseCost × (1 - min(0.5, affection / 1000))` |
| 灵兽进化成功率加成              | `min(15, affection / 7)`                             |
| 灵兽品质突破加成               | `min(20, affection / 5)`                             |
| LLM 对话语气               | Prompt 注入好感度描述，高好感亲密/低好感冷淡                           |
| 渡劫怜悯（渡劫失败时如满足则判定为渡劫成功） | `affection ≥ 800` 且 `防御 ≥ 攻击 × 80%` 时触发              |

## 5. 精力系统

### 5.1 基础属性

- **上限公式**: `100 + tribulationStage × 50`
- **初始值**: 100
- **恢复**: 随时间自然恢复（懒更新）

### 5.2 恢复机制

- **触发时机**: 任何使用地灵的操作（AOP 切面自动处理）
- **恢复量**: `精力上限 × 5% × 距上次使用小时数`
- **特性**: 按比例恢复，上限越高绝对值恢复越快

### 5.3 消耗规则

通过 `@ConsumeSpiritEnergy` 注解标注在 Service 方法上，AOP 切面自动处理：

| 操作     | 基础消耗   |
|--------|--------|
| LLM 对话 | 5      |
| 种植     | 3      |
| 收获（单次） | 5      |
| 收获（全部） | 5 × 数量 |
| 建造     | 3      |
| 拆除     | 3      |
| 孵化     | 10     |
| 进化/突破  | 8      |
| 收取产出   | 3      |

**好感减免**: `实际消耗 = baseCost × (1 - min(0.5, affection / 1000))`

### 5.4 情绪联动

- `精力 ≤ 0` → 情绪状态变为 `FATIGUED`
- 懒恢复机制确保正常频率下不会轻易见底

## 6. 渡劫怜悯机制

### 6.1 触发条件

```
defense >= attack × 80%    // 差点失败
   && affection >= 800       // 好感度达到上限的80%
```

### 6.2 效果

- 视为天劫胜利（劫数 +1，连胜 +1，灵气上限增长）
- **地灵精力归零**
- **情绪变为 FATIGUED**
- 结果传给 LLM 生成叙事文本（如："小狐妖燃烧灵体，九尾化作光幕挡住了天雷……她虚弱地倒在你怀里"）

## 8. MBTI 语气风格系统

### 8.1 设计原则

地灵保留 MBTI `code`（如 INTJ）和 `toneStyle`（语气风格）。语气风格是详细的角色卡，包含人格特质、说话习惯、常用句式、颜文字/Emoji，直接注入
Prompt 强约束 LLM 语气。

### 8.2 语气风格示例

**INTJ** — 冷静理性的战略型人格，简洁干练，习惯用逻辑和数据支撑观点。常用「效率上」「最优解是…」开头。(￣▽￣)~*

**ESFP** — 戏精附体的表演家型人格，随时随地嗨起来！常用「哇哈哈哈哈！」「我可爱的观众～～～」炒热气氛。🥹🎭 (ﾉ>ω<)ﾉ★

**INFJ** — 温柔深邃的洞察型人格，轻声细语带着暖意。常用「我能理解你的想法」「慢慢来，不着急」安抚和鼓励。🌙 (っ´ω`c)♡

## 9. Prompt 结构

统一 Prompt 不列出工具签名（工具定义由 Spring AI `.tools()` 机制提供），包含：

```
你是{SPIRIT_FORM}({CODE}性格)的地灵。
你的语气风格：{TONE_STYLE}
你当前的情绪状态：{EMOTION}

【福地状态】
- 灵气：{aura}/{max}
- 劫数：{stage}
- 精力：{energy}/{max}
- 好感度：{affection}/1000

【地块状态】
{grid_detail}

【地灵喜好】
你喜欢的物品 tag：{liked_tags}
你讨厌的物品 tag：{disliked_tags}

【规则】
1. 自主判断是否需要操作福地
2. 纯聊天时不操作，直接人格化回复
3. 操作完成后根据执行结果生成人格化回复
4. 严格保持语气风格，根据当前情绪和好感度调整语气
5. 精力不足时主动提醒玩家休息
6. 玩家赠送礼物时触发 giveGift 工具
7. 玩家说了冒犯/讨厌的话时调用 reportPlayerOffense
```

## 10. AOP 精力管理

### 10.1 注解

```java

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ConsumeSpiritEnergy {
    int value() default 5;
}
```

### 10.2 切面逻辑 (SpiritEnergyAspect)

```
环绕 @ConsumeSpiritEnergy 标注的方法：
  1. 从方法参数中提取 userId
  2. 查询当前用户 Fudi
  3. 执行懒恢复：
     - 计算 (当前时间 - lastEnergyUpdate) 的小时数
     - 恢复量 = min(上限, 当前精力 + 上限 × 5% × 小时数)
  4. 计算实际消耗：
     - 减免 = min(0.5, affection / 1000)
     - actualCost = baseCost × (1 - 减免)
  5. if 当前精力 < actualCost → 允许执行，但消耗至 0（触发 FATIGUED）
  6. 扣除精力，更新 lastEnergyUpdate，保存 Fudi
  7. proceed() 执行原方法
```
