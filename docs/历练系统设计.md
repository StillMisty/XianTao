# 历练系统设计

## 1. 概述

历练系统是玩家挂机获取修为和物品的核心玩法。玩家在历练区地图中自动遇怪战斗，结算时获得修为、物品和铜币奖励。

**核心流程：**
1. 玩家在历练区地图使用 `历练` 命令开始历练
2. 系统记录开始时间，玩家状态变为 `EXERCISING`
3. 玩家使用 `历练结算` 命令结束历练
4. 系统计算历练时长，模拟战斗，结算奖励

**设计原则：**
- 历练只能在用户当前所在地图进行，不支持指定其他地图
- 旅行系统提供移动成本，历练系统提供挂机收益，两者形成完整的游戏循环

## 3. 核心组件

### 3.1 服务层

**TrainingService** — 历练核心服务 + 统一事件循环
- `startTraining(PlatformType, String openId)` - 开始历练
- `endTraining(PlatformType, String openId)` - 结束历练（计算时长→统一事件循环→结算）
- `runUnifiedEventLoop(...)` — 一个 for 循环：读 `xt_activity_event` 统一池 → COMBAT 走 `CombatEventHandler`，NUMERIC 走 `SubEventEffectExecutor`

**CombatEventHandler** — 单次遇怪处理
- 从 ActivityEvent.params 读取 monster_template_id/min_count/max_count
- 委托 CombatEngine 执行回合战斗
- 处理战后：HP回写、掉落、顿悟（可能悟得技能）

**SubEventEffectExecutor** — NUMERIC 事件效果引擎
- 读 `params.effects` JSONB 数组 → 分发到 12 种 effect handler（策略模式）

**EncounterCalculator** — 动态间隔计算
- `compute(userId, user, mapNode, durationMinutes)` → slots + perRollChance
- 公式：`interval = (12 - richness) × levelMismatch / mapDanger`

**TrainingCompleter** — 叙事产出 + 隐藏事件
- `produceCompletionEvent()` / `produceInterruptedEvent()` / `checkHiddenEvents()`

详见 [异步事件系统.md](./异步事件系统.md)。

**DropProcessor** - 掉落处理器
- `processMonsterDrops(MonsterTemplate tmpl)` - 从怪物模板计算掉落
- `distributeDrops(Long userId, List<DropItem> drops)` - 掉落分发到背包

战斗引擎详见 [战斗系统.md](./战斗系统.md)
遇怪公式详见 [地图旅行历练.md](./地图旅行历练.md#63-动态遇怪事件间隔)

## 6. 输出格式

### 6.1 普通战斗统计

```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
修为+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85
```

### 6.2 高光战斗（LLM渲染）

```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
修为+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85

━━━ 高光时刻 ━━━
第5场战斗，你遭遇了一只剧毒蟾蜍王。它喷出的毒雾几乎遮蔽了你的视线，
但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
一剑刺穿了它的要害。这场战斗持续了5个回合，最终你以微弱优势胜出。
```

### 6.3 高光战斗触发条件

- 战斗回合数超过10回合
- 双方血量都低于30%（势均力敌）
- 触发了稀有技能或连击（>=3次）
