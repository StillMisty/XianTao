# 历练系统设计

## 1. 概述

历练系统是玩家挂机获取经验和物品的核心玩法。玩家在历练区地图中自动遇怪战斗，结算时获得经验、物品和铜币奖励。

**核心流程：**
1. 玩家在历练区地图使用 `历练` 命令开始历练
2. 系统记录开始时间，玩家状态变为 `EXERCISING`
3. 玩家使用 `历练结算` 命令结束历练
4. 系统计算历练时长，模拟战斗，结算奖励

**设计原则：**
- 历练只能在用户当前所在地图进行，不支持指定其他地图
- 旅行系统提供移动成本，历练系统提供挂机收益，两者形成完整的游戏循环

## 3. 核心组件

### 3.1 服务层

**TrainingService** - 历练核心服务 + 遇敌编排
- `startTraining(PlatformType, String openId)` - 公开API，开始历练
- `endTraining(PlatformType, String openId)` - 公开API，结束历练
- `simulateTraining(Long userId, int durationMinutes)` - 遇敌编排（遇怪循环 + 战斗模拟 + 掉落 + 经验）

**TrainingCombatLogic** - 战斗模拟编排（从 TrainingService 抽取）
- `executeCombatSequence(...)` - 执行一轮遇敌：遇怪选择 → 队伍构建 → 委托 CombatService → 掉落 → HP回写

**DropProcessor** - 掉落处理器
- `processMonsterDrops(MonsterTemplate tmpl)` - 从怪物模板计算掉落
- `distributeDrops(Long userId, List<DropItem> drops)` - 掉落分发到背包

战斗引擎详见 [战斗系统.md](./战斗系统.md)
遇怪计算详见 [地图旅行历练.md](./地图旅行历练.md#6-历练系统)

## 4. 数据结构

历练奖励包含：经验值、物品掉落、灵兽经验、战斗统计（遇敌场次、击杀数、战败数、法决触发统计）。

## 6. 输出格式

### 6.1 普通战斗统计

```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85
```

### 6.2 高光战斗（LLM渲染）

```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85

━━━ 高光时刻 ━━━
第5场战斗，你遭遇了一只剧毒蟾蜍王。它喷出的毒雾几乎遮蔽了你的视线，
但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
一剑刺穿了它的要害。这场战斗持续了5个回合，最终你以微弱优势胜出。
```

### 6.3 高光战斗触发条件

- 战斗回合数超过10回合
- 双方血量都低于30%（势均力敌）
- 触发了稀有技能或连击（>=3次）
