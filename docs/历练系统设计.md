# 历练系统设计

## 1. 概述

历练系统是玩家挂机获取经验和物品的核心玩法。玩家在历练区地图中自动遇怪战斗，结算时获得经验、物品和铜币奖励。

**核心流程：**
1. 玩家在历练区地图使用 `历练` 命令开始历练
2. 系统记录开始时间，玩家状态变为 `EXERCISING`
3. 玩家使用 `历练结算` 命令结束历练
4. 系统计算历练时长，模拟战斗，结算奖励

## 3. 改进方案

### 3.1 动态遇怪间隔

**当前实现：**
```java
int encounterChances = durationMinutes / 10;  // 每10分钟一次遇怪机会
if (ThreadLocalRandom.current().nextDouble() >= BASE_ENCOUNTER_CHANCE_PER_10MIN) continue;  // 40%概率
```

**优化方案：** 组合因素动态计算

```
encounterInterval = baseInterval × (1 - difficultyFactor) × (1 - levelFactor) × (1 - gearFactor)
```

**因子计算：**
- `baseInterval` = 10分钟（基础间隔）
- `difficultyFactor` = `mapLevel / 100`（地图难度因子，最高0.5）
- `levelFactor` = `playerLevel / 200`（玩家等级因子，最高0.3）
- `gearFactor` = `gearScore / 1000`（装备评分因子，最高0.2）

**示例：**
- 玩家等级50，地图等级30，装备评分200
- `difficultyFactor` = 30/100 = 0.3
- `levelFactor` = 50/200 = 0.25
- `gearFactor` = 200/1000 = 0.2
- `encounterInterval` = 10 × (1-0.3) × (1-0.25) × (1-0.2) = 10 × 0.7 × 0.75 × 0.8 = 4.2分钟

**实现位置：** `CombatService.simulateTraining()`

### 3.2 等级衰减机制

**当前实现：**
```java
long expGained = (long) (BASE_EXP_PER_MINUTE * minutesTraining * efficiencyMultiplier);
```

**优化方案：** 引入等级衰减

```
expMultiplier = max(0.1, 1 - max(0, (playerLevel - mapLevel - 10)) * 0.05)
```

**规则：**
- 玩家等级 ≤ 地图等级 + 10：无衰减（100%经验）
- 玩家等级 > 地图等级 + 10：每高1级衰减5%
- 最低衰减到10%（保留保底经验）

**示例：**
- 玩家等级50，地图等级30
- 等级差 = 50 - 30 - 10 = 10
- 衰减 = 10 × 0.05 = 0.5
- `expMultiplier` = max(0.1, 1 - 0.5) = 0.5（50%经验）

**实现位置：** `TrainingService.endTraining()`

### 3.4 地图限制说明

**设计原则：** 历练只能在用户当前所在地图进行，不支持指定其他地图。

**原因：**
- 旅行系统的核心价值在于移动成本（时间消耗、事件风险）
- 如果允许直接指定地图历练，旅行系统将失去意义
- 玩家需要先通过旅行到达目标地图，才能在该地图历练

**当前实现：** 只能在当前所在地图历练（符合设计）

**约束条件：**
- 当前地图必须是 `TRAINING_ZONE` 类型
- 玩家等级必须满足当前地图的等级要求
- 玩家状态必须为 `IDLE`

## 4. 战斗系统改进

### 4.1 技能效果系统

**当前状态：** 7种效果类型定义，只有2种实现

**待实现效果：**

| 效果 | 说明 | 触发条件 |
|------|------|----------|
| `ARMOR_BREAK` | 破甲（降低防御） | 技能命中时 |
| `SLOW` | 减速（降低速度） | 技能命中时 |
| `DOT` | 持续伤害 | 技能命中时 |
| `STUN` | 眩晕（跳过行动） | 技能命中时（概率） |
| `FREEZE` | 冰冻（跳过行动+受伤增加） | 技能命中时（概率） |
| `SILENCE` | 沉默（禁止技能） | 技能命中时（概率） |
| `HEAL` | 治疗 | 主动技能 |
| `BUFF` | 增益（提升属性） | 主动技能 |

**Buff/Debuff 系统设计：**
```
Buff/Debuff 系统
├── BuffType → 效果类型
├── value → 效果数值
├── duration → 持续回合数
├── source → 来源（技能/装备）
└── stackable → 是否可叠加
```

### 4.2 灵兽AI行为系统

**当前状态：** 灵兽无配合玩家策略，技能未启用

**设计方案：**

**AI行为接口：**
```
BeastAI（接口）
├── decideAction(beast, battleContext) → BeastAction
├── selectTarget(beast, enemies, allies) → Combatant
└── getPriority() → int（行动优先级）
```

**行为策略：**
| 策略 | 说明 | 触发条件 |
|------|------|----------|
| `ATTACK` | 攻击敌人 | 默认行为 |
| `DEFEND` | 防御（减少受到的伤害） | 玩家血量低时 |
| `SUPPORT` | 辅助（治疗/增益） | 有治疗/增益技能时 |
| `SKILL` | 使用技能 | 技能CD就绪时 |

**配合玩家策略：**
1. 优先攻击对玩家威胁最大的敌人（攻击力最高的怪物）
2. 如果玩家血量低于30%，优先攻击攻击玩家的怪物
3. 如果有怪物即将被击杀（血量低于20%），优先集火

**行动决策：**
1. 如果有治疗技能且玩家血量低于50%，使用治疗
2. 如果有增益技能且未激活，使用增益
3. 如果有控制技能且目标可控制，使用控制
4. 否则，使用普通攻击

**启用灵兽技能：**
```java
// 当前实现（返回空列表）
public List<Skill> getSkills() {
    return List.of();
}

// 改进实现
public List<Skill> getSkills() {
    if (beast.getSkills() == null || beast.getSkills().isEmpty()) {
        return List.of();
    }
    return skillRepository.findByIds(beast.getSkills());
}
```
