# 地图系统设计文档

## 1. 模块概述

地图系统是仙道游戏的世界架构基础，负责管理地理位置、区域连接、特产掉落和旅行事件等核心功能。

### 1.1 核心职责
- 地图节点管理（名称、描述、等级要求）
- 地图连通性（相邻关系、旅行耗时）
- 特产掉落池（加权随机掉落）
- 旅行事件系统（D20判定、事件触发）
- 地图访问权限控制（等级限制）

### 1.2 技术栈
- **ORM**：MyBatis-Flex
- **数据库**：PostgreSQL 18 (JSONB存储连接关系和掉落池)
- **算法**：加权随机算法、图论连通性检测

#### 2.1.2 JSONB字段结构

**neighbors（相邻地图）**：
```json
{
  "黑金主城": 5,
  "枯骨林": 10,
  "迷雾沼泽": 15
}
```
- Key：相邻地图名称
- Value：旅行耗时（分钟）

**specialties（特产掉落池）**：
```json
[
  {"name": "毒龙草", "templateId": 101, "weight": 30},
  {"name": "铁矿石", "templateId": 202, "weight": 50},
  {"name": "灵泉", "templateId": 303, "weight": 20}
]
```
- name：物品名称
- templateId：物品模板ID
- weight：掉落权重（数值越大越容易掉落）

**travel_events（旅行事件权重）**：
```json
[
  {"eventType": "ambush", "weight": 40},
  {"eventType": "find_treasure", "weight": 10},
  {"eventType": "weather", "weight": 20}
]
```
- eventType：事件类型（ambush/find_treasure/weather/safe_passage）
- weight：触发权重

## 3. 核心业务规则

### 3.1 地图连通性

#### 3.1.1 有向图结构
地图系统本质上是一个**有向加权图**：
- **节点**：地图（MapNode）
- **边**：相邻关系（带权重=旅行耗时）
- **方向性**：A→B的旅行耗时可能与B→A不同

#### 3.1.2 相邻判定
```java
boolean isAdjacentTo(String targetMapName) {
    return neighbors.containsKey(targetMapName);
}
```

**示例**：
- 黑金主城的neighbors包含"枯骨林": 10
- 则从黑金主城可以前往枯骨林，耗时10分钟
- 若枯骨林的neighbors不包含"黑金主城"，则无法反向通行

#### 3.1.3 获取相邻地图列表
```java
List<String> getAdjacentMapNames() {
    return List.copyOf(neighbors.keySet());
}
```

### 3.2 地图访问权限

#### 3.2.1 等级限制检查
```java
boolean isAccessibleBy(int playerLevel) {
    if (levelRequirement == null) return true;
    return playerLevel >= levelRequirement;
}
```

**设计意图**：
- 防止低等级玩家误入高等级危险区域
- 引导玩家按等级梯度探索世界

**示例**：
- 枯骨林等级要求：10级
- 玩家等级：8级 → 无法进入
- 玩家等级：10级 → 可以进入

### 3.3 旅行耗时查询

#### 3.3.1 获取到指定地图的旅行时间
```java
Integer getTravelTimeTo(String mapName) {
    if (neighbors == null) return null;
    return neighbors.get(mapName);
}
```

**返回值**：
- 存在相邻关系：返回旅行耗时（分钟）
- 不存在相邻关系：返回null

## 4. 特产掉落系统

### 4.1 加权随机算法

#### 4.1.1 算法原理
根据物品的weight属性进行加权随机抽取，权重越高掉落概率越大。

**计算步骤**：
1. 计算总权重：`totalWeight = sum(all weights)`
2. 生成随机数：`random = [0, totalWeight)`
3. 遍历物品列表，累加权重
4. 当`random < currentWeight`时，选中该物品

#### 4.1.2 实现示例
```java
Map<String, Object> getRandomSpecialty() {
    if (specialties == null || specialties.isEmpty()) return null;
    
    // 计算总权重
    int totalWeight = specialties.stream()
        .mapToInt(item -> ((Number) item.get("weight")).intValue())
        .sum();
    
    if (totalWeight == 0) return specialties.get(0);
    
    // 生成随机数
    int random = (int) (Math.random() * totalWeight);
    int currentWeight = 0;
    
    // 遍历查找
    for (Map<String, Object> specialty : specialties) {
        int weight = ((Number) specialty.get("weight")).intValue();
        currentWeight += weight;
        if (random < currentWeight) {
            return specialty;
        }
    }
    
    return specialties.get(specialties.size() - 1);
}
```

#### 4.1.3 掉落概率计算
假设掉落池配置：
```json
[
  {"name": "毒龙草", "weight": 30},
  {"name": "铁矿石", "weight": 50},
  {"name": "灵泉", "weight": 20}
]
```

总权重 = 30 + 50 + 20 = 100

**掉落概率**：
- 毒龙草：30/100 = 30%
- 铁矿石：50/100 = 50%
- 灵泉：20/100 = 20%

### 4.2 历练掉落应用

历练服务调用地图的getRandomSpecialty()方法获取掉落物品：

```java
// TrainingService.calculateItemsReward()
for (int i = 0; i < dropChances; i++) {
    Map<String, Object> specialty = mapNode.getRandomSpecialty();
    if (specialty != null) {
        int quantity = (int) (Math.random() * 3) + 1; // 1-3个
        // 添加到奖励列表
    }
}
```

**掉落次数计算**：
```
dropChances = (历练分钟数 / 10) × 效率倍率
效率倍率 = 1.0 + 敏捷 × 0.01
```

**示例**：
- 历练60分钟，敏捷20点
- 效率倍率 = 1.0 + 20×0.01 = 1.2
- 掉落次数 = (60/10) × 1.2 = 7.2 → 7次

## 5. 旅行事件系统

### 5.1 D20骰点判定

#### 5.1.1 事件触发规则
到达目的地时触发D20（20面骰）判定：

**判定逻辑**：
- D20 ≤ 10：触发随机事件（50%概率）
- D20 > 10：安全通行（50%概率）

```java
int d20Roll = (int) (Math.random() * 20) + 1; // 1-20

if (d20Roll <= 10 && mapNode.getTravelEvents() != null) {
    // 触发随机事件
    TravelEventType eventType = calculateTravelEvent(mapNode, d20Roll);
} else {
    // 安全通行
    return TravelEventType.SAFE_PASSAGE;
}
```

#### 5.1.2 事件类型选择
当事件触发时，根据地图的travel_events配置进行加权随机选择。

**示例配置**：
```json
[
  {"eventType": "ambush", "weight": 40},
  {"eventType": "find_treasure", "weight": 10},
  {"eventType": "weather", "weight": 20}
]
```

总权重 = 40 + 10 + 20 = 70

**事件概率**：
- 伏击(ambush)：40/70 ≈ 57.1%
- 发现宝藏(find_treasure)：10/70 ≈ 14.3%
- 恶劣天气(weather)：20/70 ≈ 28.6%

### 5.2 事件处理逻辑

#### 5.2.1 伏击 (Ambush)
**效果**：玩家受到10-30点伤害

```java
int damage = 10 + (int) (Math.random() * 20);
user.takeDamage(damage);
return String.format("途中遭遇敌人袭击！受到了 %d 点伤害，剩余 HP: %d", 
                     damage, user.getHpCurrent());
```

#### 5.2.2 发现宝藏 (Find Treasure)
**效果**：获得50-150铜币

```java
long coins = 50 + (long) (Math.random() * 100);
user.setCoins(user.getCoins() + coins);
return String.format("运气真好！您在路边发现了一个宝箱，获得了 %d 铜币", coins);
```

#### 5.2.3 恶劣天气 (Weather)
**效果**：额外消耗体力或降低收益（当前为提示性事件）

```java
return "遭遇毒雾天气，视线受阻，小心翼翼地通过了";
```

#### 5.2.4 安全通行 (Safe Passage)
**效果**：无事件发生

```java
return "一路平安，没有任何意外发生";
```

## 6. 服务层设计

### 6.1 MapService

**职责**：地图查询、连通性验证

**核心方法**：
- `getMapById(mapId)`：根据ID查询地图
- `getMapByName(mapName)`：根据名称查询地图
- `getAdjacentMaps(currentMapId)`：获取当前地图的所有相邻地图
- `checkAccessibility(mapId, playerLevel)`：检查玩家是否可访问某地图
- `getTravelTime(fromMapId, toMapId)`：获取两地图间的旅行耗时

### 6.2 TravelService

**职责**：旅行流程管理、事件触发

**核心方法**：

#### 6.2.1 旅行模式
- `startTravel(userId, targetMapId, useStamina)`：开始旅行
  - useStamina=true：体力模式（立即完成，消耗体力）
  - useStamina=false：真实时间模式（记录开始时间，等待到达）

- `completeTravel(userId)`：完成旅行（真实时间模式）
  - 检查是否到达时间
  - 触发D20事件判定
  - 更新玩家位置和状态

- `useStaminaToSkip(userId)`：使用体力跳过等待
  - 立即完成当前旅行
  - 触发事件判定

#### 6.2.2 事件处理
- `rollD20()`：D20骰点
- `calculateTravelEvent(mapNode, d20Roll)`：计算旅行事件类型
- `processTravelEvent(user, eventType)`：处理事件效果

## 7. 旅行系统设计

### 7.1 双模式旅行

#### 7.1.1 真实时间模式
**流程**：
1. 检查玩家状态（必须为IDLE）
2. 验证地图连通性和等级要求
3. 设置玩家状态为RUNNING
4. 记录旅行开始时间和目的地
5. 返回预计到达时间

**到达后**：
1. 玩家手动触发完成旅行
2. 检查是否已到达（当前时间 ≥ 开始时间 + 旅行耗时）
3. 触发D20事件判定
4. 更新玩家位置为目的地
5. 恢复玩家状态为IDLE

**优势**：
- 增强沉浸感，模拟真实旅行过程
- 鼓励玩家长期在线规划行程

#### 7.1.2 体力跳过模式
**流程**：
1. 检查玩家状态（必须为IDLE）
2. 验证地图连通性和等级要求
3. 计算体力消耗：`旅行耗时(分钟) × 5`
4. 检查体力是否足够
5. 扣除体力
6. 立即触发D20事件判定
7. 更新玩家位置为目的地
8. 保持玩家状态为IDLE

**优势**：
- 快速移动，适合紧急场景
- 消耗资源（体力）换取时间

### 7.2 旅行约束

#### 7.2.1 前置条件
- 玩家状态必须为IDLE（空闲）
- 目标地图必须与当前地图相邻
- 玩家等级必须满足目标地图要求

#### 7.2.2 状态限制
- 旅行中（RUNNING状态）无法执行其他操作（历练、探索等）
- 必须先完成旅行才能进行其他活动

## 8. 地图类型设计

### 8.1 地图分类 (MapType)

| 类型 | 说明 | 特点 |
|------|------|------|
| CITY | 主城 | 安全区域，无旅行事件，NPC聚集 |
| WILDERNESS | 野外 | 普通区域，中等风险，常见材料 |
| DUNGEON | 秘境 | 高风险高回报，稀有材料，高等级要求 |
| SPECIAL | 特殊 | 剧情地图、活动地图等 |

### 8.2 地图设计原则

#### 8.2.1 等级梯度
- 新手村：0级
- 初级野外：5-10级
- 中级区域：15-30级
- 高级秘境：40+级

#### 8.2.2 连通性设计
- 每个地图至少有1个相邻地图（避免孤立）
- 主城作为交通枢纽，连接多个区域
- 高等级地图通过多条路径可达（增加策略性）

#### 8.2.3 特产差异化
- 不同地图拥有独特的特产掉落池
- 鼓励玩家探索多个区域收集资源
- 高等级地图产出稀有材料

## 9. 性能优化

### 9.1 缓存策略

#### 9.1.1 地图配置缓存
- 地图节点信息变化频率低，适合缓存
- 使用Spring Cache或Caffeine缓存常用地图
- 缓存Key：mapId或mapName

#### 9.1.2 特产掉落池缓存
- specialties字段解析为Java对象后缓存
- 避免每次掉落都解析JSONB

### 9.2 查询优化

#### 9.2.1 批量查询相邻地图
```java
// 避免N+1查询
List<MapNode> adjacentMaps = mapNodeRepository.findByIdIn(neighborIds);
```

#### 9.2.2 索引优化
- 对name字段创建索引（按名称查询）
- 对level_requirement创建索引（等级筛选）

## 10. 扩展性设计

### 10.1 新地图添加

**步骤**：
1. 创建Flyway迁移脚本（V0.0.X__add_new_map.sql）
2. 插入xt_map_node记录
3. 配置neighbors、specialties、travel_events
4. 更新相关地图的neighbors字段（建立双向连接）

**示例SQL**：
```sql
INSERT INTO xt_map_node (name, description, map_type, level_requirement, 
                         neighbors, specialties, travel_events)
VALUES (
  '火焰山',
  '炽热的火山地带，常年喷发岩浆',
  'DUNGEON',
  30,
  '{"黑金主城": 20, "枯骨林": 25}'::jsonb,
  '[{"name": "火莲", "templateId": 103, "weight": 40}, 
    {"name": "熔岩晶", "templateId": 404, "weight": 60}]'::jsonb,
  '[{"eventType": "ambush", "weight": 50}, 
    {"eventType": "weather", "weight": 30}]'::jsonb
);

-- 更新黑金主城的neighbors，添加火焰山
UPDATE xt_map_node 
SET neighbors = neighbors || '{"火焰山": 20}'::jsonb
WHERE name = '黑金主城';
```

### 10.2 新事件类型

**步骤**：
1. 在TravelEventType枚举中添加新类型
2. 在processTravelEvent()中添加处理逻辑
3. 在地图的travel_events配置中使用新类型

**示例**：
```java
public enum TravelEventType {
    AMBUSH,
    FIND_TREASURE,
    WEATHER,
    SAFE_PASSAGE,
    MEET_NPC  // 新增：偶遇NPC
}

private String processTravelEvent(User user, TravelEventType eventType) {
    return switch (eventType) {
        case MEET_NPC -> {
            // 偶遇NPC逻辑
            yield "您遇到了一位云游商人，他向您展示了稀有的商品";
        }
        // ... 其他事件
    };
}
```
