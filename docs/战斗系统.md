# 战斗系统

## 1. 设计原则

- **懒结算**：所有战斗均为后台模拟，无实时交互、无后台任务
- **通用引擎**：一套引擎覆盖所有战斗场景（历练 / 秘境 / PVP）
- **Team vs Team**：不关心具体类型，只操作 `Combatant` 接口

---

## 2. 核心抽象

```
Combatant（接口）
├── getId()            → 唯一标识
├── getSpeed()         → 行动速度（决定出手顺序）
├── getAttack()        → 攻击力（已包含所有加成）
├── getDefense()       → 防御力
├── getHp() / getMaxHp() → 生命值
├── takeDamage(amount) → 受到伤害
├── isAlive()          → 是否存活
├── normalAttack(target) → 普攻
├── getSkills()        → 已装载法决列表
│
├── PlayerCombatant    → 玩家 + 装备 + 法决槽位
│   ├── getWeaponType()       → 法器类型（克制判定用）
│   ├── getMonsterTypeAdvantage() → 克制倍率
│   └── 攻防来源于 User 属性 + Equipment 加成
│
├── BeastCombatant     → 灵兽
│   ├── 攻防由品质 + 等级自动推算
│   └── skills 来自 xt_beast.skills
│
└── MonsterCombatant   → 怪物
    ├── 由 MonsterTemplate + 等级缩放生成
    └── skills 来自模板配置
```

```
Team（战斗团队）
├── ownerId            → 所属者（用于掉落归属）
├── members[]          → Combatant 列表
├── aliveMembers()     → 存活成员
├── isAllDead()        → 是否全灭
└── aliveCount()       → 存活数量
```

---

## 3. 通用战斗流程

```
simulate(teamA, teamB, maxRounds=20):
  repeat maxRounds:
    ① 按 speed 降序排列所有存活 combatant → 行动序列
    ② for each combatant in 行动序列:
         if not combatant.isAlive() → 跳过
         a. 普攻（按 targetStrategy 选目标）
         b. 法决自触发（CD 轮播）
    ③ 清理死亡单位
    ④ if teamB.isAllDead() → winner=teamA
    ⑤ if teamA.isAllDead() → winner=teamB
  → draw（超时平局）
```

### 返回 (CombatResult)

```
CombatResult
├── winner              → teamA / teamB / DRAW
├── rounds              → 实际回合数
├── hpChangesA[]        → teamA 各成员 HP 变化
├── hpChangesB[]        → teamB 各成员 HP 变化
├── damageDealt[]       → 各 combatant 造成的总伤害
├── skillProcs[]        → 法决触发记录 [{combatantId, skillName, count}]
└── combatLog[]         → 每次攻击的记录，按序排列
```

### 战斗日志条目 (CombatLogEntry)

```
CombatLogEntry
├── round               → 回合数 (1-based)
├── sequence            → 本回合内出手序号
├── attackerId          → 攻击方 ID
├── attackerName        → 攻击方名称
├── attackerType        → PLAYER / BEAST / MONSTER
├── defenderId          → 防御方 ID
├── defenderName        → 防御方名称
├── defenderType        → PLAYER / BEAST / MONSTER
├── attackType          → NORMAL / SKILL
├── skillName           → 法决名称（attackType=SKILL 时）
├── damageDealt         → 造成伤害值
├── isCrit              → 是否暴击（预留）
├── defenderHpBefore    → 防御方攻击前 HP
├── defenderHpAfter     → 防御方攻击后 HP
├── isKill              → 本次攻击是否击杀
└── advantageMultiplier → 克制倍率（仅玩家攻击时有效）
```

## 4. 目标选择策略 (TargetStrategy)

| 策略 | 适用场景 | 规则 |
|------|---------|------|
| **PVE_FOCUS_FIRE** | 历练 / 秘境 | teamA → 集火 teamB 中 HP 比例最低者；teamB → 随机选择 |
| **PVP_BALANCED** | PVP（预留） | 按 speed 比例分配攻击目标 |
| **PVP_TARGETED** | PVP 组队（预留） | 玩家可手动指定目标 |

## 5. 伤害公式

```
// 玩家伤害
playerAttack = totalStr × 2 + equipmentAttack
playerDamage = playerAttack × weaponTypeAdvantage(weaponType, monsterType)

// 灵兽伤害
beastDamage = beastAttack（不受法器克制影响）

// 怪物伤害
monsterDamage = monsterAttack × reverseAdvantage(weaponType, monsterType)

// 统一减免
最终伤害 = max(1, attackPower - targetDefense × 0.3)
```

## 6. 克制矩阵

| 法器克制怪物 | 玩家伤害 | 怪物伤害 |
|------------|---------|----------|
| 克制 | ×1.5 | ×0.8 |
| 被克制 | ×0.8 | ×1.5 |
| 互无关系 | ×1.0 | ×1.0 |

见 [物品装备.md §3](./物品装备.md#3-法器类型-weapontype-与克制矩阵) 完整对照表。

## 7. 法决集成

每场战斗中，每个 Combatant 的已装载法决按独立 CD 自动触发。

触发规则详见 [法决.md §7](./法决.md#7-战斗自动施放)。

**适用 Combatant**：仅 `PlayerCombatant` 和 `BeastCombatant`。
`MonsterCombatant` 的法决由怪物模板的 `skills` 字段定义，固定 CD 轮播。

## 8. 出击 / 参战规则

### 8.1 玩家

- 自动参战，不可离队
- 阵亡 → 全队判负

### 8.2 灵兽

来源 [福地.md](./福地.md)

- 在福地中选择"出战"灵兽（`xt_beast.is_deployed = true`）
- 仅 HP > 0 且未在休养中的灵兽可出战
- 出战上限：3 只（按玩家等级逐步解锁更多）
- 战斗获取经验，升级后属性自动成长

阵亡：

| 品质 | 休养时长 |
|------|---------|
| 凡品 | 30min |
| 灵品 | 1h |
| 仙品 | 2h |
| 圣品 | 4h |
| 神品 | 8h |

休养期间不可出战。

### 8.3 怪物

- 由 `MonsterTemplate` → 等级缩放生成
- 战斗中死亡即销毁
- 掉落归属由击杀者的 `Team.ownerId` 决定

## 9. 技能效果系统

### 9.1 效果类型枚举 (EffectType)

| 效果 | 说明 | 实现状态 |
|------|------|----------|
| `DAMAGE` | 基础伤害 | ✅ 已实现 |
| `MULTI_HIT` | 连击（伤害×3） | ✅ 已实现 |
| `ARMOR_BREAK` | 破甲（降低防御） | ⏳ 待实现 |
| `SLOW` | 减速（降低速度） | ⏳ 待实现 |
| `DOT` | 持续伤害 | ⏳ 待实现 |
| `EXECUTE` | 斩杀（低血量额外伤害） | ⏳ 待实现 |
| `LIFESTEAL` | 吸血 | ⏳ 待实现 |
| `STUN` | 眩晕（跳过行动） | ⏳ 待实现 |
| `FREEZE` | 冰冻（跳过行动+受伤增加） | ⏳ 待实现 |
| `SILENCE` | 沉默（禁止技能） | ⏳ 待实现 |
| `HEAL` | 治疗 | ⏳ 待实现 |
| `BUFF` | 增益（提升属性） | ⏳ 待实现 |

### 9.2 效果实现设计

```
Buff/Debuff 系统
├── BuffType → 效果类型
├── value → 效果数值
├── duration → 持续回合数
├── source → 来源（技能/装备）
└── stackable → 是否可叠加
```

**破甲效果 (ARMOR_BREAK)**
- 触发：技能命中时
- 效果：降低目标防御力 20-30%
- 持续：3-5回合
- 叠加：最多3层

**减速效果 (SLOW)**
- 触发：技能命中时
- 效果：降低目标速度 30-50%
- 持续：2-3回合
- 叠加：刷新持续时间

**持续伤害 (DOT)**
- 触发：技能命中时
- 效果：每回合造成固定伤害
- 持续：3-5回合
- 叠加：刷新持续时间

**控制效果 (STUN/FREEZE/SILENCE)**
- 触发：技能命中时（概率触发）
- 效果：跳过目标下一回合行动
- 持续：1-2回合
- 叠加：刷新持续时间

**治疗效果 (HEAL)**
- 触发：主动技能
- 效果：恢复目标生命值
- 目标：自身或队友
- 计算：`healAmount = wisdom * multiplier + base`

**增益效果 (BUFF)**
- 触发：主动技能
- 效果：提升目标属性（攻击/防御/速度）
- 持续：3-5回合
- 叠加：刷新持续时间

## 10. 灵兽AI行为系统

### 10.1 AI行为接口

```
BeastAI（接口）
├── decideAction(beast, battleContext) → BeastAction
├── selectTarget(beast, enemies, allies) → Combatant
└── getPriority() → int（行动优先级）
```

### 10.2 行为策略枚举 (BeastAction)

| 策略 | 说明 | 触发条件 |
|------|------|----------|
| `ATTACK` | 攻击敌人 | 默认行为 |
| `DEFEND` | 防御（减少受到的伤害） | 玩家血量低时 |
| `SUPPORT` | 辅助（治疗/增益） | 有治疗/增益技能时 |
| `SKILL` | 使用技能 | 技能CD就绪时 |

### 10.3 配合玩家策略

**目标选择逻辑：**
1. 优先攻击对玩家威胁最大的敌人（攻击力最高的怪物）
2. 如果玩家血量低于30%，优先攻击攻击玩家的怪物
3. 如果有怪物即将被击杀（血量低于20%），优先集火

**行动决策逻辑：**
1. 如果有治疗技能且玩家血量低于50%，使用治疗
2. 如果有增益技能且未激活，使用增益
3. 如果有控制技能且目标可控制，使用控制
4. 否则，使用普通攻击

### 10.4 灵兽技能启用

修改 `BeastCombatant.getSkills()` 返回实际技能列表：

```java
// 当前实现（返回空列表）
public List<Skill> getSkills() {
    return List.of();
}

// 改进实现
public List<Skill> getSkills() {
    if (beast.getSkills() == null || beast.getSkills().isEmpty()) {
        return List.of();
    }
    return skillRepository.findByIds(beast.getSkills());
}
```
