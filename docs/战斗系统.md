# 战斗系统

## 1. 设计原则

- **懒结算**：所有战斗均为后台模拟，无实时交互、无后台任务
- **通用引擎**：一套引擎覆盖所有战斗场景（历练 / 秘境 / PVP）
- **Team vs Team**：不关心具体类型，只操作 `Combatant` 接口

---

## 2. 核心抽象

```
Combatant（接口）
├── getId()            → 唯一标识
├── getSpeed()         → 行动速度（决定出手顺序）
├── getAttack()        → 攻击力（已包含所有加成）
├── getDefense()       → 防御力
├── getHp() / getMaxHp() → 生命值
├── takeDamage(amount) → 受到伤害
├── isAlive()          → 是否存活
├── normalAttack(target) → 普攻
├── getSkills()        → 已装载法决列表
│
├── PlayerCombatant    → 玩家 + 装备 + 法决槽位
│   ├── getWeaponType()       → 法器类型（克制判定用）
│   ├── getMonsterTypeAdvantage() → 克制倍率
│   └── 攻防来源于 User 属性 + Equipment 加成
│
├── BeastCombatant     → 灵兽
│   ├── 攻防由品质 + 等级自动推算
│   └── skills 来自 xt_beast.skills
│
└── MonsterCombatant   → 怪物
    ├── 由 MonsterTemplate + 等级缩放生成
    └── skills 来自模板配置
```

```
Team（战斗团队）
├── ownerId            → 所属者（用于掉落归属）
├── members[]          → Combatant 列表
├── aliveMembers()     → 存活成员
├── isAllDead()        → 是否全灭
└── aliveCount()       → 存活数量
```

---

## 3. 通用战斗流程

```
simulate(teamA, teamB, maxRounds=20):
  repeat maxRounds:
    ① 按 speed 降序排列所有存活 combatant → 行动序列
    ② for each combatant in 行动序列:
         if not combatant.isAlive() → 跳过
         a. 普攻（按 targetStrategy 选目标）
         b. 法决自触发（CD 轮播）
    ③ 清理死亡单位
    ④ if teamB.isAllDead() → winner=teamA
    ⑤ if teamA.isAllDead() → winner=teamB
  → draw（超时平局）
```

### 返回 (CombatResult)

```
CombatResult
├── winner              → teamA / teamB / DRAW
├── rounds              → 实际回合数
├── hpChangesA[]        → teamA 各成员 HP 变化
├── hpChangesB[]        → teamB 各成员 HP 变化
├── damageDealt[]       → 各 combatant 造成的总伤害
├── skillProcs[]        → 法决触发记录 [{combatantId, skillName, count}]
└── combatLog[]         → 每次攻击的记录，按序排列
```

### 战斗日志条目 (CombatLogEntry)

```
CombatLogEntry
├── round               → 回合数 (1-based)
├── sequence            → 本回合内出手序号
├── attackerId          → 攻击方 ID
├── attackerName        → 攻击方名称
├── attackerType        → PLAYER / BEAST / MONSTER
├── defenderId          → 防御方 ID
├── defenderName        → 防御方名称
├── defenderType        → PLAYER / BEAST / MONSTER
├── attackType          → NORMAL / SKILL
├── skillName           → 法决名称（attackType=SKILL 时）
├── damageDealt         → 造成伤害值
├── isCrit              → 是否暴击（预留）
├── defenderHpBefore    → 防御方攻击前 HP
├── defenderHpAfter     → 防御方攻击后 HP
├── isKill              → 本次攻击是否击杀
└── advantageMultiplier → 克制倍率（仅玩家攻击时有效）
```

## 4. 目标选择策略 (TargetStrategy)

| 策略 | 适用场景 | 规则 |
|------|---------|------|
| **PVE_FOCUS_FIRE** | 历练 / 秘境 | teamA → 集火 teamB 中 HP 比例最低者；teamB → 随机选择 |
| **PVP_BALANCED** | PVP（预留） | 按 speed 比例分配攻击目标 |
| **PVP_TARGETED** | PVP 组队（预留） | 玩家可手动指定目标 |

## 5. 伤害公式

```
// 玩家伤害
playerAttack = totalStr × 2 + equipmentAttack
playerDamage = playerAttack × weaponTypeAdvantage(weaponType, monsterType)

// 灵兽伤害
beastDamage = beastAttack（不受法器克制影响）

// 怪物伤害
monsterDamage = monsterAttack × reverseAdvantage(weaponType, monsterType)

// 统一减免
最终伤害 = max(1, attackPower - targetDefense × 0.3)
```

## 6. 克制矩阵

| 法器克制怪物 | 玩家伤害 | 怪物伤害 |
|------------|---------|----------|
| 克制 | ×1.5 | ×0.8 |
| 被克制 | ×0.8 | ×1.5 |
| 互无关系 | ×1.0 | ×1.0 |

见 [物品装备.md §3](./物品装备.md#3-法器类型-weapontype-与克制矩阵) 完整对照表。

## 7. 法决集成

每场战斗中，每个 Combatant 的已装载法决按独立 CD 自动触发。

触发规则详见 [法决.md §7](./法决.md#7-战斗自动施放)。

**适用 Combatant**：仅 `PlayerCombatant` 和 `BeastCombatant`。
`MonsterCombatant` 的法决由怪物模板的 `skills` 字段定义，固定 CD 轮播。

## 8. 出击 / 参战规则

### 8.1 玩家

- 自动参战，不可离队
- 阵亡 → 全队判负

### 8.2 灵兽

来源 [福地.md](./福地.md)

- 在福地中选择"出战"灵兽（`xt_beast.is_deployed = true`）
- 仅 HP > 0 且未在休养中的灵兽可出战
- 出战上限：3 只（按玩家等级逐步解锁更多）
- 战斗获取经验，升级后属性自动成长

阵亡：

| 品质 | 休养时长 |
|------|---------|
| 凡品 | 30min |
| 灵品 | 1h |
| 仙品 | 2h |
| 圣品 | 4h |
| 神品 | 8h |

休养期间不可出战。

### 8.3 怪物

- 由 `MonsterTemplate` → 等级缩放生成
- 战斗中死亡即销毁
- 掉落归属由击杀者的 `Team.ownerId` 决定
