# 战斗系统

## 1. 设计原则

- **懒结算**：所有战斗均为后台模拟，无实时交互、无后台任务
- **通用引擎**：一套引擎覆盖所有战斗场景（历练 / 秘境 / PVP）
- **Team vs Team**：不关心具体类型，只操作 `Combatant` 接口

---

## 2. 核心架构

### 2.1 战斗引擎接口 (CombatEngine)

```
CombatEngine（接口）
├── simulate(context) → BattleResultVO
└── getEngineName() → String
```

**实现类：**
- `DefaultCombatEngine` - 默认PVE战斗引擎

### 2.2 战斗上下文 (BattleContext)

```
BattleContext
├── teamA → 攻击方队伍
├── teamB → 防御方队伍
├── maxRounds → 最大回合数（默认20）
├── scene → 战斗场景（TRAINING/DUNGEON/PVP/BOUNTY）
├── mapId → 地图ID
├── mapLevel → 地图等级
├── playerLevel → 玩家等级
└── gearScore → 装备评分
```

### 2.3 战斗单位 (Combatant)

```
Combatant（接口）
├── getId()            → 唯一标识
├── getName()          → 名称
├── getSpeed()         → 行动速度（决定出手顺序）
├── getAttack()        → 攻击力（已包含所有加成）
├── getDefense()       → 防御力
├── getHp() / getMaxHp() → 生命值
├── takeDamage(amount) → 受到伤害
├── heal(amount)       → 恢复生命值
├── isAlive()          → 是否存活
├── getSkills()        → 已装载法决列表
│
├── PlayerCombatant    → 玩家 + 装备 + 法决槽位
│   ├── getWeaponType()       → 法器类型（克制判定用）
│   └── getWis()              → 智慧（技能伤害计算用）
│
├── BeastCombatant     → 灵兽
│   ├── 攻防由品质 + 等级自动推算
│   └── skills 来自 xt_beast.skills（已启用）
│
└── Monster            → 怪物
    ├── 由 MonsterTemplate + 等级缩放生成
    └── skills 来自模板配置
```

### 2.4 战斗团队 (Team)

```
Team
├── ownerId            → 所属者（用于掉落归属）
├── name               → 团队名称
├── members[]          → Combatant 列表
├── aliveMembers()     → 存活成员
├── isAllDead()        → 是否全灭
├── aliveCount()       → 存活数量
├── selectTargetForPVE() → PVE目标选择（集火低血量）
└── selectTargetRandom() → 随机目标选择
```

## 3. 战斗流程

### 3.1 通用战斗流程

```
simulate(context):
  teamA = context.teamA
  teamB = context.teamB
  maxRounds = context.maxRounds
  buffManager = new BuffManager()

  repeat maxRounds:
    ① 处理持续效果（DOT/HEAL）
    ② 按 speed 降序排列所有存活 combatant → 行动序列
    ③ for each combatant in 行动序列:
         if 战斗单位被控制（眩晕/冰冻）→ 跳过行动
         if 战斗单位被沉默 → 不能使用技能
         a. 尝试法决触发（CD轮播）
         b. 根据技能效果类型处理：
            - DAMAGE/MULTI_HIT → 造成伤害
            - ARMOR_BREAK → 施加破甲Debuff
            - SLOW → 施加减速Debuff
            - DOT → 施加持续伤害Debuff
            - EXECUTE → 对低血量目标造成额外伤害
            - LIFESTEAL → 造成伤害并恢复生命
            - HEAL → 恢复队友生命值
            - ATTACK_BUFF/DEFENSE_BUFF/SPEED_BUFF → 施加增益Buff
            - STUN/FREEZE/SILENCE → 施加控制Debuff
         c. 普通攻击（无技能时）
    ④ 清理死亡单位
    ⑤ if teamB.isAllDead() → winner=teamA
    ⑥ if teamA.isAllDead() → winner=teamB
  → draw（超时平局）
```

### 3.2 返回结果 (BattleResultVO)

```
BattleResultVO
├── winner              → teamA / teamB / DRAW / NONE
├── rounds              → 实际回合数
├── playerHpChange      → 玩家HP变化 {name: {before, after}}
├── beastHpChanges[]    → 灵兽HP变化 [{name, after}]
├── monsterHpChanges[]  → 怪物HP变化 [{name, after}]
├── damageDealt[]       → 各combatant造成的总伤害 [{name, total}]
├── skillProcs[]        → 法决触发记录 [{key, count}]
├── combatLog[]         → 每次攻击的记录，按序排列
├── drops[]             → 掉落物品 [{type, templateId, name, quantity}]
├── expGained           → 获得的经验
└── summary             → 战斗摘要
```

### 3.3 战斗日志条目 (CombatLogEntry)

```
CombatLogEntry
├── round               → 回合数 (1-based)
├── sequence            → 本回合内出手序号
├── attackerName        → 攻击方名称
├── defenderName        → 防御方名称
├── attackType          → NORMAL / SKILL / CONTROLLED
├── skillName           → 法决名称（attackType=SKILL 时）
├── effectType          → 效果类型（如ARMOR_BREAK、SLOW等）
├── damageDealt         → 造成伤害值
├── isCrit              → 是否暴击（预留）
├── defenderHpBefore    → 防御方攻击前HP
├── defenderHpAfter     → 防御方攻击后HP
├── isKill              → 本次攻击是否击杀
├── buffApplied         → 是否施加了Buff/Debuff
├── buffTarget          → Buff/Debuff目标
└── description         → 控制效果描述（attackType=CONTROLLED时）
```

## 4. 目标选择策略

| 策略 | 适用场景 | 规则 |
|------|---------|------|
| **PVE_FOCUS_FIRE** | 历练 / 秘境 | teamA → 集火 teamB 中 HP 比例最低者；teamB → 随机选择 |

## 5. 伤害公式

```
// 玩家伤害
playerAttack = totalStr × 2 + equipmentAttack
playerDamage = playerAttack × weaponTypeAdvantage(weaponType, monsterType) × attackModifier

// 灵兽伤害
beastDamage = beastAttack × attackModifier（不受法器克制影响）

// 怪物伤害
monsterDamage = monsterAttack × reverseAdvantage(weaponType, monsterType) × attackModifier

// 统一减免（考虑破甲效果）
最终伤害 = max(1, attackPower - targetDefense × 0.3 × defenseModifier × defenseBuffModifier)

// 攻击增益
attackModifier = 1.0 + sum(ATTACK_BUFF.value)

// 防御修正（破甲效果）
defenseModifier = 1.0 - sum(ARMOR_BREAK.value × stackCount)

// 防御增益
defenseBuffModifier = 1.0 + sum(DEFENSE_BUFF.value)
```

```
Buff
```

## 8. 灵兽AI行为系统

### 8.1 AI行为接口 (BeastAI)

```
BeastAI（接口）
├── decideAction(beast, allies, enemies) → BeastAction
├── selectTarget(beast, enemies, allies) → Combatant
└── selectSkill(beast, target, allies, enemies) → Skill
```

### 8.2 行为策略枚举 (BeastAction)

| 策略 | 说明 | 触发条件 |
|------|------|----------|
| `ATTACK` | 攻击敌人 | 默认行为 |
| `DEFEND` | 防御（减少受到的伤害） | 玩家血量低时 |
| `SUPPORT` | 辅助（治疗/增益） | 有治疗/增益技能时 |
| `SKILL` | 使用技能 | 技能CD就绪时 |

### 8.3 配合玩家策略 (DefaultBeastAI)

**目标选择逻辑：**
1. 优先攻击对玩家威胁最大的敌人（攻击力最高的怪物）
2. 如果玩家血量低于30%，优先攻击攻击玩家的怪物
3. 如果有怪物即将被击杀（血量低于20%），优先集火

**行动决策逻辑：**
1. 如果有治疗技能且玩家血量低于50%，使用治疗
2. 如果有增益技能且未激活，使用增益
3. 如果有控制技能且目标可控制，使用控制
4. 否则，使用普通攻击

**技能选择逻辑：**
1. 如果有治疗技能且玩家血量低，选择治疗技能
2. 如果有增益技能，选择增益技能
3. 如果有控制技能且目标可控制，选择控制技能
4. 如果有伤害技能，选择伤害技能

## 9. 高光战斗系统

### 9.1 设计目标

在文字MUD挂机游戏中，每次历练可能有多场战斗，但只有1场高光战斗值得用LLM美化渲染，提升沉浸感。

### 9.2 高光战斗触发条件

| 条件 | 说明 | 权重 |
|------|------|------|
| 长回合战斗 | 战斗回合数 > 10 | 高 |
| 势均力敌 | 双方血量都低于30% | 高 |
| 稀有技能触发 | 触发了稀有法决或连击（>=3次） | 中 |

### 9.3 高光战斗检测器 (HighlightBattleDetector)

```
HighlightBattleDetector
├── detectHighlight(battleResult, battleIndex) → HighlightInfo
└── HighlightInfo
    ├── battleIndex → 战斗序号
    ├── reason → 高光原因
    └── rounds → 战斗回合数
```

### 9.4 LLM渲染策略

**输入：** 高光战斗数据 + 玩家角色信息 + 地图环境

**输出：** 修仙小说风格的战斗叙述（50-100字）

**风格要求：**
- 简洁有力，符合文字MUD风格
- 突出关键时刻和转折
- 包含技能名称和效果描述
- 体现灵兽配合和战斗策略

**示例：**
```
第5场战斗，你遭遇了一只剧毒蟾蜍王。它喷出的毒雾几乎遮蔽了你的视线，
但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
一剑刺穿了它的要害。这场战斗持续了5个回合，最终你以微弱优势胜出。
```

## 10. 动态遇怪系统

### 10.1 遇怪间隔计算 (EncounterCalculator)

```
encounterInterval = baseInterval × (1 - difficultyFactor) × (1 - levelFactor) × (1 - gearFactor)
```

**因子计算：**
- `baseInterval` = 10分钟（基础间隔）
- `difficultyFactor` = min(0.5, mapLevel × 0.005)（地图难度因子）
- `levelFactor` = min(0.3, playerLevel × 0.0015)（玩家等级因子）
- `gearFactor` = min(0.2, gearScore × 0.0002)（装备评分因子）

**最小间隔：** 2分钟

### 10.2 遇怪概率

```
encounterChance = min(1.0, 0.4 × (baseInterval / encounterInterval))
```

间隔越短，概率越高。

### 10.3 装备评分计算

```
gearScore = sum(equipment.finalAttack + equipment.finalDefense × 2) + rarityBonus
```

**稀有度加成：**
- 普通 (common) → +10
- 稀有 (rare) → +40
- 史诗 (epic) → +80
- 传说 (legendary) → +160

## 11. 出击 / 参战规则

### 11.1 玩家

- 自动参战，不可离队
- 阵亡 → 全队判负

### 11.2 灵兽

来源 [福地.md](./福地.md)

- 在福地中选择"出战"灵兽（`xt_beast.is_deployed = true`）
- 仅 HP > 0 且未在休养中的灵兽可出战
- 出战上限：3 只（按玩家等级逐步解锁更多）
- 战斗获取经验，升级后属性自动成长
- **技能已启用**：灵兽可以使用 `xt_beast.skills` 中的技能

阵亡：

| 品质 | 休养时长 |
|------|---------|
| 凡品 | 30min |
| 灵品 | 1h |
| 仙品 | 2h |
| 圣品 | 4h |
| 神品 | 8h |

休养期间不可出战。

### 11.3 怪物

- 由 `MonsterTemplate` → 等级缩放生成
- 战斗中死亡即销毁
- 掉落归属由击杀者的 `Team.ownerId` 决定
