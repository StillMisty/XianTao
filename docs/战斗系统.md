# 战斗系统

## 1. 设计原则

- **懒结算**：所有战斗均为后台模拟，无实时交互、无后台任务
- **通用引擎**：一套引擎覆盖所有战斗场景（历练 / 秘境 / PVP）
- **Team vs Team**：不关心具体类型，只操作 `Combatant` 接口

---

## 2. 服务层架构

### 2.1 职责划分

```
TrainingService（历练编排）
├── 遇敌概率计算（EncounterCalculator）
├── 遇怪循环：加权选择 → 构建队伍 → 调用 CombatService.simulate()
├── 掉落计算（DropProcessor.processMonsterDrops()）
├── 战后 HP 回写（PostCombatProcessor）
├── 灵兽休养/觉醒判定
└── 掉落分发到背包（DropProcessor.distributeDrops()）

CombatService（单场战斗，纯引擎）
├── simulate(Team, Team, int) → BattleResultVO
├── buildPlayerTeam(User) → Team
└── buildMonsterTeam(MonsterTemplate, int) → Team
```

| 组件 | 位置 | 职责 |

## 3. 核心架构

### 3.0 战斗实体 (Battle)

```
Battle（领域实体）
├── teamA → 攻击方队伍
├── teamB → 防御方队伍
├── scene → 战斗场景
├── maxRounds → 最大回合数
├── of(teamA, teamB, scene, maxRounds, engine) → Battle  // 工厂方法
├── execute() → BattleResultVO                             // 执行战斗
├── isTeamAWin() → boolean                                 // 攻击方是否胜
└── isTeamADead() → boolean                                // 攻击方是否全灭
```

泛型设计：只依赖 `Combatant` 接口和 `Team`，不关心具体类型（玩家/灵兽/怪物）。
战后结算（HP 回写、掉落、灵兽休养）由调用方（Service 层）处理。

### 2.3 战斗单位 (Combatant)

```
Combatant（接口）
├── getId()            → 唯一标识
├── getName()          → 名称
├── getSpeed()         → 行动速度（决定出手顺序）
├── getAttack()        → 攻击力（已包含所有加成）
├── getDefense()       → 防御力
├── getHp() / getMaxHp() → 生命值
├── takeDamage(amount) → 受到伤害
├── heal(amount)       → 恢复生命值
├── isAlive()          → 是否存活
├── getSkills()        → 已装载法决列表
│
├── PlayerCombatant    → 玩家 + 装备 + 法决槽位
│   ├── getWeaponType()       → 法器类型（克制判定用）
│   └── getWis()              → 智慧（技能伤害计算用）
│
├── BeastCombatant     → 灵兽（视为普通 Combatant，无独立 AI）
│   ├── 攻防由品质 + 等级自动推算

│
└── Monster            → 怪物
    ├── 由 MonsterTemplate + 等级缩放生成
    └── skills 来自模板配置

> 所有 Combatant 子类在战斗中行为一致：按速度排序出手，有技能则轮播触发，无技能则普攻。
> 灵兽、怪物、玩家之间不做差异化处理。
```

## 4. 战斗流程

### 4.1 通用战斗流程

```
simulate(context):
  teamA = context.teamA
  teamB = context.teamB
  maxRounds = context.maxRounds
  buffManager = new BuffManager()

  repeat maxRounds:
    ① 处理持续效果（DOT/HEAL）
    ② 按 speed 降序排列所有存活 combatant → 行动序列
    ③ for each combatant in 行动序列:
         if 战斗单位被控制（眩晕/冰冻）→ 跳过行动
         if 战斗单位被沉默 → 不能使用技能
         a. 尝试法决触发（CD轮播，受攻速影响）
         b. 根据技能效果类型处理：
            - DAMAGE/MULTI_HIT → 造成伤害
            - ARMOR_BREAK → 施加破甲Debuff
            - SLOW → 施加减速Debuff
            - DOT → 施加持续伤害Debuff
            - EXECUTE → 对低血量目标造成额外伤害
            - LIFESTEAL → 造成伤害并恢复生命
            - HEAL → 恢复队友生命值
            - ATTACK_BUFF/DEFENSE_BUFF/SPEED_BUFF → 施加增益Buff
            - STUN/FREEZE/SILENCE → 施加控制Debuff
         c. 普通攻击（无技能时）
    ④ 更新法决CD（每回合 CD -= attackSpeed）
    ⑤ 清理死亡单位
    ⑥ if teamB.isAllDead() → winner=teamA
    ⑦ if teamA.isAllDead() → winner=teamB
  → draw（超时平局）
```

### 4.2 返回结果 (BattleResultVO)

```
BattleResultVO
├── winner              → teamA / teamB / DRAW / NONE
├── rounds              → 实际回合数
├── playerHpChange      → 玩家HP变化 {name: {before, after}}
├── beastHpChanges[]    → 灵兽HP变化 [{name, after}]
├── monsterHpChanges[]  → 怪物HP变化 [{name, after}]
├── damageDealt[]       → 各combatant造成的总伤害 [{name, total}]
├── skillProcs[]        → 法决触发记录 [{key, count}]
├── combatLog[]         → 每次攻击的记录，按序排列
├── drops[]             → 掉落物品 [{type, templateId, name, quantity}]
├── expGained           → 获得的经验
└── summary             → 战斗摘要
```

### 4.3 战斗日志条目 (CombatLogEntry)

```
CombatLogEntry
├── round               → 回合数 (1-based)
├── sequence            → 本回合内出手序号
├── attackerName        → 攻击方名称
├── defenderName        → 防御方名称
├── attackType          → NORMAL / SKILL / CONTROLLED
├── skillName           → 法决名称（attackType=SKILL 时）
├── effectType          → 效果类型（如ARMOR_BREAK、SLOW等）
├── damageDealt         → 造成伤害值
├── isCrit              → 是否暴击（预留）
├── defenderHpBefore    → 防御方攻击前HP
├── defenderHpAfter     → 防御方攻击后HP
├── isKill              → 本次攻击是否击杀
├── buffApplied         → 是否施加了Buff/Debuff
├── buffTarget          → Buff/Debuff目标
└── description         → 控制效果描述（attackType=CONTROLLED时）
```

## 6. 伤害公式

```
// 玩家伤害
playerAttack = totalStr × 2 + equipmentAttack
playerDamage = playerAttack × weaponTypeAdvantage(weaponType, monsterType) × attackModifier

// 灵兽伤害
beastDamage = beastAttack × attackModifier（不受法器克制影响）

// 怪物伤害
monsterDamage = monsterAttack × reverseAdvantage(weaponType, monsterType) × attackModifier

// 统一减免（考虑破甲效果）
最终伤害 = max(1, attackPower - targetDefense × 0.3 × defenseModifier × defenseBuffModifier)

// 攻击增益
attackModifier = 1.0 + sum(ATTACK_BUFF.value)

// 防御修正（破甲效果）
defenseModifier = 1.0 - sum(ARMOR_BREAK.value × stackCount)

// 防御增益
defenseBuffModifier = 1.0 + sum(DEFENSE_BUFF.value)
```

## 7. 速度与攻速

### 7.1 行动顺序（速度）

每回合按速度降序排列，速度高的先行动：

```
// 玩家
speed = agility × 2 + 10

// 怪物
speed = baseSpeed × (1 + (level - baseLevel) × 0.1)

// 灵兽
speed = level × 2 + 8
```

### 7.2 法决 CD 恢复（攻速）

每回合法决 CD 恢复量受攻速影响：

```
每回合 CD 减少量 = 1 × attackSpeed
```

- `attackSpeed`：法器专属属性，存储在 `EquipmentTemplate.attack_speed`
- 攻速 1.0 → 正常恢复（每回合 CD -1）
- 攻速 1.2 → CD 恢复快 20%（每回合 CD -1.2）
- 攻速 0.8 → CD 恢复慢 20%（每回合 CD -0.8）

**效果：** 高攻速法器法决触发更频繁，但不增加普通攻击次数。

### 7.3 法器攻速参考

| 法器 | 攻速 | 说明 |
|------|------|------|
| 剑 | 1.2 | 快速，法决频繁触发 |
| 刀 | 1.0 | 标准 |
| 枪 | 0.9 | 略慢 |
| 棍 | 0.8 | 慢，但单次伤害高 |
| 斧 | 0.7 | 最慢，爆发力强 |
| 弓 | 1.1 | 远程，较快 |

## 11. 高光战斗系统

### 11.1 设计目标

在文字MUD挂机游戏中，每次历练可能有多场战斗，但只有1场高光战斗值得用LLM美化渲染，提升沉浸感。

### 11.2 高光战斗触发条件

| 条件 | 说明 | 权重 |
|------|------|------|
| 长回合战斗 | 战斗回合数 > 10 | 高 |
| 势均力敌 | 双方血量都低于30% | 高 |
| 稀有技能触发 | 触发了稀有法决或连击（>=3次） | 中 |

### 11.3 高光战斗检测器 (HighlightBattleDetector)

```
HighlightBattleDetector
├── detectHighlight(battleResult, battleIndex) → HighlightInfo
└── HighlightInfo
    ├── battleIndex → 战斗序号
    ├── reason → 高光原因
    └── rounds → 战斗回合数
```

### 11.4 LLM渲染策略

**输入：** 高光战斗数据 + 玩家角色信息 + 地图环境

**输出：** 修仙小说风格的战斗叙述（50-100字）

**风格要求：**
- 简洁有力，符合文字MUD风格
- 突出关键时刻和转折
- 包含技能名称和效果描述
- 体现灵兽配合和战斗策略

**示例：**
```
第5场战斗，你遭遇了一只剧毒蟾蜍王。它喷出的毒雾几乎遮蔽了你的视线，
但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
一剑刺穿了它的要害。这场战斗持续了5个回合，最终你以微弱优势胜出。
```

## 12. 动态遇怪系统

### 12.1 遇怪间隔计算 (EncounterCalculator)

```
encounterInterval = baseInterval × (1 - difficultyFactor) × (1 - levelFactor) × (1 - gearFactor)
```

**因子计算：**
- `baseInterval` = 10分钟（基础间隔）
- `difficultyFactor` = min(0.5, mapLevel × 0.005)（地图难度因子）
- `levelFactor` = min(0.3, playerLevel × 0.0015)（玩家等级因子）
- `gearFactor` = min(0.2, gearScore × 0.0002)（装备评分因子）

**最小间隔：** 2分钟

### 12.2 遇怪概率

```
encounterChance = min(1.0, 0.4 × (baseInterval / encounterInterval))
```

间隔越短，概率越高。

### 12.3 装备评分计算

```
gearScore = sum(equipment.finalAttack + equipment.finalDefense × 2) + rarityBonus
```

**稀有度加成：**
- 普通 (common) → +10
- 稀有 (rare) → +40
- 史诗 (epic) → +80
- 传说 (legendary) → +160

## 13. 出击 / 参战规则

### 13.1 玩家

- 自动参战，不可离队
- 阵亡 → 全队判负

### 13.2 灵兽

来源 [福地.md](./福地.md)

- 在福地中选择"出战"灵兽（`xt_beast.is_deployed = true`）
- 仅 HP > 0 且未在休养中的灵兽可出战
- 出战上限：3 只（按玩家等级逐步解锁更多）
- 战斗获取经验，升级后属性自动成长
- **技能已启用**：灵兽可以使用 `xt_beast.skills` 中的技能
- **无独立 AI**：灵兽在战斗中与其他 Combatant 行为一致，无特殊策略

阵亡：

| 品质 | 休养时长 |
|------|---------|
| 凡品 | 30min |
| 灵品 | 1h |
| 仙品 | 2h |
| 圣品 | 4h |
| 神品 | 8h |

休养期间不可出战。

### 13.3 怪物

- 由 `MonsterTemplate` → 等级缩放生成
- 战斗中死亡即销毁
- 掉落归属由击杀者的 `Team.ownerId` 决定
