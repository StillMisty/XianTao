# 仙道(XianTao)系统设计文档

## 1. 系统概述

### 1.1 项目简介
仙道是一款跨平台文字修仙MUD游戏，采用极低频操作设计，支持离线懒加载计算和AI驱动的NPC深度交互。系统基于领域驱动设计(DDD)架构，支持多平台接入(QQ、Web等)。

### 1.2 核心特性
- **跨平台统一命令系统**：单一命令逻辑适配多个平台入口
- **离线懒加载计算**：摒弃定时任务，在玩家查询时即时计算离线收益
- **AI深度集成**：地灵MBTI人格化对话与Function Calling意图识别
- **虚拟线程并发**：基于Java 25 Virtual Threads处理高并发聊天消息
- **JSONB灵活存储**：PostgreSQL JSONB字段支持无模式数据存储

### 1.3 技术栈
- **运行环境**：Java 25 (Virtual Threads, Records, Sealed Classes)
- **框架**：Spring Boot 4.x + Spring AI 2.x
- **ORM**：MyBatis-Flex
- **数据库**：PostgreSQL 18 (重度使用JSONB)
- **机器人引擎**：SimBot (跨平台：OneBotV11、官方QQ、Web)
- **构建工具**：Gradle Kotlin DSL

## 2. 架构设计

### 2.1 分层架构 (Ports & Adapters)

#### 表现层 (handle/)
负责所有外部输入输出的解析与渲染，是系统的适配器层。
- **command/**：统一命令分发中心，调用服务层并格式化返回数据
- **onebotv11/**：OneBotV11协议监听器，处理QQ平台消息
- **web/**：REST API控制器，提供HTTP接口

**职责边界**：仅负责平台I/O转换，不包含业务逻辑。

#### 领域层 (domain/)
包含核心业务逻辑的限界上下文，遵循依赖倒置原则。
- **user/**：用户上下文（角色、属性、状态）
- **map/**：地图上下文（节点、连接、事件）
- **item/**：物品上下文（背包、装备、锻造）
- **land/**：福地上下文（种植、养殖、地灵）

每个子域包含：
- **entity/**：数据库实体
- **enums/**：领域枚举
- **repository/**：仓储接口（端口定义）
- **vo/**：值对象/数据传输对象

#### 服务层 (service/)
集中化的命令实现与调度器，平台无关的业务编排。
- **CultivationService**：修仙核心服务（加点、洗点、突破、护道）
- **StaminaService**：体力管理服务
- **TrainingService**：历练收益计算
- **ExplorationService**：区域探索
- **TravelService**：地图旅行
- **FudiService**：福地管理
- **SpiritChatService**：地灵对话

#### 基础设施层 (infrastructure/)
技术实现细节，是领域层的适配器。
- **mapper/**：MyBatis-Flex数据访问接口
- **repository/**：仓储接口实现
- **mybatis/handler/**：自定义类型处理器（如PgJsonbTypeHandler）

### 2.2 跨平台命令流转

```
平台监听器 → 命令分发器 → 领域服务 → 数据持久化 → 命令分发器 → 平台视图渲染
```

**关键设计**：
- 所有平台共享同一套命令处理逻辑（handle/command/）
- 平台特定包只负责I/O转换，不重复实现业务逻辑
- 命令处理器通过UserAuthService进行身份认证

## 3. 核心领域模型

### 3.1 用户系统 (User Context)

#### 3.1.1 角色实体 (xt_user)
游戏角色的核心数据载体，包含以下维度：

**基础信息**
- 内部唯一ID、道号（唯一约束）
- 等级、经验值、铜币、灵石

**四维属性体系**
- **力量(STR)**：影响破坏力、锻造成功率
- **体质(CON)**：影响生命值上限、物理防御、体力上限
- **敏捷(AGI)**：影响出手顺序、历练效率倍率
- **智慧(WIS)**：影响经验加成、探索掉宝率、炼药成功率

**战斗状态**
- 当前生命值、当前体力值
- 状态枚举（空闲、打坐、历练、旅行等）
- 当前位置ID

**时间戳追踪**
- 历练开始时间（用于懒加载结算）
- 旅行开始时间与目的地（用于进度计算）
- 上次体力更新时间（用于离线恢复）
- 上次洗点时间（用于冷却判定）

**扩展数据(JSONB)**
- 称号、成就、临时状态等小规模系统数据

#### 3.1.2 跨平台授权 (xt_user_auth)
实现一个游戏角色绑定多个平台账号的能力。

**核心约束**：
- 同一平台的OpenID只能绑定到一个角色
- 一个角色可以绑定多个不同平台的账号
- 外键级联删除：角色删除时自动清除所有绑定

#### 3.1.3 护道关系 (xt_dao_protection)
记录修士之间的护道契约关系。

**字段说明**：
- 护道者ID、被护道者ID
- 建立时间

**业务规则**：
- 护道者境界必须 ≥ 被护道者境界
- 单个护道者最多同时为3人护道
- 突破成功或失败后自动清除该被护道者的所有护道关系

### 3.2 地图系统 (Map Context)

#### 3.2.1 地图节点 (xt_map_node)
表示游戏中的地理位置，构成有向图结构。

**基础属性**
- 名称、描述、类型（主城、野外、秘境等）
- 推荐等级要求

**连接关系(JSONB)**
- 相邻地图及其旅行耗时
- 示例：{"黑金主城": 5, "枯骨林": 10} 表示到黑金主城需5分钟，到枯骨林需10分钟

**特产掉落池(JSONB)**
- 加权随机掉落列表
- 格式：[{"name": "毒龙草", "weight": 30}, {"name": "铁矿石", "weight": 50}]

**旅行事件权重(JSONB)**
- 触发事件的概率分布
- 格式：[{"eventType": "ambush", "weight": 40}, {"eventType": "find_treasure", "weight": 10}]

#### 3.2.2 地图连通性验证
- 检查玩家等级是否满足目标地图要求
- 验证两地图是否相邻（直接可达）
- 获取相邻地图列表供玩家选择

### 3.3 物品系统 (Item Context)

#### 3.3.1 物品模板 (xt_item_template)
定义物品的静态配置，是所有物品实例的原型。

**分类体系**
- 装备类：武器、防具、饰品
- 消耗品类：丹药、材料
- 种子/灵蛋类：可种植/孵化的特殊物品

**装备属性**
- 稀有度（破损、普通、稀有、史诗、传说）
- 装备部位（头、胸、腿、手、足、主手、副手）
- 基础攻击力、防御力
- 四维属性加成(JSONB)
- 装备等级（用于动态计算词条数值）

**掉落权重配置(JSONB)**
- 控制不同稀有度的生成概率
- 示例：{"BROKEN": 50, "COMMON": 30, "RARE": 15, "EPIC": 4, "LEGENDARY": 1}

**标签系统(JSONB)**
- 用于AI检索和NPC交互的语义标签
- 示例：["ore", "metal", "forge_base"]

**生长机制（种子/灵蛋专用）**
- 生长时间（小时）
- 成熟后产出的物品模板ID
- 存活率百分比

#### 3.3.2 装备实例 (xt_equipment)
玩家背包中的具体装备对象，继承模板属性并附加随机词条。

**实例化属性**
- 持有者用户ID、模板ID
- 实际名称（可能因前缀/后缀变化）
- 当前装备状态
- 品质乘数（影响最终属性）
- 随机词条(JSONB)
- 锻造等级

### 3.4 福地系统 (Land Context)

#### 3.4.1 福地实体 (xt_fudi)
玩家的私人领地，包含灵气管理、网格布局、地灵成长等子系统。

**灵气系统**
- 当前灵气值、灵气上限
- 每小时灵气消耗（基于聚灵核心等级和地块数量）
- 最后灵气更新时间（用于懒加载计算）

**网格布局(JSONB)**
- 网格大小（初始3x3，可升级扩展）
- 地块列表（灵田、兽栏、阵眼）
- 每个地块包含：坐标、类型、五行属性、作物/灵兽信息、生长进度

**地灵系统**
- MBTI人格类型（16种人格之一）
- 形态阶段（初创期、底蕴期、化形期）
- 地灵等级、精力值、好感度
- 情绪状态（平静、愉悦、疲惫、愤怒等）
- 表情变体列表（根据情绪动态切换Emoji）

**天劫系统**
- 上次渡劫时间
- 连胜次数（连续成功渡劫的次数）
- 焦土坐标列表（渡劫失败导致的损坏地块）

**自动化控制**
- 自动模式开关（地灵自主决策）
- 休眠模式开关（降低灵气消耗）

#### 3.4.2 地块类型

**灵田(FARM)**
- 五行属性（金木水火土）
- 作物ID、作物名称
- 种植时间、预计成熟时间
- 生长进度（0.0-1.0）
- 生长速度修正系数（受相邻地块五行相生相克影响）

**兽栏(PEN)**
- 灵兽ID、灵兽名称、灵兽品阶
- 饥饿值（0-100，低于30无法产出）
- 最后喂食时间

**阵眼(NODE)**
- 五行属性
- 等级（影响灵气产出效率）
- 耐久度

## 4. 核心业务规则

### 4.1 修炼系统

#### 4.1.1 升级机制
**经验公式**：升级到下一级所需经验 = 100 × 当前等级²

**经验存储上限**：最多存储当前等级升级经验的5倍，超出部分无法获取。

**升级流程**：
1. 检查当前经验是否达到升级阈值
2. 扣除升级所需经验
3. 等级+1，获得1点自由属性点
4. 生命值恢复至满值
5. 重置突破失败计数为0
6. 清除该角色的所有被护道关系

#### 4.1.2 突破成功率算法
突破成功率随等级提升呈S型曲线下降，并通过失败补偿机制保证长期可玩性。

**基础概率公式**：
```
基础概率 = 100 / (1 + (等级 / 50)^4.5)
```

**失败补偿增量**：
```
单次补偿 = 3 + 17 / (1 + (等级 / 50)²)
总成功率 = 基础概率 + (失败次数 × 单次补偿)
```

**范围限制**：最终成功率限制在 [0%, 100%] 区间内。

**突破结果**：
- **成功**：等级提升、经验扣除、获得属性点、重置失败计数、清除护道关系
- **失败**：扣除当前境界升至下境界所需的标准修为值、失败次数+1、清除护道关系

#### 4.1.3 属性分配
**基础值**：每个属性初始值为5点。

**分配规则**：
- 消耗可用属性点进行分配
- 可随时调整分配方案（需通过洗点机制）

**洗点机制**：
- 将所有已分配的属性点返还为基础值（5点）
- 返还的点数加入可用属性点池
- 冷却时间：72小时（3天）
- 用途：允许玩家重新规划属性Build

### 4.2 体力系统

#### 4.2.1 体力上限计算
```
最大体力 = 100 + 体质 × 5
```

#### 4.2.2 离线恢复机制（懒加载）
系统不主动更新体力值，而是在玩家查询或执行操作时即时计算。

**恢复速率**：
- **空闲状态**：每5分钟恢复1点体力
- **打坐状态**：每分钟恢复2点体力
- **其他状态**：不恢复

**计算公式**：
```
经过分钟数 = (当前时间 - 上次更新时间) 的分钟差
恢复量 = 经过分钟数 × 恢复速率
实际恢复量 = min(恢复量, 最大体力 - 当前体力)
```

#### 4.2.3 体力消耗场景
- **历练**：每分钟消耗1点体力
- **探索**：每次固定消耗20点体力
- **旅行（体力模式）**：每分钟消耗5点体力（立即完成旅行）

### 4.3 历练系统 (AFK收益)

#### 4.3.1 懒加载结算原理
系统仅在玩家结束历练时计算收益，而非实时累积。

**结算时机**：
- 玩家主动结束历练
- 玩家查询历练状态时展示预估收益

**时长计算**：
```
历练分钟数 = (当前时间 - 历练开始时间) 的分钟差
```

#### 4.3.2 收益计算公式

**铜币收益**：
```
铜币 = 10 × 历练分钟数 × 效率倍率
效率倍率 = 1.0 + (敏捷 × 0.01)
```

**灵石收益**：
```
灵石 = 5 × (历练分钟数 / 60)  // 按小时计算，不足1小时不计
```

**物品掉落**：
```
掉落次数 = (历练分钟数 / 10) × 效率倍率
每次掉落从当前地图特产池中随机抽取
```

**经验收益**：
```
经验 = 10 + (智慧 × 2) + (地图等级 × 5)
```

### 4.4 探索系统

#### 4.4.1 有效智慧计算
探索难度受地图等级影响，高等级地图需要更高智慧才能发现稀有物品。

```
有效智慧 = 智慧 - (地图等级 - 1) × 5
最低有效智慧 = 1
```

#### 4.4.2 探索结果分级

| 有效智慧范围 | 结果类型 | 奖励内容 |
|------------|---------|---------|
| 1-20 | 基础发现 | 1个基础材料 |
| 21-50 | 稀有发现 | 1个稀有材料 |
| 51-80 | 重大发现 | 2个稀有材料 + 30%概率获得配方 |
| 81+ | 惊人发现 | 3个稀有材料 + 必定获得高级配方 + 20%概率获得灵蛋 |

#### 4.4.3 LLM美化描述
探索结果通过Spring AI调用LLM生成个性化描述，传入参数包括：
- 地图名称
- 事件类型
- 发现的物品名称列表

### 4.5 旅行系统

#### 4.5.1 旅行模式
**真实时间模式**：
- 设置旅行开始时间和目的地
- 玩家状态变更为"旅行中"
- 到达时间 = 开始时间 + 地图间旅行耗时
- 到达后需手动确认完成旅行

**体力跳过模式**：
- 立即完成旅行
- 体力消耗 = 旅行耗时(分钟) × 5
- 直接到达目的地

#### 4.5.2 旅行事件系统
到达目的地时触发D20骰点判定：

**事件触发条件**：
- D20 ≤ 10：触发随机事件（根据地图事件权重表）
- D20 > 10：安全通行

**事件类型**：
- **伏击(Ambush)**：受到10-30点伤害
- **发现宝藏(Find Treasure)**：获得50-150铜币
- **恶劣天气(Weather)**：额外消耗体力或降低收益
- **安全通行(Safe Passage)**：无事件发生

#### 4.5.3 旅行约束
- 只能前往相邻地图
- 必须满足目标地图等级要求
- 旅行中无法执行其他操作（历练、探索等）

### 4.6 护道系统

#### 4.6.1 护道加成计算
**单人加成公式**：
```
单人加成 = 5% + (护道者等级 - 被护道者等级) × 1%
```

**总加成计算**：
- 仅统计与被护道者在同一地点的护道者
- 累加所有同地点护道者的单人加成
- 总加成上限：20%

#### 4.6.2 护道约束
- 护道者境界必须 ≥ 被护道者境界
- 单个护道者最多同时为3人护道
- 被护道者突破（无论成功或失败）后，所有护道关系自动解除

#### 4.6.3 护道查询
提供双向查询能力：
- **我护道的人**：显示被护道者信息、是否同地点、提供的加成
- **护道我的人**：显示护道者信息、是否同地点、实际生效的加成

### 4.7 福地系统

#### 4.7.1 灵气管理机制

**灵气消耗公式**：
```
每小时消耗 = 基础消耗(10) + 地块数量 × 2 + 聚灵核心等级 × 5
```

**灵气恢复**：
- 通过献祭装备获得灵气
- 献祭收益 = 物品基础价值 × 品质乘数 × 等级修正

**品质乘数表**：
- 白色(White)：0.5x
- 绿色(Green)：1.0x
- 蓝色(Blue)：2.0x
- 紫色(Purple)：5.0x
- 橙色(Orange)：10.0x

**灵气溢出保护**：当前灵气不能超过灵气上限，超出部分浪费。

#### 4.7.2 网格布局系统

**坐标系统**：
- 使用(x,y)坐标表示地块位置
- 初始网格大小为3x3，可通过升级扩展
- 坐标示例："0,0"、"1,1"、"2,2"

**地块占用检查**：
- 建造/种植前检查坐标是否为空
- 拆除前检查坐标是否存在地块

**相邻地块五行影响**：
种植作物时，相邻地块的五行属性会影响生长速度。

**五行相生相克修正系数**：
- 相生关系：1.2x 加速
- 相同属性：1.0x 无影响
- 相克关系：0.8x 减速

#### 4.7.3 种植收获机制

**生长进度计算（懒加载）**：
```
生长进度 = min(1.0, (当前时间 - 种植时间) / (成熟时间 - 种植时间))
```

**实际生长时间**：
```
实际生长时间 = 基础生长时间 / 生长速度修正系数
```

**产量计算**：
```
基础产量 = 1-3随机
聚灵核心加成 = 核心等级 / 5（向下取整）
最终产量 = 基础产量 + 聚灵核心加成
```

**收获约束**：
- 只有生长进度 ≥ 1.0 的作物才能收获
- 收获消耗地灵精力（每块灵田5点精力）
- 收获后地块变为空地

#### 4.7.4 兽栏喂养机制

**饥饿值系统**：
- 初始饥饿值：100
- 每次喂食恢复30点饥饿值
- 饥饿值低于30时灵兽停止产出
- 喂养消耗地灵精力（每次3点精力）

#### 4.7.5 地灵MBTI人格系统

**16种人格分类**：
- **理性组**：INTJ、INTP、ENTJ、ENTP
- **理想组**：INFJ、INFP、ENFJ、ENFP
- **行动组**：ISTP、ISFP、ESTP、ESFP
- **关怀组**：ISTJ、ISFJ、ESTJ、ESFJ

**人格化对话**：
- 根据MBTI类型生成不同的语气和表达方式
- 理性组：简洁、逻辑性强
- 理想组：温和、富有同理心
- 行动组：热情、直接
- 关怀组：体贴、细致

**情绪状态机**：
- 平静(Calm)：默认状态
- 愉悦(Joyful)：灵气充足、福地繁荣时
- 疲惫(Tired)：精力值低时
- 愤怒(Angry)：灵气不足、地块损坏时
- 庆祝(Celebrating)：渡劫成功时

**形态演化三阶段**：
1. **初创期(Stage 1)**：基础对话功能，无福地详情感知
2. **底蕴期(Stage 2)**：可感知福地状态，提供建设建议
3. **化形期(Stage 3)**：完全人格化，主动管理福地

#### 4.7.6 Function Calling意图识别

**工作流程**：
1. 用户输入自然语言指令（如"@地灵 在0,0种灵芝"）
2. LLM分析用户意图，提取结构化参数
3. 后端执行对应操作（种植、收获、建造等）
4. LLM根据地灵人格生成回复

**支持的意图类型**：
- PLANT：种植作物
- HARVEST：收获作物
- BUILD：建造地块
- REMOVE：拆除地块
- SACRIFICE：献祭装备
- FEED：喂养灵兽
- STATUS：查询福地状态
- GRID：查询网格布局
- AURA：查询灵气详情
- SPIRIT_INFO：查询地灵信息
- CHAT：纯聊天

**降级方案**：
当LLM服务不可用时，使用规则匹配（关键词提取）作为后备方案。

#### 4.7.7 天劫系统

**触发周期**：每7天触发一次天劫。

**天劫结果**：
- **成功**：地灵等级提升、连胜次数+1、获得奖励
- **失败**：随机地块变为焦土（无法使用）、连胜次数归零

**焦土修复**：
- 需要消耗特殊道具或等待自动恢复
- 修复期间该坐标无法建造任何地块

## 5. 数据持久化策略

### 5.1 PostgreSQL JSONB应用

#### 5.1.1 使用场景
- **地图连接关系**：neighbors字段存储相邻地图及耗时
- **掉落池配置**：specialties字段存储加权掉落列表
- **装备词条**：affixes字段存储随机生成的属性词条
- **福地网格**：gridLayout字段存储所有地块的完整状态
- **用户扩展数据**：extraData字段存储称号、成就等

#### 5.1.2 索引优化
对所有JSONB字段创建GIN索引以支持高效查询：
```sql
CREATE INDEX idx_item_template_tags ON xt_item_template USING GIN (tags);
```

#### 5.1.3 类型映射
使用自定义PgJsonbTypeHandler实现Java对象与PostgreSQL JSONB的双向序列化。

### 5.2 懒加载设计模式

#### 5.2.1 核心思想
避免后台定时任务，改为在数据读取时即时计算。

#### 5.2.2 应用场景

**体力恢复**：
- 不每分钟更新体力值
- 查询时计算：(当前时间 - 上次更新时间) × 恢复速率

**历练收益**：
- 不实时累积铜币/经验
- 结束时计算：(当前时间 - 开始时间) × 单位时间收益

**作物生长**：
- 不每秒更新生长进度
- 查询时计算：(当前时间 - 种植时间) / 总生长时间

**优势**：
- 减少数据库写入压力
- 支持真正的离线计算
- 简化并发控制

## 7. 并发与性能优化

### 7.1 Virtual Threads应用

#### 7.1.1 适用场景
- SimBot消息监听器（高并发聊天消息）
- HTTP请求处理（Web平台接入）
- LLM异步调用

#### 7.1.2 注意事项
- 避免使用ThreadLocal（虚拟线程不支持）
- 使用UserContext.setCurrentUserId()传递用户上下文
- 确保在finally块中清理上下文（UserContext.clear()）

### 7.2 事务管理

#### 7.2.1 事务边界
- Service层方法添加@Transactional注解
- 确保数据一致性（如：消耗体力+开始历练）

#### 7.2.2 懒加载与事务
- 懒加载计算不涉及数据库写入时无需事务
- 涉及状态更新时必须保证原子性

### 7.3 数据库优化

#### 7.3.1 索引策略
- 排行榜常用字段：level、coins、spirit_stones（降序索引）
- 外键字段：user_id、platform_open_id
- JSONB字段：使用GIN索引支持标签搜索

#### 7.3.2 查询优化
- 避免N+1查询问题（使用JOIN或批量查询）
- 合理使用缓存（如地图配置、物品模板）

## 附录A：核心公式汇总

### A.1 角色成长
- 升级所需经验：`100 × level²`
- 经验存储上限：`升级经验 × 5`
- 最大生命值：`100 + 体质 × 20`
- 最大体力值：`100 + 体质 × 5`

### A.2 突破系统
- 基础成功率：`100 / (1 + (level / 50)^4.5)`
- 失败补偿：`3 + 17 / (1 + (level / 50)²)`
- 总成功率：`min(100, 基础概率 + 失败次数 × 单次补偿)`

### A.3 历练收益
- 铜币：`10 × 分钟数 × (1 + 敏捷 × 0.01)`
- 灵石：`5 × (分钟数 / 60)`
- 经验：`10 + 智慧 × 2 + 地图等级 × 5`

### A.4 护道加成
- 单人加成：`5% + (护道者等级 - 被护道者等级) × 1%`
- 总加成上限：`20%`

### A.5 福地灵气
- 每小时消耗：`10 + 地块数量 × 2 + 核心等级 × 5`
- 献祭收益：`物品价值 × 品质乘数 × 等级修正`
