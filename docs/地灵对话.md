# 地灵对话系统

## 1. 系统概述

地灵是福地的人格化 AI 精灵，通过 Spring AI 2.x 的 Function Calling 机制实现自然语言驱动的福地管理。**所有福地等级统一使用 Function Calling 模式**，LLM 自主判断是纯聊天还是调用工具。

### 核心流程

```
玩家: "@地灵 你好"
  ↓
SpiritChatService.chatWithSpirit()
  ├─ 1. 检查是否需要生成福地事件（懒生成，不存库）
  ├─ 2. 读取对话历史（按劫数限制条数）
  ├─ 3. 组装系统 Prompt（MBTI + 形态 + 好感 + 事件 + 历史）
  ├─ 4. 调用 LLM（含 tools 参数）
  │     └─ LLM 自主判断 → 调用 updateEmotion 工具更新情绪
  │     └─ 或纯人格化回复
  ├─ 5. 保存对话到历史表
  └─ 6. 返回回复
```

### 核心概念

- **好感度 (affection)**：地灵对玩家的情感值，连续数值，上限 1000。
- **精力 (energy)**：地灵的活力值，每次服务消耗，随时间自然恢复。上限 = `100 + 劫数 × 20`。
- **形态 (form)**：地灵的外观形象（如小狐妖、春秋蝉等），创建时随机分配，不可更改。
- **MBTI 人格**：地灵的性格类型，创建时玩家选择，不可更改。
- **情绪状态 (emotionState)**：地灵的即时情绪，由 LLM 自主判断，体现 AI 的自主性。

## 3. AI 驱动的情绪系统

### 3.1 设计理念

情绪由 LLM **自主判断**，体现地灵的"个性"和"自主性"，而非简单的数值映射。

### 3.2 情绪状态枚举 (EmotionState)

| 状态 | 说明 |
|------|------|
| HAPPY | 开心 |
| CALM | 平静（默认） |
| ANXIOUS | 焦虑 |
| FATIGUED | 疲惫（精力耗尽） |
| ANGRY | 生气 |
| EXCITED | 兴奋 |
| CURIOUS | 好奇 |
| BORED | 无聊 |
| SAD | 伤心 |

```java

    public String updateEmotion(
        @ToolParam(description = "你选择的情绪状态") String emotion,

    ) {
        // 更新 Spirit.emotionState
        // 记录到 history
        return "情绪已更新为: " + emotion;
    }
    
    @Tool(description = "记录一条地灵的内心活动或感受")
    public String addThought(
        @ToolParam(description = "你的想法或感受") String thought
    ) {
        // 写入 history，不改变情绪

    }

```

### 3.4 情绪对 LLM 回复的影响

情绪状态注入到回复 Prompt 中，影响语气：

```

根据你的情绪调整回复风格：
- HAPPY：热情、主动、多用语气词
- ANGRY：冷淡、简短、带刺
- ANXIOUS：犹豫、不确定
- CURIOUS：追问、感兴趣
- SAD：低落、需要安慰
- ...
```

## 4. 福地事件系统

### 4.1 设计理念

地灵有自己的"生活"，即使玩家不来对话，福地也会发生各种事件，影响地灵的情绪。

### 4.2 事件生成机制

**懒生成，不存库：** 对话时检查时间间隔，如果足够则随机生成事件，直接注入 Prompt。

```java
public class FudiEventGenerator {
    
    public List<FudiEvent> checkAndGenerate(Fudi fudi, Spirit spirit) {
        LocalDateTime lastEventTime = spirit.getLastEventTime();
        long hoursSince = Duration.between(lastEventTime, now()).toHours();
        
        if (hoursSince >= randomInterval()) {
            FudiEvent event = pickRandomEvent(fudi, spirit);
            spirit.setLastEventTime(now());
            return List.of(event);
        }
        return List.of();
    }
    
    private int randomInterval() {
        return 4 + ThreadLocalRandom.current().nextInt(5);  // 4~8 小时
    }
}
```

### 4.3 预定义事件池

| 事件 | 描述 | 建议情绪 | 好感变化 |
|------|------|---------|---------|
| WEATHER_RAIN | 福地下起细雨，灵田里的灵药似乎长势更好 | CALM | 0 |
| WEATHER_WIND | 一阵灵风吹过，地灵的头发被吹乱了 | CALM | 0 |
| STRAY_BEAST | 一只受伤的流浪灵兽出现在福地门口 | SAD | +10 |
| SPIRIT_BUTTERFLY | 一群五彩灵蝶飞入福地 | HAPPY | +5 |
| MYSTERIOUS_VISITOR | 一位神秘老者路过福地，与地灵聊了几句 | CURIOUS | 0 |
| CROP_WITHER | 有一株灵药因为太久没收割而枯萎了 | SAD | -5 |
| BEAST_BIRTH | 兽栏里传来了灵兽幼崽的叫声 | EXCITED | +10 |
| ENERGY_RESTORE | 地灵休息得很好，精力恢复了不少 | HAPPY | 0 |
| MEMORY_OLD_MASTER | 地灵想起了很久以前的主人 | ANXIOUS | 0 |
| MEMORY_FIRST_MEETING | 地灵想起了和你第一次见面的情景 | HAPPY | +5 |
| PRANK_SLIME | 一只史莱姆溜进了福地，把地灵吓了一跳 | ANGRY | -5 |
| PRANK_THEFT | 有小偷想来偷灵药，被地灵赶走了 | ANGRY | -10 |

**事件选择逻辑：** 根据当前状态加权选择。例如精力低时更可能出现 ENERGY_RESTORE。

### 4.4 事件注入 Prompt

```java
// 对话时直接注入，不存库
if (!newEvents.isEmpty()) {
    prompt.append("【最近发生的事情】\n");
    for (FudiEvent event : newEvents) {
        prompt.append("- ").append(event.description).append("\n");
        prompt.append("  （这件事让你感到有点").append(event.suggestedEmotion).append("）\n");
    }
}
```

**注意：** `suggestedEmotion` 只是建议，LLM 可以自主选择不同的情绪。

## 5. 对话历史系统

### 5.1 设计理念

地灵有"记忆"，能记住之前和玩家的对话。记忆容量随劫数增长。

### 5.2 数据表 (xt_spirit_history)

```sql
CREATE TABLE xt_spirit_history (
    id            BIGSERIAL PRIMARY KEY,
    fudi_id       BIGINT NOT NULL,
    role          VARCHAR(16) NOT NULL,  -- "player" / "spirit" / "system" / "event"
    content       TEXT NOT NULL,
    emotion_state VARCHAR(20),           -- 当时的情绪状态
    create_time   TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_spirit_history_fudi_time ON xt_spirit_history (fudi_id, create_time DESC);
```

### 5.3 记忆容量公式

```
maxHistory = 5 + floor(5 × log2(tribulationStage + 1))
```

| 劫数 | 记忆上限 |
|------|---------|
| 0 | 5 条 |
| 1 | 10 条 |
| 3 | 15 条 |
| 7 | 20 条 |
| 15 | 25 条 |

**"一条"的定义：** 一次交互（玩家说 + 地灵回算 1 条），或一个事件记录。

### 5.4 历史注入 Prompt

```java
int maxHistory = 5 + (int) Math.floor(5 * Math.log(fudi.getTribulationStage() + 1));
List<SpiritHistory> history = historyRepository.findRecent(fudiId, maxHistory);

if (!history.isEmpty()) {
    prompt.append("【你的记忆】\n");
    for (SpiritHistory h : history) {
        prompt.append("[").append(h.getCreateTime()).append("] ");
        if ("player".equals(h.getRole())) {
            prompt.append("玩家说：").append(h.getContent()).append("\n");
        } else if ("spirit".equals(h.getRole())) {
            prompt.append("你回复：").append(h.getContent()).append("\n");
        } else if ("event".equals(h.getRole())) {
            prompt.append("发生了：").append(h.getContent()).append("\n");
        }
    }
}
```

## 7. 好感度系统

### 7.1 上限

好感度上限 = **1000**（`affection_max` 字段），不可超过。

### 7.2 获取途径

| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫胜利 | +5 | 每次渡劫成功固定增加（怜悯模式下不加） |
| 赠送地灵所爱礼物 | +10~50 | 物品 tags 命中 `liked_tags`，幅度取决于 qualityMultiplier |
| 赠送中性礼物 | +1~3 | tags 既不命中喜欢也不命中讨厌 |
| 赠送地灵讨厌的礼物 | -5~20 | 物品 tags 命中 `disliked_tags` |
| 天劫失败 | -clearCount | 按被毁地块数扣除 |
| 冒犯言论 | -1~5 | LLM 通过 `reportPlayerOffense` 上报 |
| 福地事件 | ±5~10 | 事件预定义的好感变化 |

### 7.3 作用

| 作用 | 公式 |
|------|------|
| 精力消耗减免 | `实际消耗 = 基础消耗 × (1 - min(0.5, affection / 1000))` |
| 灵兽进化成功率加成 | `min(15, affection / 7)` |
| 灵兽品质突破成功率加成 | `min(20, affection / 5)` |
| LLM 对话语气 | Prompt 注入好感度描述：≥800 亲密 / 600 喜欢 / 400 平和 / 200 疏远 / 0 冷淡 |
| 渡劫怜悯 | `affection ≥ 800` 时触发（见第 10 节） |

## 8. 精力系统

### 8.1 上限公式

```
精力上限 = 100 + 劫数 × 20
```

### 8.2 恢复机制（懒更新）

- **触发时机**：任何使用地灵的操作（`getFudiByUserId`、`getFudiStatus` 等都会触发 `restoreEnergy()`）
- **恢复量**：`精力上限 × 5% × 距上次使用小时数`
- **上限**：最多恢复到精力上限
- **特点**：约 20 小时从 0 回满；比例恢复使高劫数玩家单位时间恢复绝对值更高

### 8.3 消耗范围

| 操作 | 基础消耗 | 方法 |
|------|---------|------|
| LLM 对话 | 5 | `SpiritChatService.chatWithSpirit()` |
| 种植 | 3 | `FudiService.plantCrop()` |
| 收获（单次） | 5 | `FudiService.harvestCrop()` |
| 批量收获 | 5/个 | `FudiService.harvestAllCrops()`（首个 5 + 额外 N-1×5） |
| 建造 | 3 | `FudiService.buildCell()` |
| 拆除 | 3 | `FudiService.removeCell()` |
| 孵化 | 10 | `FudiService.hatchBeast()` |
| 进化/突破 | 8 | `FudiService.evolveBeast()` |
| 收取产出 | 3 | `FudiService.collectProduce()` / `collectAllProduce()` |

### 8.4 实现方式

- **FudiService 内部方法**：调用 `consumeSpiritEnergy(fudi, baseCost)` 私有辅助方法
- **跨服务调用**（如 SpiritChatService）：通过 `@ConsumeSpiritEnergy(N)` 注解 + `SpiritEnergyAspect` AOP 切面

**SpiritEnergyAspect 执行流程：**
1. 从方法参数提取 `userId`（第一个 Long 类型参数）
2. 查询 Fudi
3. 懒恢复精力
4. 好感减免计算实际消耗
5. 扣除精力，精力 ≤ 0 自动触发 `FATIGUED` 情绪
6. 更新 `lastEnergyUpdate`，保存 Fudi

## 9. 地灵形态系统

### 9.1 数据模型

形态数据存于 `xt_spirit_form` 表：

```sql
CREATE TABLE xt_spirit_form (
    id            SERIAL PRIMARY KEY,
    name          VARCHAR(50) NOT NULL UNIQUE,
    description   TEXT NOT NULL,
    liked_tags    JSONB NOT NULL DEFAULT '[]',
    disliked_tags JSONB NOT NULL DEFAULT '[]'
);
```

Java 实体 `SpiritForm` 中 `likedTags` / `dislikedTags` 使用 `Set<String>` 存储。
地灵实例数据存于 `xt_spirit` 表，与福地 1:1 绑定。

### 9.2 分配方式

- 创建福地时从 `xt_spirit_form` 表中 **随机抽取**，不可更改
- 从形态的 `liked_tags` 池中随机抽取 **3~5** 个，作为该地灵的喜好
- 从 `disliked_tags` 池中随机抽取 **2~3** 个，作为该地灵的厌恶
- 结果存入 `xt_spirit` 表的 `liked_tags` / `disliked_tags` 列中，运行时直接读取

### 9.3 内置形态

| 形态名 | 描述 | 喜爱的 tag 示例 |
|--------|------|----------------|
| 小狐妖 | 傲娇九尾幼狐，喜欢蹭人 | wine, art, book, prestige, shiny |
| 春秋蝉 | 晶莹金蝉，看尽时光流转 | time, medicinal, art, book, sacred |
| 石中鲤 | 灵石中游弋的锦鲤，带来好运 | gem, ore, shiny, sacred, water |
| 剑灵残片 | 上古神剑碎片，性格冷傲 | weapon, forge, metal, sharp, sacred |
| 千年何首乌 | 成了精的何首乌，喜欢引经据典 | medicinal, herb, book, wood, time |
| 乌云踏雪 | 黑色麒麟幼崽，活泼好动 | beast, sacred, shiny, food, meat |
| 酒葫芦 | 成了精的紫金酒葫芦，说话带三分醉意 | wine, food, sweet, fruit, art |

### 9.4 送礼判定

玩家通过 `#地灵送礼 [物品名]` 或 LLM 调用 `giveGift` 工具送礼。从 `spiritConfig` 获取喜好进行匹配：

| 匹配 | 好感变化 | 反应 |
|------|---------|------|
| tags 命中 liked | +10~50（qualityMultiplier 增幅） | 开心 |
| tags 命中 disliked | -5~20 | 嫌弃 |
| 均未命中 | +1~3 | 平淡 |

## 10. 天劫系统

### 10.1 触发机制

- **主动渡劫**：玩家可随时发起渡劫
- **强制渡劫**：距上次劫后 7 天自动触发
- **渡劫失败**：劫数不增加，按差额比例清空地块

### 10.2 渡劫怜悯机制

**触发条件：**
```
affection ≥ 800
AND defense ≥ attack × 80%
AND defense < attack（确实会失败）
```

**效果：**

| 项目 | 变化 |
|------|------|
| 天劫结果 | 视为胜利（劫数+1、连胜+1、灵气上限增长） |
| 好感度 | 不加好感（正常渡劫胜利给 +5，怜悯模式不给） |
| 精力 | 归零 |
| 情绪 | → FATIGUED |

## 11. LLM 冒犯上报

`reportPlayerOffense` 工具让 LLM 在自然对话中自主判断玩家是否说了冒犯的话：

```java
@Tool(description = "当玩家说了不礼貌、冒犯、或让地灵不开心的话时调用此工具")
ReportOffenseResponse reportPlayerOffense(
    @ToolParam(description = "玩家说了什么") String reason,
    @ToolParam(description = "冒犯程度 1~5") int severity
)
```

- 由 LLM 自主判断是否触发，无需硬编码关键词
- severity 直接影响好感扣除量（-severity）

【地块状态】

| 600~799 | 非常喜欢，对你温柔体贴 |
| 400~599 | 有好感，愿意主动帮忙 |
| 200~399 | 态度平和，礼貌相待 |
| <200 | 态度冷淡，不愿多说话 |

## 13. 关键技术决策

- **AI 自主情绪**：情绪由 LLM 通过 `updateEmotion` 工具自主判断，体现地灵的个性和自主性
- **懒生成事件**：福地事件在对话时懒生成，不持久化，直接注入 Prompt
- **对话历史持久化**：对话历史存入 `xt_spirit_history` 表，容量按对数公式随劫数增长
- **Lazy 懒更新**：精力恢复不依赖定时任务，在任何地灵相关操作时按 `(当前时间 - lastEnergyUpdate)` 计算增量后一气更新
- **AOP + 辅助方法混合**：FudiService 内部方法直接调用 `consumeSpiritEnergy()` 辅助方法，跨服务调用（如 SpiritChatService）走 `@ConsumeSpiritEnergy` AOP 切面
- **好感度 capping**：通过 `Fudi.addAffection(amount)` 统一处理，限制在 `[0, affectionMax]` 范围
- **形态喜好独立随机**：同一种形态的不同地灵因随机抽取而拥有不同喜好组合，增加个性化
- **零 LLM 创建**：福地创建时形态分配和喜好生成是纯随机 Java 逻辑，不依赖 LLM，创建瞬间完成
