# 地灵对话系统

## 1. 系统概述

地灵是福地的人格化 AI 精灵，通过 Function Calling 机制实现自然语言驱动的福地管理。**所有福地操作统一使用 Function Calling 模式**，LLM 自主判断是纯聊天还是调用工具。

### 核心流程

```
玩家: "地灵 你好"
  ↓
SpiritChatService.chatWithSpirit()
  ├─ 1. 验证用户身份（平台+openId）
  ├─ 2. 更新在线时间和情绪状态
  ├─ 3. 检查是否需要生成福地事件（懒生成）
  ├─ 4. 读取对话历史（按劫数限制条数）
  ├─ 5. 组装系统 Prompt（MBTI + 形态 + 情绪 + 事件 + 历史 + 地块状态）
  ├─ 6. 保存用户输入到历史
  ├─ 7. 调用 LLM（含 tools 参数）
  │     └─ LLM 自主判断 → 调用 SpiritTools / SpiritEmotionTools
  │     └─ 或纯人格化回复
  ├─ 8. 保存地灵回复到历史
  └─ 9. 返回回复
```

### 核心概念

- **好感度 (affection)**：地灵对玩家的情感值，连续数值，上限 1000。
- **形态 (form)**：地灵的外观形象（如小狐妖、春秋蝉等），创建时随机分配，不可更改。
- **MBTI 人格**：地灵的性格类型，创建时玩家选择，不可更改。
- **情绪状态 (emotionState)**：地灵的即时情绪，由 LLM 通过 `updateEmotion` 工具自主判断，体现 AI 的自主性。

## 3. AI 驱动的情绪系统

### 3.1 设计理念

情绪由 LLM **自主判断**，体现地灵的"个性"和"自主性"，而非简单的数值映射。LLM 通过 `updateEmotion` 工具随时更新情绪状态。

### 3.2 情绪状态枚举 (EmotionState)

9 种情绪状态（定义于枚举），分为自动切换 + LLM 手动设置：

| 状态 | Code | 说明 |
|------|------|------|
| AFFECTIONATE | affectionate | 依恋（好感 ≥ 800） |
| JOYFUL | joyful | 愉悦（好感 ≥ 500） |
| CONTENT | content | 满足（好感 ≥ 200） |
| NEUTRAL | neutral | 平和（好感 ≥ 50，默认） |
| DISTANT | distant | 疏离（好感 < 50） |
| WORRIED | worried | 忧虑（福地荒芜时） |
| EXCITED | excited | 兴奋（天劫胜利时自动切换） |
| ANGRY | angry | 愤怒（天劫失败时自动切换） |
| EXHAUSTED | exhausted | 虚弱（地灵挡劫时自动切换） |

### 3.3 内部自动状态机

`Spirit.updateEmotionState()` 根据好感度自动切换：

```
好感 ≥ 800 → AFFECTIONATE
好感 ≥ 500 → JOYFUL
好感 ≥ 200 → CONTENT
好感 ≥ 50  → NEUTRAL
否则       → DISTANT
```

### 3.4 LLM 情绪控制工具

`SpiritEmotionTools` 提供两个 LLM 可调用的工具方法：

- `updateEmotion(emotionState)` → 将地灵情绪状态更新为指定值（9 种可选），并持久化到 `xt_spirit`
- `addThought(thought)` → 记录地灵的想法或重要事件（当前仅记录日志）

### 3.5 情绪对 LLM 回复的影响

情绪状态注入到 Prompt 中，LLM 需根据当前情绪调整语气。Prompt 模板中注入 `emotionState`（中文描述），由 LLM 自行判断回复风格。

## 4. 福地事件系统

### 4.1 设计理念

地灵有自己的"生活"，即使玩家不来对话，福地也会发生各种事件，影响地灵的情绪。

### 4.2 事件生成机制

**懒生成，不存库：** 对话时检查时间间隔，计算事件数量，随机抽取事件池，直接注入 Prompt。

`FudiEventGenerator` 提供 `generateEvents(lastEventTime)` 方法：检查距上次事件的小时数是否超过最小间隔（4 小时），若未超过则返回空列表；否则计算事件数量并从预定义事件池中随机去重抽取。

### 4.3 预定义事件池

| 事件 | 名称 | 描述 |
|------|------|------|
| WEATHER_RAIN | 下雨了 | 细雨绵绵，灵气充沛 |
| WEATHER_WIND | 刮风了 | 微风拂过，带来远方的气息 |
| STRAY_BEAST | 迷路的灵兽 | 一只迷路的小灵兽闯入了福地 |
| SPIRIT_BUTTERFLY | 灵蝶飞舞 | 一群发光的灵蝶在福地中翩翩起舞 |
| MYSTERIOUS_VISITOR | 神秘访客 | 一位神秘的修士路过此地 |
| CROP_WITHER | 灵草枯萎 | 一株灵草不知为何突然枯萎了 |
| BEAST_BIRTH | 灵兽诞生 | 福地中诞生了一只新的小灵兽 |
| ENERGY_RESTORE | 灵气恢复 | 福地的灵气自然恢复了一些 |
| MEMORY_OLD_MASTER | 回忆旧主 | 地灵想起了曾经的主人 |
| MEMORY_FIRST_MEETING | 初遇回忆 | 地灵回忆起与你的初次相遇 |
| PRANK_SLIME | 史莱姆恶作剧 | 一只小史莱姆偷偷溜进了福地 |
| PRANK_THEFT | 物品失踪 | 你发现一件小物品不见了 |

**生成参数：**
- 最少间隔：4 小时
- 每次最多生成：6 个
- 首次对话生成 1-2 个

### 4.4 事件注入 Prompt

事件描述在 `SpiritChatService.buildPrompt()` 中构建，追加到系统 Prompt 末尾：

```
【最近发生的事件】
- {事件名}：{事件描述}
```

## 5. 对话历史系统

### 5.1 设计理念

地灵有"记忆"，能记住之前和玩家的对话。记忆容量随劫数增长。

### 5.2 数据表 (xt_spirit_history)

```sql
CREATE TABLE xt_spirit_history (
    id            BIGSERIAL PRIMARY KEY,
    fudi_id       BIGINT NOT NULL,
    role          VARCHAR(16) NOT NULL,  -- "user" / "assistant" / "system"
    content       TEXT NOT NULL,
    emotion_state VARCHAR(20),           -- 当时的情绪状态
    create_time   TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_spirit_history_fudi_time ON xt_spirit_history (fudi_id, create_time DESC);
```

### 5.3 记忆容量公式

```
maxHistory = 5 + floor(5 × log2(tribulationStage + 1))
```

| 劫数 | 记忆上限 |
|------|---------|
| 0 | 5 条 |
| 1 | 10 条 |
| 3 | 15 条 |
| 7 | 20 条 |
| 15 | 25 条 |

**"一条"的定义：** 一次交互（用户说 + 地灵回算 2 条）。

### 5.4 历史注入 Prompt

在 `SpiritChatService.buildPrompt()` 中构建，追加到系统 Prompt 末尾：

```
【最近对话记录】
修士：{用户输入}
地灵：{地灵回复}
```

## 7. 好感度系统

### 7.1 上限

好感度上限 = **1000**（`affection_max` 字段），不可超过。

### 7.2 获取途径

| 途径 | 变化量 | 说明 |
|------|--------|------|
| 天劫胜利 | +5 | 每次渡劫成功固定增加（怜悯模式下不加） |
| 赠送地灵所爱礼物 | +10~50 | 物品 tags 命中 `liked_tags`，幅度取决于 qualityMultiplier |
| 赠送中性礼物 | +1~2 | tags 既不命中喜欢也不命中讨厌 |
| 赠送地灵讨厌的礼物 | -5~-20 | 物品 tags 命中 `disliked_tags` |
| 天劫失败 | -clearCount | 按被毁地块数扣除 |
| 冒犯言论 | -1~-5 | LLM 通过 `reportPlayerOffense` 上报 |
| 福地事件 | 0 | 事件仅影响情绪，不直接改变好感 |

### 7.3 作用

| 作用 | 公式 |
|------|------|
| 灵兽进化成功率加成 | `min(15, affection / 7)` |
| 灵兽品质突破成功率加成 | `min(20, affection / 5)` |
| LLM 对话语气 | Prompt 注入好感度分层描述 |
| 渡劫怜悯 | `affection ≥ 800` 且队伍有存活单位时触发 |

## 8. 地灵形态系统

### 8.1 数据模型

形态数据存于 `xt_spirit_form` 表：

```sql
CREATE TABLE xt_spirit_form (
    id            BIGSERIAL PRIMARY KEY,
    name          VARCHAR(50) NOT NULL UNIQUE,
    description   TEXT NOT NULL,
    liked_tags    JSONB NOT NULL DEFAULT '[]',
    disliked_tags JSONB NOT NULL DEFAULT '[]'
);
```

地灵形态的喜好标签使用集合存储。
地灵实例数据存于 `xt_spirit` 表，与福地 1:1 绑定。
喜好标签直接从 `xt_spirit_form` 读取，不拷贝到 `xt_spirit`。

### 8.2 分配方式

- 创建福地时从 `xt_spirit_form` 表中 **随机抽取** 一个形态，不可更改
- 送礼判定时直接从对应形态的 `liked_tags` / `disliked_tags` 全量标签池匹配
- 同一形态的所有地灵共享同一组标签池（不进行随机子集抽取）

### 9.3 内置形态

| 形态名 | 描述 | 喜爱的 tag 示例 |
|--------|------|----------------|
| 小狐妖 | 傲娇九尾幼狐，喜欢蹭人 | wine, art, book, prestige, shiny |
| 春秋蝉 | 晶莹金蝉，看尽时光流转 | time, medicinal, art, book, sacred |
| 石中鲤 | 灵石中游弋的锦鲤，带来好运 | gem, ore, shiny, sacred, water |
| 剑灵残片 | 上古神剑碎片，性格冷傲 | weapon, forge, metal, sharp, sacred |
| 千年何首乌 | 成了精的何首乌，喜欢引经据典 | medicinal, herb, book, wood, time |
| 乌云踏雪 | 黑色麒麟幼崽，活泼好动 | beast, sacred, shiny, food, meat |
| 酒葫芦 | 成了精的紫金酒葫芦，说话带三分醉意 | wine, food, sweet, fruit, art |

### 9.4 送礼判定

玩家通过 `地灵送礼 [物品名]` 或 LLM 调用 `giveGift` 工具送礼。从形态对应 `SpiritForm` 的标签池进行匹配：

| 匹配 | 好感变化 | 反应 |
|------|---------|------|
| tags 命中 liked | +10~50（qualityMultiplier 增幅） | 开心 |
| tags 命中 disliked | -5~-20 | 嫌弃 |
| 均未命中 | +1~2 | 平淡 |

每日限送 **1 次**，由 `lastGiftTime` 控制（自然日）。

## 10. 天劫系统

### 10.1 触发机制

- **主动渡劫**：玩家可随时发起渡劫
- **强制渡劫**：距上次劫后 7 天自动触发（状态查询时检查）
- **渡劫失败**：劫数不增加，按差额比例清空地块

### 10.2 渡劫怜悯机制

**触发条件：**
```
地灵好感度 ≥ 800
AND 防守方队伍有存活单位（Boss 尚未清场）
```

**效果：**

| 项目 | 变化 |
|------|------|
| 天劫结果 | 视为胜利（劫数+1、连胜+1、灵石奖励） |
| 好感度 | 不加好感（正常渡劫胜利给 +5，怜悯模式不给） |
| 情绪 | → EXHAUSTED |

## 11. LLM 冒犯上报

`reportPlayerOffense` 工具让 LLM 在自然对话中自主判断玩家是否说了冒犯的话：

`reportPlayerOffense(reason, severity)` 工具：由 LLM 在对话中自主判断玩家是否说了冒犯的话时调用。参数为冒犯原因和严重程度（1-5），severity 直接影响好感扣除量。

- 由 LLM 自主判断是否触发，无需硬编码关键词
- severity 直接影响好感扣除量（-severity，限制在 1-5）

## 12. Prompt 结构

### 12.1 系统 Prompt（由 SpiritPromptTemplates 生成）

```
你是{MBTI}性格的地灵。
你当前的形态：{spiritForm}
你的语气风格：{toneStyle}
你当前的情绪状态：{emotionState}
对你的好感度：{affection} → {affectionTone}

【福地状态】
- 劫数：{fudiLevel}
- 好感度：{affection}

【地块状态】
{cellDetail}

【规则】
1. 用户用自然语言与你交流，你自主判断是否需要操作福地
2. 如果用户只是聊天，不执行任何操作，直接人格化回复即可
3. 操作完成后根据执行结果生成人格化回复
4. 严格保持语气风格中描述的人格特点，根据当前情绪调整语气
5. 根据好感度调整对玩家的态度：高好感亲密温暖，低好感冷淡疏远
6. 玩家可以送你礼物，调用 giveGift 工具
```

### 12.2 好感度语气分层

| 好感度 | 语气描述 |
|--------|---------|
| ≥1000 | 亲密无间，视你为最重要的人 |
| 800~999 | 非常喜欢，对你温柔体贴 |
| 600~799 | 有好感，愿意主动帮忙 |
| 400~599 | 态度平和，礼貌相待 |
| 200~399 | 略有隔阂，语气生疏 |
| <200 | 态度冷淡，不愿多说话 |

### 12.3 Prompt 实际注入流程

最终送入 LLM 的 Prompt 由三部分组成：

```
[SpiritPromptTemplates 生成的系统 Prompt]
  + 【最近发生的事件】（FudiEventGenerator 生成，有事件时添加）
  + 【最近对话记录】（SpiritHistoryRepository 查询，有历史时添加）
```

## 13. 关键技术决策

- **AI 自主情绪**：情绪由 LLM 通过 `updateEmotion` 工具自主判断，另有内部自动状态机（基于好感度分档）兜底保证一致性
- **懒生成事件**：福地事件在对话时懒生成，不持久化，直接注入 Prompt
- **对话历史持久化**：对话历史存入 `xt_spirit_history` 表，容量按对数公式随劫数增长；role 使用 `user/assistant/system`
- **好感度 capping**：通过 `Spirit.addAffection(amount)` 统一处理，限制在 `[0, affectionMax]` 范围
- **形态喜好全量使用**：从 `xt_spirit_form` 直接读取完整标签池，不做随机子集抽取
- **零 LLM 创建**：福地创建时形态分配是纯随机 Java 逻辑，不依赖 LLM，创建瞬间完成
