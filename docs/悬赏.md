# 悬赏系统

## 1. 概述

悬赏是玩家在地图节点接取的限时任务。接取后需等待指定时长，提交后根据预确定的奖励池发放物品/灵石/灵兽卵，期间可能触发 D20 旅行事件。

核心特性：
- **种子预确定奖励**: 接取时即确定奖励内容并写入 DB，提交时直接发放，不重算
- **同日幂等**: 同一用户同一天接取同一悬赏，奖励相同
- **LLM 美化**: 完成时调用 AI 生成叙事描述

## 2. 数据库设计

### 2.1 xt_bounty — 悬赏配置表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `map_id` | BIGINT | 所属地图 ID |
| `name` | VARCHAR(100) | 悬赏名称 |
| `description` | TEXT | 悬赏描述 |
| `duration_minutes` | INTEGER | 悬赏耗时（分钟） |
| `rewards` | JSONB | 奖励池（加权随机） |
| `require_level` | INTEGER | 最低接取等级 |
| `event_weight` | INTEGER | 旅行事件触发权重 |

**rewards JSONB 格式**:

```json
[
  {"type": "rare_item",     "weight": 50, "count": 3},
  {"type": "spirit_stones", "weight": 30, "amount": 100},
  {"type": "beast_egg",     "weight": 20}
]
```

- `type`: 奖励类型，支持 `rare_item` / `spirit_stones` / `beast_egg`
- `weight`: 加权随机权重
- `count`: (rare_item) 产出稀有物品数量
- `amount`: (spirit_stones) 灵石数量

### 2.2 xt_user_bounty — 用户悬赏记录表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | 用户 ID |
| `bounty_id` | BIGINT | 悬赏 ID |
| `bounty_name` | VARCHAR(100) | 悬赏名称（快照） |
| `start_time` | TIMESTAMP | 接取时间 |
| `duration_minutes` | INTEGER | 悬赏耗时（分钟） |
| `rewards` | JSONB | 预确定的奖励物品 |
| `status` | VARCHAR(16) | `active` / `completed` / `abandoned` |

- `rewards` 在接取时由种子随机计算后写入，提交时直接读取发放
- `status` 状态机: `active` → `completed` / `abandoned`

## 3. 奖励体系

### 3.1 奖励类型

| 类型 | 来源 | 说明 |
|---|---|---|
| `rare_item` | 地图特产 (rarity=rare) | 加权随机选取 N 个稀有物品 |
| `spirit_stones` | 固定数值 | 直接发放灵石到用户账户 |
| `beast_egg` | ItemTemplate (BEAST_EGG) | 随机选取一颗灵兽卵 |

### 3.2 稀有物品选取 (rare_item)

从当前地图 `xt_map_node.specialties` 中筛选 `rarity = "rare"` 的条目，按 `weight` 加权随机选取 `count` 件。

特产 JSONB 格式:

```json
[
  {"name": "铁矿石", "weight": 50, "templateId": 1, "rarity": "common"},
  {"name": "灵草",   "weight": 10, "templateId": 3, "rarity": "rare"}
]
```

- `rarity: "common"` → 历练掉落
- `rarity: "rare"` → 悬赏掉落

## 4. 奖励预确定机制 (种子随机)

接取悬赏时，用确定性的种子构建 `java.util.Random`，预计算奖励内容并写入 `xt_user_bounty.rewards`。

```
seed = userId * 31 + LocalDate.now().toEpochDay()
rng = new Random(seed)
```

**特性**:
- 同用户同日接取同一悬赏 → 奖励相同（幂等）
- 跨日 seed 变化 → 不同日期奖励不同
- 奖励写入 DB 后不再变动，提交时直接读取

### 4.1 预确定流程

```
1. 构建 rng(seed)
2. 加权随机从 bounty.rewards 中选取一条顶层奖励类型 (rare_item/spirit_stones/beast_egg)
3. 根据类型展开具体物品:
   - rare_item  → 从 mapNode.specialties(rarity=rare) 加权选取 N 件
   - spirit_stones → 生成元信息 {_rewardType: "spirit_stones", amount: N}
   - beast_egg  → 从 ItemTemplate(BEAST_EGG) 随机选 1 个
4. 写入 xt_user_bounty.rewards (JSONB)
```

### 4.2 内部标记

预确定的 rewards 内部使用 `_rewardType` 标记字段区分奖励类型，读取时过滤掉:

```json
[
  {"_rewardType": "spirit_stones", "amount": 100},
  {"name": "灵草", "templateId": 3, "quantity": 1, "_rewardType": "beast_egg"}
]
```

## 5. 业务流程

### 5.1 操作流程

```
#悬赏列表
  → 查询当前地图的悬赏 (过滤等级)
  → 显示 ID / 名称 / 描述 / 要求等级

#悬赏接取 [ID]
  → 验证: 用户 IDLE / 悬赏存在 / 地图匹配 / 等级满足
  → 种子预确定奖励 → 写入 xt_user_bounty (status=active)
  → 用户 status → BOUNTY

  ... 等待 duration_minutes 分钟 ...

#悬赏提交
  → 验证: 用户 BOUNTY / 存在 active 记录 / 时间 ≥ duration_minutes
  → 读取预存 rewards → 发放物品 + 灵石
  → D20 事件判定 (D20 ≤ 10 触发)
  → LLM 美化叙事
  → 标记 completed → 用户 status → IDLE

#悬赏放弃
  → 验证: 用户 BOUNTY / 存在 active 记录
  → 标记 abandoned (无任何产出) → 用户 status → IDLE
```

### 5.2 状态约束

| 操作 | 要求用户状态 | 完成后状态 |
|---|---|---|
| 悬赏列表 | IDLE | — |
| 悬赏接取 | IDLE | BOUNTY |
| 悬赏提交 | BOUNTY | IDLE |
| 悬赏放弃 | BOUNTY | IDLE |

### 5.3 提交时时间检查

```java
long minutesElapsed = Duration.between(startTime, now()).toMinutes();
if (minutesElapsed < durationMinutes) {
    long remaining = durationMinutes - minutesElapsed;
    throw new IllegalArgumentException("悬赏还需 " + remaining + " 分钟");
}
```

## 6. 旅行事件

悬赏完成时触发 D20 事件判定，与旅行系统共享同一事件机制。

- D20 ≤ 10 → 触发事件（从 `mapNode.travel_events` 加权随机选取）
- D20 > 10 → 安全通过，无事件

| 事件 | 效果 |
|---|---|
| AMBUSH | 受到 10-29 点伤害 |
| FIND_TREASURE | 获得 50-149 铜币 |
| WEATHER | 纯叙事（"遭遇毒雾天气..."） |
| SAFE_PASSAGE | 无事发生 |

## 7. 代码结构

### 7.1 目录树

```

│   └── UserBountyRepository.java   # 用户悬赏记录仓储接口
└── vo/

│   ├── BountyMapper.java           # MyBatis-Flex Mapper
│   └── UserBountyMapper.java
└── repository/

├── command/

└── onebotv11/
    └── MapHandle.java              # QQ 平台监听器
```

### 7.2 核心类

**Bounty** (`domain/bounty/entity/Bounty.java`):

悬赏配置实体，映射 `xt_bounty` 表。包含 `mapId`, `name`, `description`, `durationMinutes`, `rewards` (JSONB), `requireLevel`, `eventWeight` 等字段。使用 MyBatis-Flex 注解 `@Table("xt_bounty")` 映射，`rewards` 字段通过 `PgJsonbTypeHandler` 处理 JSONB。

**UserBounty** (`domain/bounty/entity/UserBounty.java`):

用户悬赏记录实体，映射 `xt_user_bounty` 表。包含 `userId`, `bountyId`, `bountyName` (快照), `startTime`, `durationMinutes`, `rewards` (预确定奖励 JSONB), `status` 等字段。提供 `isActive()` / `isCompleted()` 便捷方法。

**BountyRepository** (`domain/bounty/repository/BountyRepository.java`):

```java
Optional<Bounty> findById(Long id);
List<Bounty> findByMapId(Long mapId);
```

**UserBountyRepository** (`domain/bounty/repository/UserBountyRepository.java`):

```java
Optional<UserBounty> findActiveByUserId(Long userId);  // status=active
void save(UserBounty userBounty);
void update(UserBounty userBounty);
```

**BountyService** (`service/BountyService.java`):

遵循项目 Service 双层 API 模式:

| 方法 (Public) | 方法 (Internal) | 说明 |
|---|---|---|
| `listBounties(platform, openId)` | `listBounties(userId)` | 查看悬赏列表 |
| `startBounty(platform, openId, bountyId)` | `startBounty(userId, bountyId)` | 接取悬赏 |
| `completeBounty(platform, openId)` | `completeBounty(userId)` | 提交悬赏 |
| `abandonBounty(platform, openId)` | `abandonBounty(userId)` | 放弃悬赏 |

Internal 方法实现核心逻辑，Public 方法通过 `AuthenticationService` 进行状态校验后委托给 Internal 方法，返回 `ServiceResult<T>`。

关键 private 方法:

| 方法 | 说明 |
|---|---|
| `determineRewards()` | 种子随机预计算奖励 |
| `findRareItems()` | 从地图特产选取稀有物品 |
| `addRewardsToInventory()` | 发放物品到背包 |
| `buildRewardDescription()` | 构建奖励文本描述 |
| `resolveEvent()` | D20 旅行事件判定 |
| `processEvent()` | 事件效果处理 |
| `beautifyBountyCompletion()` | LLM 美化叙事 (调用 ExplorationDescriptionFunction) |

## 9. 数据流图

```
接取悬赏:
  Player → MapHandle (QQ监听)
        → MapCommandHandler.handleStartBounty()
        → BountyService.startBounty()
        → determineRewards() (种子随机)
        → userBountyRepository.save() (写入预确定奖励)
        → user.setStatus(BOUNTY)
        → 返回确认文本

提交悬赏:
  Player → MapHandle (QQ监听)
        → MapCommandHandler.handleCompleteBounty()
        → BountyService.completeBounty()
        → 时间检查 (minutesElapsed ≥ durationMinutes)
        → 读取 xt_user_bounty.rewards
        → addRewardsToInventory() (发放物品)
        → user.setSpiritStones() (发放灵石)
        → resolveEvent() (D20 事件)
        → beautifyBountyCompletion() (LLM 美化)
        → record.setStatus(completed) + user.setStatus(IDLE)
        → 返回 BountyRewardVO
```
