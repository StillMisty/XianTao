## 3. 地块布局 (grid_layout JSONB)

### 3.1 数据结构

```json
{
  "cells": [
    { "cell_id": 1, "type": "empty" },
    { "cell_id": 2, "type": "farm", "cell_level": 1, "crop_id": 101, ... },
    { "cell_id": 3, "type": "pen", "cell_level": 1, "beast_id": 1, ... },
    { "cell_id": 4, "type": "empty" },
    { "cell_id": 5, "type": "empty" }
  ]
}
```

- `cell_id` 从 1 开始连续递增，唯一标识地块
- `type: "empty"` 的空地块只有 `cell_id` 和 `type` 两个字段
- 建造/拆除操作不改变 cell_id，empty 保留槽位

### 3.2 地块类型 (CellType)

| 类型 | 作用 |
|---|---|
| FARM (灵田) | 种植灵药 |
| PEN (兽栏) | 孵化/饲养灵兽，详见 [灵兽.md](灵兽.md) |

### 3.3 地块等级

灵田和兽栏均有独立等级 (Lv1~5)：

| 等级 | 升级消耗 | 累积 | 加速系数 |
|---|---|---|---|
| Lv1 | — | 0 | 1.00× |
| Lv2 | 200 | 200 | 1.15× |
| Lv3 | 400 | 600 | 1.30× |
| Lv4 | 800 | 1400 | 1.45× |
| Lv5 | 1600 | 3000 | 1.60× |

加速系数公式：`加速系数 = 1 + (地块等级 - 最低需求等级) × 0.15`，表格为预计算结果。

### 3.4 地块扩建

- 初始地块数：**3**（cell_id = 1, 2, 3）
- 增长公式：`maxCells = 3 + floor(tribulationStage / 3)`
- 扩建方式：**自动解锁**，无消耗
- 触发时机：任何操作调用 `getFudiByUserId()` 时懒加载

## 4. 种植系统 (灵田)

**定位**：低门槛、快周转、活跃型。

### 4.1 核心公式

```
种植消耗 = 种子 × 1 + tier × 5 灵石
生长时间 = 基础生长时间 / 等级加速系数
产量 = (1~3) + 劫数/5
收获消耗 = 地灵精力 5 点
```

### 4.2 多季作物

高 tier 种子产出 `1 + tier/3` 次，地块不消失，重置生长进度。

### 4.3 枯萎机制

普通作物：成熟后超 `2×基础生长时间` 未收 → 枯萎消失。
**多季作物免除枯萎检查。**

## 5. 天劫系统

### 5.1 周期

每 7 天触发一次（懒求值）。

### 5.2 攻防公式

```
天劫攻击力 = 玩家等级 × 80 + 劫数 × 200
福地总防 = Σ(出战灵兽战力 × 护主乘数) + 玩家STR × 10 + 劫数 × 50
```

- 护主乘数：拥有护主特性 ×1.5，无特性 ×1.0
- 仅出战状态的灵兽提供防御加成，详见 [灵兽.md](灵兽.md)
- 无出战灵兽时此项为 0

### 5.3 天劫胜利

- 连胜 +1、好感 +5、灵石奖励（+ 连胜 × 100）、劫数 +1
- 情绪 → EXCITED

### 5.4 天劫失败

- 连胜归零
- 按差额比例随机清空地块（重置为 empty，保留 cell_id）
- 好感降低
- 情绪 → ANGRY

|---|---|

| `#福地拆除 [编号]` | 拆除地块 |
| `#福地升级 [编号]` | 升级地块等级 |
| `地灵 [内容]` | 与地灵自然语言交互 |
