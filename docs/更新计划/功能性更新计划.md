# 功能性更新计划

> 本文档记录 XianTao（仙道）系统从游戏性角度的现存问题及未来迭代方向。

---

## 一、现存问题

### 1.1 核心缺陷

| 问题 | 说明 |
|------|------|
| **旅途叙事未接入历练** | `ExplorationDescriptionFunction` 已实现但仅悬赏中部分使用，常规历练无 flavor |

### 1.2 长期缺失

| 缺失系统 | 说明 |
|----------|------|
| 副本系统 | `BattleScene.DUNGEON` 枚举存在但零实现 |
| 打造系统 | 只有炼药，无装备打造、符文等 |
| 装备锻造系统是空壳 | `forgeLevel` 字段存在但无锻造/强化服务，装备到手即终态 |
| 境界无命名 | 只有数字层数，缺代入感和阶段性成就感 |

---

## 二、迭代路线

### 第一阶段：玩法深度

基于当前架构直接扩展，无需改表或新增基础设施。

| 功能 | 描述 | 架构适配 |
|------|------|----------|
| **顿悟/悟道事件** | 历练中低概率触发，悟性加成或领悟技能 | 扩展 `TrainingCombatLogic` 事件模式 |
| **丹炉品质** | 不同丹炉影响炼药成功率和品质上限 | furnace 修饰器层包装 `PillRefiningService` |

### 第二阶段：社交与经济

需新增 domain 包或关联表。

| 功能 | 描述 | 架构需求 |
|------|------|----------|
| **交易市场** | 玩家间摆摊/拍卖 | `market_listing` 关联表 | **已设计** → `docs/交易系统设计.md` |
| **宗门系统** | 驻地、任务、宗门战 | `domain/sect/` 包 |
| **师徒系统** | 师傅指导突破加成（护道泛化） | 扩展 `DaoProtection` |
| **双修系统** | 双人修炼获额外效率 | `dual_cultivation` 状态 |
| **世界 Boss** | 全服定时妖兽潮 | 复用 `MonsterTemplate` + `CombatEngine` |

### 第三阶段：内容扩展

需新生命周期管理和实体。

| 功能 | 描述 | 架构需求 |
|------|------|----------|
| **秘境系统** | 限时多层副本，稀有掉落 | `DungeonInstance` + 副本生命周期 | **已设计** → `docs/秘境系统设计.md` |
| **轮回/转生** | 达境界后转生，保留属性获天赋点 | Prestige 机制，`extraData` 记录 |
| **法宝系统** | 独立养成，提供主动技能 | `domain/artifact/` 包 |
| **灵根系统** | 角色初始灵根，影响功法适配和突破 | `User` 新增字段或 `extraData` |
| **LLM NPC 系统** | 地图可交互 NPC，独立人格、知识库、行为逻辑，支持对话/交易/任务 | `domain/npc/` 包（NPCTemplate、NPCInstance），复用 `SpiritChatService` 的 ChatClient + Function Calling | **商铺掌柜已设计** → `docs/交易系统设计.md` |

---

## 三、架构预留

当前代码设计为扩展预留的接口：

| 接口/组件 | 可扩展方向 |
|-----------|------------|
| `CombatEngine` 接口 | 不同战斗模式（擂台、秘境、阵法战） |
| `ItemUseHandler` 策略模式 | 新增物品类型只需加 handler |
| `User.extraData` JSONB | 功法、经脉、成就等无需改表 |
| `BuffManager` | 阵法、丹药、功法 buff 均可复用 |
| `TrainingCombatLogic` 事件模式 | 顿悟、奇遇等直接扩展 |
| `SpiritChatService` + Function Calling | 泛化到任意 NPC 对话，每个 NPC 独立 prompt + tool set |
