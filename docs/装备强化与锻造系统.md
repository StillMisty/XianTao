# 装备强化与锻造系统

## 1. 核心概念

### 1.1 锻材三性

锻造使用独立于五行（药材）的**锻材属性体系**，数据完全隔离——药材不能锻造，锻材不能炼丹。

| 属性 | Key | 中文 | 影响 |
|------|-----|------|------|
| 刚 | `RIGIDITY` | 材料硬度/密度 | 成品基础攻防值 |
| 韧 | `TOUGHNESS` | 材料韧性/延展 | qualityMultiplier 区间偏向 |
| 灵 | `SPIRIT` | 灵气蕴含度 | 词缀数量及特殊词缀概率 |

锻材（`ItemType.MATERIAL`）在 `properties` JSONB 中存储三性值：

```json
{"RIGIDITY": 12, "TOUGHNESS": 5, "SPIRIT": 3}
```

只存非零值，missing key 视为 0。

### 1.2 锻造 vs 炼丹对照

| 概念 | 炼丹 | 锻造 |
|------|------|------|
| 输入物 | 药材 (HERB) | 锻材 (MATERIAL) |
| 属性体系 | 五行 (5元素) | 锻材三性 (3属性) |
| 配方载体 | `ItemProperties.Scroll` + `ItemType.RECIPE_SCROLL` | `ItemProperties.ForgingBlueprint` + `ItemType.FORGING_BLUEPRINT` |
| 配方学习表 | `xt_player_pill_recipe` | `xt_player_forging_recipe` |
| 品质判定 | 成色（上成/中成/下成） | 品质分 → 稀有度池 |
| 产出 | 丹药 (POTION)，受抗性衰减 | 装备实例 (Equipment)，受稀有度波动 |
| 核心算法 | `PillCombinationFinder` | `ForgingCombinationFinder`（同算法，不同属性） |

## 2. 锻造系统

### 2.1 锻造图纸

作为 `ItemProperties` 的密封子类型，存储于物品模板 JSONB 属性中：

```json
{
  "equipment_template_id": 42,
  "grade": 3,
  "requirements": {
    "RIGIDITY": {"min": 20, "max": 40},
    "TOUGHNESS": {"min": 10, "max": 25},
    "SPIRIT": {"min": 5, "max": 15}
  }
}
```

| 字段 | 说明 |
|------|------|
| `equipment_template_id` | 产出装备模板 ID（→ `xt_equipment_template.id`） |
| `grade` | 图纸品阶 1~9，影响品质分基线 |
| `requirements` | 三性要求，key 为属性名 (RIGIDITY/TOUGHNESS/SPIRIT)，val 为 min/max |

图纸物品类型为 `FORGING_BLUEPRINT`，命名如"铁剑锻造图"、"寒冰剑图谱"。使用后玩家学会该图纸。

### 2.2 玩家已学图纸 (xt_player_forging_recipe)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | FK → xt_user(id) |
| `blueprint_template_id` | BIGINT | FK → xt_item_template(id)，图纸模板 |
| `equipment_template_id` | BIGINT | FK → xt_equipment_template(id)，产出装备模板 |
| `learn_time` | TIMESTAMP | 学习时间 |

UNIQUE(user_id, blueprint_template_id)：不可重复学习。

### 2.3 锻造流程

```
选图纸 → 放入锻材 → 累计三性总量 → 匹配约束范围 → 算品质分 → 映射稀有度池 → Roll稀有度 → 生成装备
```

#### 步骤 1：累计三性

```
累计刚 = Σ(锻材A RIGIDITY × 数量A + 锻材B RIGIDITY × 数量B + ...)
累计韧 = Σ(锻材A TOUGHNESS × 数量A + 锻材B TOUGHNESS × 数量B + ...)
累计灵 = Σ(锻材A SPIRIT × 数量A + 锻材B SPIRIT × 数量B + ...)
```

#### 步骤 2：约束匹配

```
图纸.刚.min ≤ 累计刚 ≤ 图纸.刚.max
图纸.韧.min ≤ 累计韧 ≤ 图纸.韧.max
图纸.灵.min ≤ 累计灵 ≤ 图纸.灵.max
```

全部满足 → 进入品质判定。否则 → 锻造失败，材料不消耗。

#### 步骤 3：品质分计算（同丹药成色算法）

```
每项属性的达标度 = 1 - |实际值 - 范围中心| / 范围半宽
范围中心 = (max + min) / 2
范围半宽 = (max - min) / 2（精确值时达标度为 1）

品质分 = Σ(三项达标度) / 3
```

图纸品阶 `grade` 提供品质分保底加成：`最终分 = 品质分 + grade × 0.02`。

### 2.4 品质分 → 稀有度池

| 品质分 | 稀有度权重池 |
|--------|------------|
| ≥ 0.9 | LEGENDARY 10 : EPIC 40 |
| ≥ 0.7 | EPIC 30 : RARE 40 : COMMON 10 |
| ≥ 0.5 | RARE 20 : COMMON 50 : BROKEN 10 |
| < 0.5 | COMMON 30 : BROKEN 50 |

稀有度加权随机 roll 出后，`qualityMultiplier` 和词缀在对应稀有度范围内随机生成（与掉落装备一致）。

#### 品质计算示例

```
图纸 寒冰剑图: RIGIDITY[20~40] TOUGHNESS[10~25] SPIRIT[5~15]

投入材料后累计: 刚=30 韧=18 灵=10

刚达标度 = 1 - |30-30|/10     = 1.00
韧达标度 = 1 - |18-17.5|/7.5  = 0.93
灵达标度 = 1 - |10-10|/5      = 1.00

品质分 = (1.00 + 0.93 + 1.00) / 3 = 0.98 → 可 roll LEGENDARY/EPIC
```

### 2.5 图纸获取途径

- 灵石购买（NPC 商店）
- 地图探索掉落图纸卷轴
- 悬赏奖励
- 秘境产出

## 3. 强化系统

### 3.1 强化上限

| 稀有度 | 最高强化等级 |
|--------|------------|
| BROKEN | +3 |
| COMMON | +5 |
| RARE | +8 |
| EPIC | +12 |
| LEGENDARY | +18 |

### 3.2 三阶段设计

| 阶段 | 成功率来源 | 消耗 | 图纸要求 |
|------|-----------|------|---------|
| +0 → +3 | **100%**（必定成功） | 纯灵石 | 不需要 |
| +4 → +9 | **固定概率**（80% → 50% 递减） | 灵石 + 锻材 | 不需要 |
| +10 → 上限 | **品质分 = 成功率**（完美命中 = 100%） | 灵石 + 锻材 | **必须持有图纸** |

### 3.3 成功率曲线

#### +4 → +9（固定概率）

```
+4: 80%   +5: 75%   +6: 70%
+7: 65%   +8: 60%   +9: 55%
+10: 50%（若不持有图纸则只能到此）
```

稀有度修正：传说 ×0.9，史诗 ×0.95，稀有 ×1.0，普通 ×1.05，破旧 ×1.1。

#### +10 → 上限（品质分 = 成功率）

强化消耗的材料同样需要匹配三性约束。约束取自装备对应的**锻造图纸**，并按强化等级放大：

```
强化约束 = 图纸基础约束 × (1 + (强化目标等级 - 9) × 0.3)

例如强化到 +12：
  图纸 RIGIDITY[20~40] → 强化约束 RIGIDITY[20~40] × (1 + 3×0.3) = [38~76]
```

材料累计三性越接近约束中心，成功率越高。

```
成功率 = calculateQualityScore(材料三性总量, 强化约束)

完美匹配中心点 → 100%
有一定偏差     → 线性下降
严重不匹配     → 低至 0%
```

### 3.4 消耗

#### 灵石消耗

```
灵石消耗 = 基础值 × 稀有度系数 × (1 + forgeLevel × 0.3)

基础值 = 50（可配置）
稀有度系数: BROKEN=0.5, COMMON=1, RARE=2, EPIC=4, LEGENDARY=8
```

示例：强化传说装备到 +15
```
50 × 8 × (1 + 15 × 0.3) = 400 × 5.5 = 2200 灵石
```

#### 锻材消耗（+4 以上）

- +4 → +9：消耗通用锻材（稀有度决定最低三性要求）
- +10 → 上限：消耗锻材须满足强化约束（≥ 图纸约束 × 增幅系数），**实际消耗的材料数量由玩家自行选择**，只要累计三性落在约束范围内即可

### 3.5 失败惩罚

装备**不会损坏**。失败只降 1 级（`forgeLevel -= 1`），材料正常消耗。

### 3.6 图纸门槛逻辑

- +0 → +9：蓝图未持有时也可强化（通用约束）
- +10 → 上限：**必须在 `xt_player_forging_recipe` 中找到该装备对应的图纸**。若装备为掉落的（无对应图纸），强化上限为 +9

这赋予锻造装备更大的成长空间——自己锻造的装备可以强化到满，掉落装备止步 +9。

### 3.7 里程碑奖励

| 里程碑 | 奖励 |
|--------|------|
| +5 | 随机一条已有属性词缀值 +1 |
| +10 | 随机一条已有词缀值 +1（含特殊词缀） |
| +15 | 若词缀未满（不足稀有度上限），解锁一条新随机词缀 |

### 3.8 自动 vs 手动强化

| 模式 | 命令 | 行为 |
|------|------|------|
| 自动 | `强化 [装备名/编号]` | +0→+3 直接扣灵石；+4 以上自动从背包选最优锻材（+4→+9 满足通用约束即止，+10→上限 尝试匹配中心点） |
| 手动 | `强化 [装备名/编号] [锻材1]×N [锻材2]×M ...` | 玩家精确指定锻材及数量，系统验证三性累计落在约束范围内后执行强化 |

手动模式让玩家在 +10 以上阶段可以追求完美中心匹配 → 100% 成功率。

## 4. 架构设计

### 4.1 新增类型

| 层 | 新增项 | 说明 |
|----|--------|------|
| enum | `ItemType.FORGING_BLUEPRINT` | 锻造图纸物品类型 |
| entity | `ItemProperties.ForgingBlueprint` | `sealed` 子类型 (equipmentTemplateId, grade, requirements) |
| entity | `PlayerForgingRecipe` | 玩家已学锻造图纸实体 |
| vo | `ForgingResultVO` | 锻造结果（成功/失败、产出装备、消耗材料、品质分） |
| vo | `EnhanceResultVO` | 强化结果（成功/失败、新等级、消耗、成功率） |

### 4.2 新增表

```sql
CREATE TABLE xt_player_forging_recipe (
    id                      BIGSERIAL PRIMARY KEY,
    user_id                 BIGINT NOT NULL REFERENCES xt_user(id),
    blueprint_template_id   BIGINT NOT NULL REFERENCES xt_item_template(id),
    equipment_template_id   BIGINT NOT NULL REFERENCES xt_equipment_template(id),
    learn_time              TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, blueprint_template_id)
);
```

### 4.3 服务拆分

```
ForgingService              → 公开 API（@Authenticated），锻造入口
ForgingCombinationFinder    → @Component，材料匹配 + 品质分计算 + 稀有度映射
EnhancementService          → 公开 API（@Authenticated），强化入口 + 成功率计算 + 约束放大
                              + autoEnhance（自动选材）/ manualEnhance（手动配材）
```

### 4.4 新增 ItemUseHandler

```
ForgingBlueprintUseHandler  → 使用 FORGING_BLUEPRINT 类型物品时学习图纸
                               consumesInternally() = false（由 ItemUseService 统一扣减）
```

### 4.5 命令接口

| 命令 | 功能 |
|------|------|
| `锻造列表` | 查看已学锻造图纸 |
| `锻造 [图纸名]` | 自动配材锻造 |
| `锻造 [锻材1]×N [锻材2]×M ...` | 手动配材锻造 |
| `强化 [装备名/编号]` | 自动强化（系统自动选材） |
| `强化 [装备名/编号] [锻材1]×N [锻材2]×M ...` | 手动强化（精确控制配比追求高成功率） |
| `使用 [图纸卷轴名]` | 使用图纸卷轴学习（走 ItemUseHandler） |
