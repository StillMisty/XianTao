# 地图与探索系统

## 1. 地图节点 (xt_map_node)

| 字段                    | 类型           | 说明                                       |
|-----------------------|--------------|------------------------------------------|
| `id`                  | BIGSERIAL    | PK                                       |
| `name`                | VARCHAR(128) | 地图名称                                     |
| `description`         | TEXT         | 描述                                       |
| `map_type`            | VARCHAR(32)  | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement`   | INT          | 等级要求                                     |
| `travel_time_minutes` | INT          | 默认旅行耗时                                   |
| `neighbors`           | JSONB        | 相邻地图 `{"地图名": 耗时分钟}`                     |
| `specialties`         | JSONB        | 特产掉落池 (加权列表)                             |
| `travel_events`       | JSONB        | 旅行事件权重                                   |

### 1.1 JSONB 结构

**neighbors**:

```json
{
  "黑金主城": 5,
  "枯骨林": 10
}
```

**specialties**:

```json
[
  {
    "name": "毒龙草",
    "templateId": 101,
    "weight": 30
  }
]
```

**travel_events**:

```json
[
  {
    "eventType": "ambush",
    "weight": 40
  }
]
```

### 1.2 连接表 (xt_map_connection)

双向连接记录，含 `from_node_id`, `to_node_id`, `travel_time`。

## 2. 旅行系统

### 2.1 双模式

| 模式 | 指令            | 逻辑                       |
|----|---------------|--------------------------|
| 自动 | `#前往 [地图]`    | 优先体力跳过(5点/分钟)，体力不足降为真实时间 |
| 等待 | `#前往 [地图] 等待` | 强制真实时间                   |

### 2.2 真实时间模式

1. 设置状态 RUNNING, 记录 `travel_start_time` + `travel_destination_id`
2. 到达后 `#完成旅行` 手动触发完成
3. 触发 D20 事件判定 (D20 ≤ 10 → 触发事件, 50%)

### 2.3 约束

- 状态必须为 IDLE
- 目标必须相邻
- 等级必须满足要求

## 3. 历练系统 (AFK)

懒加载结算，不实时累积收益。

### 3.1 收益公式

```
铜币   = 10 × 分钟数 × (1 + AGI × 0.01)
灵石   = 5 × (分钟数 / 60)  // 不足1小时不计
经验   = 10 + WIS × 2 + 地图等级 × 5
掉落次数 = (分钟数 / 10) × (1 + AGI × 0.01)
```

### 3.2 掉落机制

每次掉落从地图 `specialties` 加权随机抽取 1-3 个。

### 3.3 状态追踪

- `training_start_time` 记录开始时间
- 结束时计算：`分钟数 = (当前时间 - training_start_time) 的分钟差`
- 结束后自动重新开始新一轮（设置 `training_start_time = 当前时间`）

## 4. 探索系统

### 4.1 结果分层

| 有效智慧  | 结果   | 奖励                      |
|-------|------|-------------------------|
| 1-20  | 基础发现 | 1个基础材料 (1-3个)           |
| 21-50 | 稀有发现 | 1个稀有材料                  |
| 51-80 | 重大发现 | 2个稀有材料 + 30%配方          |
| 81+   | 惊人发现 | 3个稀有材料 + 100%配方 + 20%灵蛋 |

### 4.2 消耗

固定 20 点体力，状态必须为 IDLE。

### 4.3 LLM 美化

探索完成后调用 Spring AI 生成个性化描述文本，LLM 不可用时使用固定模板降级。
