# 算运系统

## 概述

每日运势系统，基于玩家 ID 与日期确定性生成三维运势值，影响历练、战斗、掉落、悬赏等核心玩法。运势天注定，不可逆转。

## 运势维度

| 维度 | 影响范围 | 幅度说明 |
|------|----------|----------|
| 财运 | 掉落物概率与数量、悬赏灵石奖励 | 保守，±15%~20% |
| 机缘 | 子事件触发率（旅行/历练/悬赏） | 大幅，±30%，机缘本就无常 |
| 气运 | 历练修为倍率、战斗怪物强度 | 中等，±20%，怪物等级偏移 ±0~3 |

## 运势等级

综合三维判定，分为五档：大吉、吉、平、凶、大凶。展示时附带一行八字点评（如"财源广进"、"机缘天降"、"诸事不宜，宜闭关静修"等）。

## 数据流

每天首次加载玩家状态时，检测是否存在当日运势事件。若不存在则自动生成一条 FORTUNE 事件写入事件队列，随下一条指令的回复消息送达玩家。玩家也可主动发送「今日运势」指令随时查询。

运势值不在数据库中存储（仅记录日期以避免重复生成），每次实时计算，同一玩家同一日期结果恒定。

## 接入方式

### 系统层乘法器

在历练结算、子事件选择、战斗遇怪、掉落处理、悬赏结算等关键节点直接读取运势值，作为乘数或偏移量参与计算。

### 上下文注入

所有子事件效果执行器的上下文中注入完整运势对象，后续事件配置或效果处理器可直接读取，无需改动调用链。

## 设计考量

- **确定性**：以玩家 ID + 日期为种子做 hash，无随机性，无需存储
- **非对称波动**：机缘波动范围最大（奇遇本就难以预测），财运气运相对保守
- **凶日也有价值**：凶日更适合高回报的极端事件，避免纯粹 debuff 体验

## 扩展预留

- 隐藏事件触发条件可增加运势阈值
- 事件 narratives 可引用运势值做动态分词
