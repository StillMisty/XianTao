# 炼丹系统

## 1. 核心概念

### 1.1 五行属性 (ElementType)

| 枚举值 | code | 中文 |
|-------|------|------|
| METAL | metal | 金 |
| WOOD | wood | 木 |
| WATER | water | 水 |
| FIRE | fire | 火 |
| EARTH | earth | 土 |

药材拥有五行属性中的若干种。存储于 `ItemTemplate.properties` JSONB：

```json
{
  "elements": {"wood": 3, "fire": 1, "water": 2}
}
```

- 只存非零值，missing key 视为 0
- 代码层通过 `ItemTemplate.getElementValue(code)` 读取（返回 `int`）

### 1.2 丹方数据

丹方数据直接存于 `ItemTemplate.properties`，不单独建表：

```json
{
  "recipe": {
    "grade": 3,
    "result_item_id": 123,
    "result_quantity": 1,
    "requirements": {
      "wood": {"min": 10, "max": 15},
      "water": {"min": 6, "max": 9},
      "fire": {"min": 5, "max": 7}
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `grade` | 丹药品阶 1~9 |
| `result_item_id` | 成品丹药模板 ID |
| `result_quantity` | 成丹数量 |
| `requirements` | 五行要求，key 为 ElementType code，val 为 min/max |

丹方卷轴物品（`ItemTemplate`）命名如"天元丹方"，使用后玩家学会该丹方。

### 1.3 玩家已学丹方 (PlayerPillRecipe)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | FK → xt_user(id) |
| `recipe_template_id` | BIGINT | FK → xt_item_template(id)，丹方卷轴模板 |
| `result_item_id` | BIGINT | FK → xt_item_template(id)，成品丹药模板 |
| `learn_time` | TIMESTAMP | 学习时间 |

UNIQUE(user_id, recipe_template_id)：不可重复学习。

### 1.4 丹药品质与成色

丹药存储在 `xt_inventory_item`，通过 JSONB `properties` 存储品阶与成色：

```json
{"grade": 3, "quality": "superior"}
```

| 概念 | 叫法 | 说明 |
|------|------|------|
| **品阶** | 一品~九品 | 丹方固有，决定药效基础值 |
| **成色** | 上成/中成/下成 | 炼制时由五行匹配度决定 |

## 2. 药材体系

### 2.1 药材品阶梯度

| 品阶 | 产地 | 单药材五行值 |
|------|------|------------|
| 凡阶 | 低阶地图、一级福地 | 1~2 |
| 灵阶 | 中阶地图、三级福地 | 2~4 |
| 玄阶 | 高阶地图、五级福地 | 4~7 |
| 地阶 | 特殊副本 | 6~10 |
| 天阶 | 世界 Boss、活动 | 9~15 |

### 2.2 药材获取

- 灵田种植（`ItemType.SEED` → `ItemType.HERB`）
- 地图探索掉落
- 悬赏奖励
- NPC 商店购买

## 3. 炼丹流程

### 3.1 药材槽位

炼丹炉最多支持 **5 个药材位**，每个位置放一种药材（可用数量 >1，合并计算五行）。

### 3.2 五行累加

```
累计木 = Σ(药材A 木 × 数量A + 药材B 木 × 数量B + ...)
累计火 = Σ(药材A 火 × 数量A + 药材B 火 × 数量B + ...)
```

### 3.3 成丹判定

遍历玩家已学丹方，检查累计五行是否同时在丹方要求的范围内：

```
metal_min ≤ 累计金 ≤ metal_max
wood_min ≤ 累计木 ≤ wood_max
water_min ≤ 累计水 ≤ water_max
fire_min ≤ 累计火 ≤ fire_max
earth_min ≤ 累计土 ≤ earth_max
```

全部满足 → 成丹。无一匹配 → 失败，药材消耗。

### 3.4 成色判定

每行属性计算达标度：

```
达标度 = 1 - |实际值 - 范围中心| / 范围半宽
范围中心 = (max + min) / 2
范围半宽 = (max - min) / 2（精确值时达标度为 1）
```

取所有涉及属性的达标度平均值：

| 平均达标度 | 成色 | 效果倍率 |
|-----------|------|---------|
| ≥ 0.8 | 上成 | ×1.5 |
| ≥ 0.5 | 中成 | ×1.0 |
| < 0.5 | 下成 | ×0.7 |

丹方不涉及的属性不参与成色计算。

## 4. 自动炼丹算法

给定目标丹方，系统自动从玩家背包搜索最优药材组合：

1. 筛选玩家背包中所有 `ItemType.HERB` 物品
2. 在 5 个药材位内搜索可能的组合
3. 五行累计值落在丹方范围内的为有效解
4. 多解时选成色最高的组合
5. 无解 → 提示缺哪些属性，**不消耗药材**

## 5. 丹药类型与效果

丹药效果存储在 `ItemTemplate.properties`：

```json
{
  "pill_type": "exp",
  "exp_amount": 500
}
```

| pill_type | 效果 | 示例 |
|-----------|------|------|
| exp | 增加经验 | 培元丹、聚气丹 |
| hp | 回复 HP（百分比） | 回春丹、大还丹 |
| breakthrough | 突破成功率加成 | 破境丹 |
| stat | 永久属性点 | 洗髓丹 |

服用命令：`使用 [丹药名称/编号]`

效果受成色倍率影响：上成×1.5 / 中成×1.0 / 下成×0.7

## 6. 丹方获取途径

- 灵石购买（NPC 商店）
- 怪物掉落丹方卷轴（使用后习得）
- 地图奇遇领悟

| 命令 | 功能 | 权限 |
|------|------|------|
| `丹方列表` | 查看已学丹方 | 已学 |

| `使用 [丹药名称/编号]` | 服用丹药 | 背包有 |

## 8. 品质计算示例

```
丹方 天元丹: 木[10~15] 水[6~9] 火[5~7]（金、土不参与）

方案A: 木=13 水=8 火=6
  木达标度 = 1 - |13-12.5|/2.5 = 0.8
  水达标度 = 1 - |8-7.5|/1.5  = 0.67
  火达标度 = 1 - |6-6|/1      = 1.0
  平均 = 0.82 → 上成

方案B: 木=10 水=6 火=5
  木达标度 = 0, 水达标度 = 0, 火达标度 = 0
  平均 = 0 → 下成
```
