# 炼丹系统

## 1. 核心概念

### 1.1 五行属性 (ElementType)

| 枚举值 | code | 中文 |
|-------|------|------|
| METAL | metal | 金 |
| WOOD | wood | 木 |
| WATER | water | 水 |
| FIRE | fire | 火 |
| EARTH | earth | 土 |

药材拥有五行属性中的若干种。存储于 `ItemTemplate.properties` JSONB：

```json
{
  "elements": {"wood": 3, "fire": 1, "water": 2}
}
```

- 只存非零值，missing key 视为 0
- 代码层通过 `ItemTemplate.getElementValue(code)` 读取（返回 `int`）

### 1.2 丹方数据

丹方数据直接存于 `ItemTemplate.properties`，不单独建表：

```json
{
  "recipe": {
    "grade": 3,
    "result_item_id": 123,
    "result_quantity": 1,
    "requirements": {
      "wood": {"min": 10, "max": 15},
      "water": {"min": 6, "max": 9},
      "fire": {"min": 5, "max": 7}
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `grade` | 丹药品阶 1~9 |
| `result_item_id` | 成品丹药模板 ID |
| `result_quantity` | 成丹数量 |
| `requirements` | 五行要求，key 为 ElementType code，val 为 min/max |

丹方卷轴物品（`ItemTemplate.type=RECIPE_SCROLL`）命名如"天元丹方"，使用后玩家学会该丹方。

### 1.3 玩家已学丹方 (xt_player_pill_recipe)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | FK → xt_user(id) |
| `recipe_template_id` | BIGINT | FK → xt_item_template(id)，丹方卷轴模板 |
| `result_item_id` | BIGINT | FK → xt_item_template(id)，成品丹药模板 |
| `learn_time` | TIMESTAMP | 学习时间 |

UNIQUE(user_id, recipe_template_id)：不可重复学习。

### 1.4 丹药品阶与成色

丹药存储在 `xt_inventory_item`，通过 JSONB `properties` 存储品阶：

```json
{"grade": 3, "quality": "superior"}
```

| 概念 | 叫法 | 说明 |
|------|------|------|
| **品阶** | 一品~九品 | 丹方固有，决定药效基础值，影响等级衰减 |
| **成色** | 上成/中成/下成 | 炼制时由五行匹配度决定 |

## 2. 药材体系

### 2.1 药材品阶梯度

| 品阶 | 产地 | 单药材五行值 |
|------|------|------------|
| 凡阶 | 低阶地图、一级福地 | 1~2 |
| 灵阶 | 中阶地图、三级福地 | 2~4 |
| 玄阶 | 高阶地图、五级福地 | 4~7 |
| 地阶 | 特殊副本 | 6~10 |
| 天阶 | 世界 Boss、活动 | 9~15 |

### 2.2 药材获取

- 灵田种植（`ItemType.SEED` → `ItemType.HERB`）
- 地图探索掉落
- 悬赏奖励
- NPC 商店购买

## 3. 炼丹流程

### 3.1 药材槽位

炼丹炉最多支持 **5 个药材位**，每个位置放一种药材（可用数量 >1，合并计算五行）。

### 3.2 五行累加

```
累计木 = Σ(药材A 木 × 数量A + 药材B 木 × 数量B + ...)
累计火 = Σ(药材A 火 × 数量A + 药材B 火 × 数量B + ...)
```

### 3.3 成丹判定

遍历玩家已学丹方，检查累计五行是否同时在丹方要求的范围内：

```
metal_min ≤ 累计金 ≤ metal_max
wood_min ≤ 累计木 ≤ wood_max
water_min ≤ 累计水 ≤ water_max
fire_min ≤ 累计火 ≤ fire_max
earth_min ≤ 累计土 ≤ earth_max
```

全部满足 → 成丹。无一匹配 → 失败，药材消耗。

### 3.4 成色判定

每行属性计算达标度：

```
达标度 = 1 - |实际值 - 范围中心| / 范围半宽
范围中心 = (max + min) / 2
范围半宽 = (max - min) / 2（精确值时达标度为 1）
```

取所有涉及属性的达标度平均值：

| 平均达标度 | 成色 | 效果倍率 |
|-----------|------|---------|
| ≥ 0.8 | 上成 | ×1.5 |
| ≥ 0.5 | 中成 | ×1.0 |
| < 0.5 | 下成 | ×0.7 |

丹方不涉及的属性不参与成色计算。

## 4. 自动炼丹算法

给定目标丹方，系统自动从玩家背包搜索最优药材组合：

1. 筛选玩家背包中所有 `ItemType.HERB` 物品
2. 在 5 个药材位内搜索可能的组合
3. 五行累计值落在丹方范围内的为有效解
4. 多解时选成色最高的组合
5. 无解 → 提示缺哪些属性，**不消耗药材**

## 5. 丹药效果系统

### 5.1 效果定义

丹药效果存储在 `ItemTemplate.properties` JSONB 中：

```json
{
  "effects": [
    {"type": "exp", "amount": 500},
    {"type": "hp", "amount": 0, "percentage": 0.3},
    {"type": "stat", "statAttr": "str", "amount": 2},
    {"type": "breakthrough", "rate": 0.15}
  ]
}
```

一张丹药可以有多个效果（如同时加修为和回血）。

### 5.2 效果类型

| 类型 | 说明 | 受等级衰减 | 受抗性衰减 |
|------|------|-----------|-----------|
| `exp` | 增加修为 | ✅（有 0.1 保底） | ✅ |
| `hp` | 回复 HP（固定值或百分比）。若玩家处于濒死状态，额外复活回满血并恢复 IDLE | ❌ | ❌ |
| `stat` | 永久增加指定四维属性（`statAttr`: str/con/agi/wis） | ✅（无保底） | ✅ |
| `breakthrough` | 一次性突破成功率加成（`rate`: 0.15 = +15%） | ✅（有 0.1 保底） | ❌ |

实际效果 = 效果基础值 × 成色倍率 × 等级衰减系数 × 抗性衰减系数

### 5.3 濒死复活

服用 Hp 丹药时，若玩家状态为 `DYING`：
- `hpCurrent` 设为满血
- `status` 恢复为 `IDLE`
- 效果文本提示"复活"

## 6. 等级约束与抗性系统

### 6.1 等级衰减

不同效果类型使用不同的衰减公式：

#### Exp 类（修为）

```
等级衰减 = max(0.1, min(1.0, pillGrade / (playerLevel × 0.2)))
```

低品修为丹对高等级仍有微量收益，有 0.1 保底。

#### Stat 类（永久属性）

```
等级衰减 = min(1.0, pillGrade / (playerLevel × 0.2))
```

**无保底**。低品丹对高等级可能降为零，防止无限刷低品属性丹。

| 丹药等级 | 5级玩家 | 20级玩家 | 50级玩家 | 100级玩家 |
|---------|--------|---------|---------|----------|
| 1品 | 1.00 | 0.25 | 0.10 | 0.05 |
| 3品 | 1.00 | 0.75 | 0.30 | 0.15 |
| 5品 | 1.00 | 1.00 | 0.50 | 0.25 |
| 7品 | 1.00 | 1.00 | 0.70 | 0.35 |
| 9品 | 1.00 | 1.00 | 0.90 | 0.45 |

### 6.2 抗性系统

永久收益类丹药（exp、stat）会产生抗性。每次服用同一种丹药，抗性计数 +1。

```
抗性衰减 = max(0.1, 1 / (1 + count))
```

| 服用次数 | 抗性衰减 |
|---------|---------|
| 第1次 | 1.00 |
| 第2次 | 0.50 |
| 第3次 | 0.33 |
| 第4次 | 0.25 |
| 第5次 | 0.20 |
| 第10次 | 0.10 (保底) |

抗性存储于独立表 `xt_pill_resistance`，不随时间自动衰减。

### 6.3 数据存储

```sql
-- 丹药抗性表
CREATE TABLE xt_pill_resistance (
    user_id     BIGINT NOT NULL REFERENCES xt_user(id),
    template_id BIGINT NOT NULL REFERENCES xt_item_template(id),
    count       INT NOT NULL DEFAULT 0,
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    PRIMARY KEY (user_id, template_id)
);

-- 玩家 Buff 表（战斗增益 + 突破加成）
CREATE TABLE xt_player_buff (
    id          BIGSERIAL PRIMARY KEY,
    user_id     BIGINT NOT NULL REFERENCES xt_user(id),
    buff_type   VARCHAR(32) NOT NULL,
    value       INT NOT NULL,
    expires_at  TIMESTAMP NOT NULL,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_player_buff_expires ON xt_player_buff(expires_at);
CREATE INDEX idx_player_buff_user ON xt_player_buff(user_id);
```

| `buff_type` | 含义 |
|-------------|------|
| `attack` | 战斗攻击增益 |
| `defense` | 战斗防御增益 |
| `speed` | 战斗速度增益 |
| `breakthrough` | 突破成功率加成 |

### 6.4 突破丹药

突破丹药服用后写入 `xt_player_buff`，设置过期时间（如1小时）。下次执行突破时从表中读取未过期的 `breakthrough` Buff 累加到此轮成功率，突破完成后（无论成败）清除该玩家所有 `breakthrough` Buff。

突破成功率公式更新为：

```
总成功率 = min(100, 基础概率 + 失败次数 × 单次补偿 + 护道加成 + 突破丹药加成)
```

### 6.5 战斗增益丹药

服用后写入 `xt_player_buff`，设置过期时间（如300秒）。`CombatService` 每场战斗前读取未过期的 `attack/defense/speed` Buff 叠加到战斗属性。过期后由定期清理任务清除。

## 7. 架构设计

### 7.1 服务拆分

```
PillRecipeService      → 丹方学习、已学列表、详情查询
PillRefiningService    → 自动/手动炼丹、成色计算
PillConsumptionService → 服用丹药、抗性查询更新、等级衰减计算、效果应用
```

### 7.2 物品消耗

`ItemUseService` dispatcher 在找到匹配 handler 后，**统一调用扣减逻辑**：

```
ItemUseService.useItem() →
  1. 匹配 handler
  2. itemService.reduceStackableItem(userId, templateId, 1) ← 统一扣减
  3. handler.use(userId, item, template, args)
```

各 handler 不再自行处理消耗。

### 7.3 服用丹药流程

```
PillConsumptionService.takePill(userId, pillName):
  1. 从背包查找 POTION 物品
  2. 查询 xt_pill_resistance 获取抗性 → 计算抗性衰减
  3. 读取模板 properties.effects → 逐条应用：
     a. exp:   等级衰减(有保底) × 抗性衰减 → 加修为 → 更新抗性
     b. hp:    直接加血，若 DYING 则复活 → 不更新抗性
     c. stat:  等级衰减(无保底) × 抗性衰减 → 加属性字段 → 更新抗性
     d. breakthrough: 等级衰减(有保底) → INSERT INTO xt_player_buff → 不更新抗性
  4. 扣减物品（由 ItemUseService 统一处理）
```

## 10. 品质计算示例

```
丹方 天元丹: 木[10~15] 水[6~9] 火[5~7]（金、土不参与）

方案A: 木=13 水=8 火=6
  木达标度 = 1 - |13-12.5|/2.5 = 0.8
  水达标度 = 1 - |8-7.5|/1.5  = 0.67
  火达标度 = 1 - |6-6|/1      = 1.0
  平均 = 0.82 → 上成

方案B: 木=10 水=6 火=5
  木达标度 = 0, 水达标度 = 0, 火达标度 = 0
  平均 = 0 → 下成
```
