# 地图旅行悬赏与历练

## 1. 地图节点 (xt_map_node)

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `travel_time_minutes` | INT | 默认旅行耗时 |
| `neighbors` | JSONB | 相邻地图 `{"地图名": 耗时分钟}` |
| `specialties` | JSONB | 特产掉落池 `{templateId: weight}` |
| `travel_events` | JSONB | 旅行事件权重 |

### 1.1 特产 (specialties) JSONB

```json
{
  "1": 50,
  "3": 10
}
```

- key 为 `xt_item_template.id`，value 为掉落权重

### 1.2 旅行事件 (travel_events) JSONB

```json
{
  "ambush": 40,
  "find_treasure": 10
}
```

## 2. 旅行系统

- `#前往 [地图]` → 进入 RUNNING 状态，记录 startTime
- 到达后 `#完成旅行` 触发 D20 事件 (D20 ≤ 10 触发)
- 约束: 必须 IDLE, 目标相邻, 等级满足

---

## 3. 历练系统 (Training)

AFK 挂机，懒加载结算。

### 3.1 收益

```
经验 = 2 × 分钟数 × (1 + AGI × 0.01)
物品 = (分钟数 / 10) × 效率倍率 次, 从特产加权随机
```

- 只产出经验和材料, 不含铜币/灵石
- 同名物品合并

### 3.2 状态追踪

- 开始: `#历练` → trainingStartTime, status → EXERCISING
- 结束: `#历练结算` → 计算 `now - startTime`, 发放奖励, status → IDLE

---

## 4. 地图类型枚举 (MapType)

| 类型 | 说明 |
|---|---|
| SAFE_TOWN | 安全区域 |
| TRAINING_ZONE | 野外历练区 |
| HIDDEN_ZONE | 秘境 |

## 5. 旅行事件枚举 (TravelEventType)

| 事件 | 说明 | 效果 |
|---|---|---|
| AMBUSH | 伏击 | 扣 10-29 HP |
| FIND_TREASURE | 发现宝藏 | 得 50-149 铜币 |
| WEATHER | 恶劣天气 | 纯叙事 |
| SAFE_PASSAGE | 安全通行 | 无事发生 |
