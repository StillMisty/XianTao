# 地图、怪物、战斗与历练

## 1. 地图节点 (xt_map_node)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `neighbors` | JSONB | 相邻地图 `{"地图名": 耗时分钟}` |
| `specialties` | JSONB | 特产掉落池 `{templateId: weight}` |
| `travel_events` | JSONB | 旅行事件权重 |
| `monster_encounters` | JSONB | 遇怪池 `{monsterTemplateId: weight}` |
| `encounter_size` | JSONB | 每波怪物数量 `{"min": 1, "max": 3}` |

### 1.1 特产 (specialties) JSONB

```json
{"1": 50, "3": 10}
```

key 为 `xt_item_template.id`，value 为掉落权重。

### 1.2 旅行事件 (travel_events) JSONB

```json
{"ambush": 40, "find_treasure": 10}
```

### 1.3 遇怪池 (monster_encounters) JSONB

```json
{"1": 50, "2": 30}
```

key 为 `xt_monster_template.id`，value 为遇怪权重。触发 AMBUSH 事件时，按此权重选取怪物模板。

### 1.4 遭遇规模 (encounter_size) JSONB

```json
{"min": 1, "max": 3}
```

触发 AMBUSH 时，先按 `monster_encounters` 选模板，再按 `encounter_size` 随机生成 `[min, max]` 只怪物组成一波。

## 2. 怪物系统

### 2.1 怪物模板 (xt_monster_template) — 静态配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 怪物名称 |
| `description` | TEXT | 描述 |
| `monster_type` | VARCHAR(32) | 怪物类型 (BEAST/SPIRIT/ARMORED/…) |
| `base_level` | INT | 基准等级 |
| `base_hp` | INT | 基准生命值 |
| `base_attack` | INT | 基准攻击力 |
| `base_defense` | INT | 基准防御力 |
| `base_speed` | INT | 基准速度 |
| `exp_reward` | INT | 击杀经验 |
| `skills` | JSONB | 法决ID列表 `[1, 2, 3]`，引用 `xt_skill.id` |
| `drop_table` | JSONB | 掉落表 |
| `tags` | JSONB | 标签（克制判断等） |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 2.2 掉落表 (drop_table) JSONB

```json
{
  "equipment": {"1": 50, "2": 30},
  "items": {"10": 80, "20": 30, "100": 40}
}
```

- `equipment`：key 为 `xt_equipment_template.id`，value 为权重。击杀时加权 Roll 选模板 → 按模板的 `drop_weight` 确定稀有度 → 生成 `Equipment` 实例。
- `items`：key 为 `xt_item_template.id`，value 为权重。击杀时加权 Roll 选模板 → 发放堆叠物品（含材料、丹药、法决玉简等标品）。

### 2.3 怪物实例 (Monster) — 运行时

不持久化，遇怪时由服务层动态生成一组怪物：

```
① 从 map_node.monster_encounters 加权选取 monster_template
② 根据 map_node.encounter_size 决定本波数量：[min, max] 随机
③ 每只按玩家等级缩放属性：
    实例属性 = 模板基准属性 × (玩家等级 / 模板基准等级)
④ 生成一组 transient Monster 对象投入战斗
⑤ 战斗结束后销毁
```

### 2.4 怪物类型 (MonsterType)

| 枚举 | 中文 | 被克制法器 |
|------|------|----------|
| BEAST | 妖兽 | 刀 |
| SPIRIT | 灵体 | 剑 |

## 6. 历练系统

### 6.1 历练开始

- `历练` → 记录 `trainingStartTime`，status → EXERCISING
- 可指定地图（默认当前所在地图）
- 要求：当前地图为 TRAINING_ZONE 类型

### 6.2 历练结算流程

```
endTraining(userId) → CombatService.simulateTraining(userId, mapId, durationMinutes)

① 查用户属性、装备、已装载法决
② 查玩家出战灵兽（HP>0 且非休养中，上限 3 只）
③ 查地图的 monster_encounters + encounter_size
④ 计算遇敌次数 = floor(durationMinutes / 遇敌间隔)
⑤ 按顺序模拟每场 encounter：
    组成 TeamA = [PlayerCombatant, BeastCombatant...]
    按 encounter_size 生成 M 只 MonsterCombatant → TeamB
    simulate(TeamA, TeamB, 20)
    胜利 → 每只怪物独立 Roll 掉落 → 发放 + 灵兽获得经验
    中断 → 玩家濒死 或 灵兽全灭
⑥ 返回：击杀数 / 经验 / 掉落列表 / HP变化 / 法决触发记录 / 灵兽状态
```

### 6.3 中断条件

```
玩家 HP ≤ 0 → status → DYING（濒死），灵兽自动撤退
              已获得的经验/掉落照常发放
灵兽全灭 → 继续 solo 战斗，不再召唤新灵兽
全体阵亡（玩家 + 灵兽）→ 训练中断
```

### 6.4 结算示例输出

```
===== 历练结算 =====
地图: 青木林
时长: 30 分钟
遇敌: 8 场
击杀: 22 只（3 场战败中断）
━┅━┅━┅━┅━┅━
灵兽: 火羽鸡 Lv.3 参战（击杀 8 只）
获得经验: +340（玩家）/ +120（灵兽）
获得物品:
  ×5 灵草
  ×2 兽骨
  ×1 破损的木剑
受伤: 损失 87 HP（剩余 13/100）
灵兽受伤: 火羽鸡 HP 42→15
法决: 御剑术触发 6 次
```

| 事件 | 说明 | 效果 |
|------|------|------|

| SAFE_PASSAGE | 安全通行 | 无事发生 |
