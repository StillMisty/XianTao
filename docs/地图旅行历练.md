# 地图、怪物、战斗与历练

## 1. 地图节点 (xt_map_node)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `neighbors` | JSONB | 相邻地图 `{"地图名": 耗时分钟}` |
| `specialties` | JSONB | 特产掉落池 `{templateId: weight}` |
| `travel_events` | JSONB | 旅行事件权重 |

### 1.1 特产 (specialties) JSONB

```json
{"1": 50, "3": 10}
```

key 为 `xt_item_template.id`，value 为掉落权重。

### 1.2 旅行事件 (travel_events) JSONB

```json
{"ambush": 40, "find_treasure": 10}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |

## 5. 自动战斗系统 (CombatService)

### 5.1 核心原则

- **懒结算**：开始历练只记录时间，结算时一次性模拟所有战斗
- **无后台任务**：过程中零开销、零实时状态
- **掉落自动入包**：结算时所有掉落直接进背包

### 5.2 懒结算流程

```
endTraining(userId) → CombatService.simulateTraining(userId, mapId, durationMinutes)

① 查用户属性、装备、已收纳的法决
② 查地图怪物列表（按 spawn_weight）
③ 计算遇敌次数 = floor(durationMinutes / 遇敌间隔)
④ 按顺序模拟每场战斗：
   选怪（加权随机）
   → 计算玩家伤害（含法器克制 ×1.5 / ×0.8）
   → 计算法决触发（CD 轮播）
   → 计算击杀所需轮数
   → 累计受伤
   → HP 足够 → 击杀 + 经验 + 掉落
   → HP 不够 → 中断（濒死结束）
⑤ 返回：击杀数 / 经验 / 掉落列表 / 剩余HP / 法决触发记录
```

### 5.3 伤害公式

```
# 玩家对怪物
playerDamage = playerAttack × weaponTypeAdvantage(weaponType, monsterType)
finalDmg = max(1, playerDamage - monster.defense × 0.3)

# 怪物对玩家
monsterDmg = max(1, monster.attack × reverseAdvantage - player.defense × 0.3)
```

### 5.4 敏捷影响

敏捷影响**每次战斗中的攻击轮次**（更高敏捷 = 同时间内出手更多次），从而间接提高战斗效率。

### 5.5 克制矩阵

| 法器克制怪物 | 玩家伤害 | 怪物伤害 |
|------------|---------|----------|
| 克制 | ×1.5 | ×0.8 |
| 被克制 | ×0.8 | ×1.5 |
| 互无关系 | ×1.0 | ×1.0 |

详见表：[物品装备.md §3](./物品装备.md#3-法器类型-weapontype-与克制矩阵)

### 5.6 法决集成

在每场战斗中，已放入槽位的法决按独立 CD 自动触发，详见 [法决.md §7](./法决.md#7-战斗自动施放)。

## 6. 历练系统

### 6.1 历练开始

- `历练` → 记录 `trainingStartTime`，status → EXERCISING
- 可指定地图（默认当前所在地图）
- 要求：当前地图为 TRAINING_ZONE 类型

### 6.2 历练结算

- `历练结算` → 调用 `CombatService.simulateTraining()`
- 发放所有奖励（经验 + 掉落物，自动入背包）
- 应用受伤（更新 `hpCurrent`）
- 显示结算摘要（击杀数、经验、掉落列表、HP变化）
- status → IDLE 或 DYING（如果 HP ≤ 0）

### 6.3 濒死中断

```
HP ≤ 0 → 立即停止遇敌
       → status → DYING（濒死）
       → 已获得的经验/掉落照常发放
       → 结算摘要标注"濒死中断"
```

### 6.4 结算示例输出

```
===== 历练结算 =====
地图: 青木林
时长: 30 分钟
遇敌: 12 场
击杀: 10 只（濒死中断）
━┅━┅━┅━┅━┅━
获得经验: +340
获得物品:
  ×3 灵草
  ×2 兽骨
  ×1 破损的木剑
受伤: 损失 87 HP（剩余 13/100）
法决: 火球术触发 6 次
```
