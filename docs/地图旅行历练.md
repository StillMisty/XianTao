# 地图、怪物、战斗与历练

## 1. 地图节点

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `neighbors` | JSONB | 相邻地图 `[{"targetId": 1, "minutes": 5}]` |
| `specialties` | JSONB | 特产掉落池 `[{"templateId": 1, "weight": 30}]` |
| `encounter_richness` | INT | 事件密度 1-10, 默认5。越高遇怪/事件越频发 |

### 1.1 相邻地图 (neighbors) JSONB

```json
[{"targetId": 1, "minutes": 5}, {"targetId": 2, "minutes": 3}]
```

- `targetId` 为目标地图 ID
- `minutes` 为旅行耗时（分钟）

### 1.2 特产 (specialties) JSONB

```json
[{"templateId": 1, "weight": 50}, {"templateId": 3, "weight": 10}]
```

- `templateId` 为物品模板ID
- `weight` 为掉落权重

### 1.3 事件密度 (encounter_richness)

控制历练时遇怪和事件的触发频率。值越大间隔越短。

```
encounter_richness: 1~10，默认 5
  1  = 几乎无事发生（安全城镇）
  5  = 普通野外
  10 = 凶险之地，怪事频发
```

遇怪配置已迁移到 `xt_activity_event` 表（event_type='COMBAT'），子事件配置在同一个表中（event_type='NUMERIC'）。详见 [异步事件系统.md](./异步事件系统.md#51-事件池存储--统一事件系统)。

### 1.4 遭遇规模

遇怪时按 `monster_encounters` 的 `weight` 加权选取怪物模板，再按 `min` / `max` 随机生成怪物数量组成一波。旅行事件（AMBUSH/FIND_TREASURE 等）已迁移至 `xt_activity_event` 表。

## 2. 怪物系统

### 2.1 怪物模板 — 静态配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 怪物名称 |
| `description` | TEXT | 描述 |
| `monster_type` | VARCHAR(32) | 怪物类型 (BEAST/SPIRIT/ARMORED/…) |
| `base_level` | INT | 基准等级 |
| `base_hp` | INT | 基准生命值 |
| `base_attack` | INT | 基准攻击力 |
| `base_defense` | INT | 基准防御力 |
| `base_speed` | INT | 基准速度 |
| `exp_reward` | INT | 击杀经验 |
| `skills` | JSONB | 法决ID列表 `[1, 2, 3]` |
| `drop_table` | JSONB | 掉落表 |
| `tags` | JSONB | 标签（克制判断等） |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 2.2 掉落表 (drop_table) JSONB

```json
[
  {"category": "equipment", "templateId": 1, "weight": 50},
  {"category": "items", "templateId": 10, "weight": 80}
]
```

- `category` 为 "equipment" 或 "items"，分别对应装备或物品
- `templateId` 引用对应的模板表ID
- `weight` 为百分比概率 (0-100)，表示该条目被选中时掉落对应物品

### 2.3 怪物实例 — 运行时

不持久化，遇怪时由服务层动态生成一组怪物：

```
① 从地图的 monster_encounters 按 weight 加权选取怪物模板
② 根据选中条目的 min/max 决定本波数量：[min, max] 随机
③ 每只按玩家等级缩放属性：
    实例属性 = 模板基准属性 × (玩家等级 / 模板基准等级)，整数除法
④ 生成一组临时怪物对象投入战斗
⑤ 战斗结束后销毁
```

怪物实例的属性由模板基准属性按玩家等级与模板基准等级的比例缩放计算。

### 2.4 怪物类型

| 枚举 | 中文 | 被克制法器 |
|------|------|----------|
| BEAST | 妖兽 | 刀 |
| SPIRIT | 灵体 | 剑 |
| ARMORED | 甲胄 | 斧 |
| WILD_BEAST | 猛兽 | 枪 |
| EVIL | 邪祟 | 棍 |
| FLYING | 飞行 | 弓 |
| HUMAN | 人形 | 无特定克制 |

## 3. 地图类型

| 类型 | 说明 |
|------|------|
| SAFE_TOWN | 安全区 |
| TRAINING_ZONE | 野外历练区（有怪物） |
| HIDDEN_ZONE | 秘境入口节点 |

### 3.1 秘境入口节点

`HIDDEN_ZONE` 类型的地图节点作为秘境的物理入口。

```
秘境入口 (HIDDEN_ZONE)
  │ 玩家通过旅行系统到达此处
  │ 输入「秘境 紫府秘境」
  ▼
秘境系统 (DungeonService)
  ├── 创建 dungeon_instance
  ├── 进入外围区域
  ├── 探索建筑 POI（复用 CombatEngine，BattleScene.DUNGEON）
  ├── 「秘境继续」推进至内围/核心
  └── 击败镇守 BOSS 或撤退 → 结算退出
```

- 玩家必须先旅行到此地图节点，才能进入秘境
- 入口节点本身不参与历练（无 monster_encounters）
- 秘境内部的外围/内围/核心三重天地由秘境系统独立编排，不经过地图系统
- 探索中的战斗复用 `CombatEngine`，`BattleContext.scene=DUNGEON`

详见 `docs/秘境系统设计.md`。

## 4. 旅行系统

### 4.1 旅行开始

- `前往 [地图]` → 进入 RUNNING 状态，记录出发时间和目的地ID
- 约束：必须处于空闲状态，目标地图相邻，等级满足要求
- `neighbors` 中的 value 即为旅行耗时（分钟）

### 4.2 旅行自动结算

旅行到期后，下一次用户交互时自动结算（懒加载模式）：

```
出发时间 + 旅行耗时 ≤ 当前时间 → 到达目的地
更新所在地为目的地
状态切换为空闲
清空旅行相关记录
```

**触发点**：所有服务加载用户状态时自动执行，覆盖状态查询、前往、历练、悬赏接取等所有入口。

### 4.3 状态展示优化

`状态` 命令在赶路时显示优化格式（而非仅显示"赶路"）：

```
所在地：青山镇
状态：赶路中 (青山镇 → 青云山外门)
旅途进度：12分钟/15分钟（剩余 3分钟）
```

进度条基于出发时间和旅行耗时实时计算，到期显示"即将到达"。

### 4.4 旅行事件

| 事件 | 说明 | 效果 |
|------|------|------|
| AMBUSH | 伏击 | 按 `monster_encounters` 选取怪物进入战斗 |
| FIND_TREASURE | 发现宝藏 | 获得灵石 |
| WEATHER | 恶劣天气 | 纯叙事 |
| SAFE_PASSAGE | 安全通行 | 无事发生 |

事件权重在地图的 `travel_events` JSONB 中配置。

显示当前所在地图完整信息，包含遇怪列表（怪物名称、类型、基础等级、权重、数量范围）和相邻地图（含旅行耗时）：

```
【历练区】枯骨林 (推荐等级: 5)

阴森的森林，时有妖兽出没

【遇怪列表】
  枯骨蛇 [妖兽] Lv1  权重:50  数量:1~3
  黑风狼 [妖兽] Lv3  权重:30  数量:1

【相邻地图】
  黑金主城 (5分钟)
  青云山外门 (15分钟)

使用「前往 [地图名]」开始旅行。
```

遇怪数据来自 `xt_activity_event` 表（event_type='COMBAT'），MapService 批量查询后按权重降序排列。

## 6. 历练系统

### 6.1 历练开始

- `历练` → 记录开始时间，状态切换为历练中
- 只能在当前所在地图历练（不支持指定地图）
- 要求：当前地图为 TRAINING_ZONE 类型

### 6.2 历练结算流程

```
① 查用户属性、装备、已装载法决 + 出战灵兽
② 查询 xt_activity_event（activity_type=TRAINING, owner_id=mapId, is_hidden=false）
   统一池含 COMBAT（遇怪）+ NUMERIC（子事件）
③ 计算动态遇敌间隔（encounter_richness + 等级匹配 + 地图凶险）
④ 统一事件循环（一个 for）：
     按 weight 加权抽取事件
     COMBAT → CombatEventHandler 委托 CombatEngine
        构建玩家队伍 + 怪物队伍
        模拟战斗（最大回合数限制）
        回写玩家/灵兽 HP
        计算掉落 + 经验 + 灵兽经验
        顿悟判定（wisdom触发，可能悟得技能）
        中断 → 玩家濒死 → 终止循环
     NUMERIC → SubEventEffectExecutor
        读 params.effects JSONB → 分发到 effect handler
        加经验/扣血/加物品/掉特产/纯叙事
⑤ 隐藏事件检查（TriggerConditionChecker + SubEventEffectExecutor）
⑥ 基础经验 + 特产掉落结算
⑦ 产出总结
```

### 6.3 动态遇怪/事件间隔

**公式：**
```
interval = baseInterval × levelMismatch / mapDanger
```

**因子计算：**
- `baseInterval` = 12 - map.encounter_richness → 2~11 分钟（地图配置）
- `mapDanger` = 1 + (mapLevel - 1) × 0.015 → 1.0~2.5（高级图更凶险）
- `levelMismatch` = 1 + |playerLevel - mapLevel| / max(mapLevel, 1) × 2.5 → 1.0~3.5（等级匹配时密集，偏离时稀少）

**最小间隔：** 3 分钟　　**最大间隔：** 20 分钟

**设计意图：**
- 地图越凶→越频繁（mapDanger）
- 等级越匹配→越频繁（levelMismatch 接近 1）
- 高手去低级地图→几乎碰不到事件（levelMismatch 惩罚大）
- 新手去高级地图→也碰不到（同样惩罚大）
- 装备不管→装备的作用是打赢，不是躲开

**per-roll 触发概率：**
```
perRollChance = min(1.0, 0.4 × 10 / interval)
slots = duration / interval
```

**示例：**

| 场景 | richness | interval | 60分钟slots | 期望触发 |
|------|----------|----------|------------|---------|
| Lv1玩家 Lv1图 (5) | 5 | 7.0 | 8 | ~4.5 |
| Lv50玩家 Lv50图 (7) | 7 | 4.0 | 15 | ~15 |
| Lv50玩家 Lv10图 (5) | 5 | 16.0 | 3 | ~0.8 |
| Lv10玩家 Lv50图 (5) | 5 | 17.5 | 3 | ~0.7 |

### 6.4 等级衰减机制

**公式：**
```
expMultiplier = max(0.1, 1 - max(0, (playerLevel - mapLevel - 5)) × 0.04)
```

**规则：**
- 玩家等级 ≤ 地图等级 + 5：无衰减（100%经验）
- 玩家等级 > 地图等级 + 5：每高1级衰减4%
- 最低衰减到10%（保留保底经验）

**示例：**
- 玩家等级50，地图等级30
- 等级差 = 50 - 30 - 15 = 5
- 衰减 = 5 × 0.04 = 0.20
- `expMultiplier` = max(0.1, 1 - 0.20) = 0.80（80%经验）

### 6.5 中断条件

```
玩家 HP ≤ 0 → 进入濒死状态，灵兽自动撤退
              已获得的经验/掉落照常发放
灵兽全灭 → 继续 solo 战斗，不再召唤新灵兽
全体阵亡（玩家 + 灵兽）→ 训练中断
```

### 6.6 结算输出格式

**设计原则：** 文字MUD挂机游戏，输出要简洁，只统计数据。

**输出策略：**
- **普通战斗**：只统计数据，不输出详细过程
- **高光战斗**：每次历练最多1场战斗用LLM美化渲染，提升沉浸感

**普通战斗统计（所有战斗）：**
```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85
```

**高光战斗（LLM渲染，每次历练最多1场）：**
```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85

━━━ 高光时刻 ━━━
第5场战斗，你遭遇了一只剧毒蟾蜍王。它喷出的毒雾几乎遮蔽了你的视线，
但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
一剑刺穿了它的要害。这场战斗持续了5个回合，最终你以微弱优势胜出。
```

**高光战斗触发条件：**
- 战斗回合数超过10回合
- 双方血量都低于30%（势均力敌）
- 触发了稀有技能或连击（>=3次）
