# 地图、怪物、战斗与历练

## 1. 地图节点 (xt_map_node)

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `neighbors` | JSONB | 相邻地图 `{"地图名": 耗时分钟}` |
| `specialties` | JSONB | 特产掉落池 `{templateId: weight}` |
| `travel_events` | JSONB | 旅行事件权重 |
| `monster_encounters` | JSONB | 遇怪池 `{monsterTemplateId: {weight, min, max}}` |

### 1.1 特产 (specialties) JSONB

```json
{"1": 50, "3": 10}
```

key 为 `xt_item_template.id`，value 为掉落权重。

### 1.2 旅行事件 (travel_events) JSONB

```json
{"ambush": 40, "find_treasure": 10}
```

### 1.3 遇怪池 (monster_encounters) JSONB

```json
{
  "1": {"weight": 50, "min": 1, "max": 3},
  "2": {"weight": 30, "min": 1, "max": 2}
}
```

- key 为 `xt_monster_template.id`
- `weight` 为遇怪权重
- `min` / `max` 为每波怪物数量范围

### 1.4 遭遇规模

触发 AMBUSH 时，按 `monster_encounters` 的 `weight` 加权选取怪物模板，再按 `min` / `max` 随机生成怪物组成一波。

## 2. 怪物系统

### 2.1 怪物模板 (xt_monster_template) — 静态配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 怪物名称 |
| `description` | TEXT | 描述 |
| `monster_type` | VARCHAR(32) | 怪物类型 (BEAST/SPIRIT/ARMORED/…) |
| `base_level` | INT | 基准等级 |
| `base_hp` | INT | 基准生命值 |
| `base_attack` | INT | 基准攻击力 |
| `base_defense` | INT | 基准防御力 |
| `base_speed` | INT | 基准速度 |
| `exp_reward` | INT | 击杀经验 |
| `skills` | JSONB | 法决ID列表 `[1, 2, 3]`，引用 `xt_skill.id` |
| `drop_table` | JSONB | 掉落表 |
| `tags` | JSONB | 标签（克制判断等） |
| `create_time` / `update_time` | TIMESTAMP | 时间戳 |

### 2.2 掉落表 (drop_table) JSONB

```json
{
  "equipment": {"1": 50, "2": 30},
  "items": {"10": 80, "20": 30, "100": 40}
}
```

- `equipment`：key 为 `xt_equipment_template.id`，value 为权重。击杀时加权 Roll 选模板 → 按模板的 `drop_weight` 确定稀有度 → 生成 `Equipment` 实例。
- `items`：key 为 `xt_item_template.id`，value 为权重。击杀时加权 Roll 选模板 → 发放堆叠物品（含材料、丹药、法决玉简等标品）。

### 2.3 怪物实例 (Monster) — 运行时

不持久化，遇怪时由服务层动态生成一组怪物：

```
① 从 map_node.monster_encounters 按 weight 加权选取 monster_template
② 根据选中条目的 min/max 决定本波数量：[min, max] 随机
③ 每只按玩家等级缩放属性：
    实例属性 = 模板基准属性 × (玩家等级 / 模板基准等级)
④ 生成一组 transient Monster 对象投入战斗
⑤ 战斗结束后销毁
```

### 2.4 怪物类型 (MonsterType)

| 枚举 | 中文 | 被克制法器 |

## 4. 旅行系统

### 4.1 旅行开始

- `前往 [地图]` → 进入 RUNNING 状态，记录 `travel_start_time` + `travel_destination_id`
- 约束: 必须 IDLE, 目标相邻, 等级满足
- `neighbors` 中的 value 即为旅行耗时（分钟）

### 4.2 旅行自动结算

旅行到期后，**下一次用户交互时自动结算**（懒加载模式）：

```
UserStateService.getUser(userId)
  └─ resolveState()
       └─ tryCompleteTravel()
            ├── arrivalTime = travelStartTime + travelTime
            ├── now >= arrivalTime → 到达
            ├── user.locationId = travelDestinationId
            ├── user.status = IDLE
            └── 清空 travelStartTime / travelDestinationId
```

**触发点**：所有服务通过 `userStateService.getUser(userId)` 加载用户时自动执行，覆盖 `状态`、`前往`、`历练`、`悬赏接取` 等所有入口。

### 4.3 状态展示优化

`状态` 命令在赶路时显示优化格式（而非仅显示"赶路"）：

```
所在地：青山镇
状态：赶路中 (青山镇 → 青云山外门)
旅途进度：12分钟/15分钟（剩余 3分钟）
```

进度条基于 `travelStartTime` + `travelTime` 实时计算，到期显示"即将到达"。

## 6. 历练系统

### 6.1 历练开始

- `历练` → 记录 `trainingStartTime`，status → EXERCISING
- 只能在当前所在地图历练（不支持指定地图）
- 要求：当前地图为 TRAINING_ZONE 类型

### 6.2 历练结算流程

```
endTraining(userId) → TrainingService.simulateTraining(userId, durationMinutes)

① 查用户属性、装备、已装载法决
② 查玩家出战灵兽（HP>0 且非休养中，上限 2 只）
③ 查地图的 monster_encounters + encounter_size
④ EncounterCalculator 计算动态遇敌间隔（考虑地图难度、玩家等级、装备评分）
⑤ TrainingService 按顺序模拟每场 encounter：
     调用 CombatService.buildPlayerTeam() 构建 TeamA = [PlayerCombatant, BeastCombatant...]
     调用 CombatService.buildMonsterTeam() 按模板 + encounter_size 生成 M 只 Monster → TeamB
     调用 CombatService.simulate(TeamA, TeamB, 20)
     调用 PostCombatProcessor.applyHpToUser() 回写玩家 HP
     调用 PostCombatProcessor.applyHpToBeasts() 回写灵兽 HP + 休养/觉醒判定
     调用 DropProcessor.processMonsterDrops() 计算掉落
     胜利 → 每只怪物独立 Roll 掉落 → 发放 + 灵兽获得经验
     中断 → 玩家濒死 或 灵兽全灭
⑥ 返回：击杀数 / 经验 / 掉落列表 / HP变化 / 法决触发记录 / 灵兽状态
```

### 6.3 动态遇怪间隔

**公式：**
```
encounterInterval = baseInterval × (1 - difficultyFactor) × (1 - levelFactor) × (1 + gearFactor)
```

**因子计算：**
- `baseInterval` = 10分钟（基础间隔）
- `difficultyFactor` = min(0.5, mapLevel × 0.005)（地图难度因子）
- `levelFactor` = min(0.3, playerLevel × 0.0015)（玩家等级因子）
- `gearFactor` = min(0.15, gearScore × 0.0002)（装备评分因子，灵压威慑——装备越好遇怪越少）

**最小间隔：** 2分钟

**示例：**
- 玩家等级50，地图等级30，装备评分200
- `difficultyFactor` = 30 × 0.005 = 0.15
- `levelFactor` = 50 × 0.0015 = 0.075
- `gearFactor` = 200 × 0.0002 = 0.04
- `encounterInterval` = 10 × (1-0.15) × (1-0.075) × (1+0.04) = 10 × 0.85 × 0.925 × 1.04 ≈ 8.2分钟

### 6.4 等级衰减机制

**公式：**
```
expMultiplier = max(0.1, 1 - max(0, (playerLevel - mapLevel - 15)) × 0.04)
```

**规则：**
- 玩家等级 ≤ 地图等级 + 15：无衰减（100%经验）
- 玩家等级 > 地图等级 + 15：每高1级衰减4%
- 最低衰减到10%（保留保底经验）

**示例：**
- 玩家等级50，地图等级30
- 等级差 = 50 - 30 - 15 = 5
- 衰减 = 5 × 0.04 = 0.20
- `expMultiplier` = max(0.1, 1 - 0.20) = 0.80（80%经验）

### 6.5 中断条件

```
玩家 HP ≤ 0 → status → DYING（濒死），灵兽自动撤退
              已获得的经验/掉落照常发放
灵兽全灭 → 继续 solo 战斗，不再召唤新灵兽
全体阵亡（玩家 + 灵兽）→ 训练中断
```

### 6.6 结算输出格式

**设计原则：** 文字MUD挂机游戏，输出要简洁，只统计数据。

**输出策略：**
- **普通战斗**：只统计数据，不输出详细过程
- **高光战斗**：每次历练最多1场战斗用LLM美化渲染，提升沉浸感

**普通战斗统计（所有战斗）：**
```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85
```

**高光战斗（LLM渲染，每次历练最多1场）：**
```
【历练结算】幽暗沼泽 | 30分钟 | 效率1.2x
遇敌8场 | 击杀22只 | 战败1场
经验+460 | 灵草×5 兽骨×2 破损的木剑×1
灵兽：火羽鸡 +120exp HP42→15 | 灵狐 +60exp HP100→85

━━━ 高光时刻 ━━━

但火羽鸡及时喷出烈焰，将毒雾驱散。你抓住机会，御剑术触发，
```

**高光战斗触发条件：**
- 战斗回合数超过10回合
- 双方血量都低于30%（势均力敌）
- 触发了稀有技能或连击（>=3次）
