# XianTao 系统核心机制设计文档

## 一、 属性洗点系统 (Reset System)

XianTao系统鼓励玩家尝试不同流派，洗点机制采用**完全基于时间的冷却判定**，无道具消耗，无次数惩罚。

### 1. 核心规则

* **冷却时间**：3 天（72小时）。
* **次数限制**：无上限，不记录累计洗点次数。
* **状态记录**：系统仅在数据库中维护字段 `last_reset_points_time`（上次洗点时间戳）。

### 2. 执行逻辑

当玩家输入洗点指令时，系统进行以下判定：

1. **冷却检查**：检测 `当前时间 - last_reset_points_time`。
    * 若 `< 3天`：拒绝请求，返回提示：“道基尚未稳固，距离下次散功重修还需 [剩余时间]”。
    * 若 `>= 3天`：允许执行。
2. **执行重置**：
    * 力量、敏捷、体质、智慧四维属性全部扣除已分配的点数（归零或归至基础值）。
    * `available_points`（可用属性点） = 历史总计获得的属性点。
    * 更新 `last_reset_points_time = 当前时间`。
3. **战斗结算生效**：由于PVP无保护机制，洗点后属性瞬间变化，直接影响后续受到的PVP攻击判定。

## 二、 修为上限与突破机制 (Breakthrough System)

在XianTao中，突破是修仙的逆天之举，系统绝不代劳，且伴随极高的风险与收益。

### 1. 修为（经验）上限设定

为防止无限挂机导致的数值失控，设定修为存储上限：

* **储能极值**：当前境界所能存储的最大修为 = `当前突破所需修为 × 3`。
* *例如：突破下一境界需要 10,000 修为，玩家挂机最多只能存到 30,000 修为，溢出部分自动消散。*

### 2. 突破规则与惩罚

* **纯手动触发**：系统绝对不提供“自动突破”。玩家必须在状态最佳时手动发送 `#突破`。
* **失败惩罚**：突破失败会造成严重的道基反噬。
    * **扣除基准**：损失修为 = **当前境界升至下一境界所需的标准修为值**。
    * **底线保护**：扣除后修为不可为负数，最低扣至 0。
    * *例如：玩家当前有 25,000 修为，突破需 10,000 修为。突破失败后，扣除 10,000，剩余 15,000 修为。*

## 三、 护道系统 (Dao Protection)

护道是XianTao系统中纯粹的**社交互助玩法**。没有功德奖励，没有系统收益，完全基于道友间的人情羁绊。

### 1. 基础建立条件

* **境界压制**：护道者的当前境界 **必须 $\ge$** 突破者的当前境界。
* **数量限制（护道者视角）**：一位大能最多同时为 **3名** 道友护道。超出限制时系统将拒绝新请求。
* **数量限制（突破者视角）**：突破者可以接受**无限多**的人为其护道（不设人数上限），但总加成受限。

### 2. 加成计算与生效条件 (核心逻辑)

护道关系的建立不受位置影响，但**是否提供加成，在按下 `#突破` 的瞬间进行实时位置检测**。

* **生效前置**：护道者与突破者必须处于**同一地图节点**。
* **单人加成公式**：`单人加成 = 5% + (护道者境界层级 - 突破者境界层级) × 1%`
* **总加成公式**：`总加成 = min(20%, 所有【同地点】护道者加成之和)`

### 3. 边界情况处理规范

| 场景                  | 系统处理方式                                     | 提示/表现                             |
|:--------------------|:-------------------------------------------|:----------------------------------|
| **护道者移动到其他地图**      | **不主动解除**护道关系。但目标突破时，该护道者因位置校验失败，提供加成为 0%。 | 护道者若移回同地点，下次突破依旧有效。               |
| **突破者移动到其他地图**      | 同上。若突破时所有护道者都不在同地点，护道加成为 0%。               | 突破者：“天地孤寂，无道友相护。”                 |
| **护道者已满3人想加第4人**    | 拦截并拒绝。                                     | “你当前已在为3位道友护道，分身乏术。请先使用 `#护道解除`。” |
| **突破者接受5人护道，总计30%** | 允许关系存在。但最终突破成功率加成强行截断至上限 **20%**。          | 显示：“群仙庇佑，护道加成已达天地极值(20%)。”        |
| **突破结束（成功/失败）**     | 无论结果如何，**自动解除**该突破者的所有护道关系。                | 护道者收到通知：“【目标】突破已落幕，护道契约自动消散。”     |

## 四、 专属指令集 (Action-Description 格式)

为了配合上述设计，玩家需通过以下指令进行交互：

### 属性与散功

* **属性加点**，加点 [属性名] [数值]，将可用属性点分配给指定属性，即时生效。
* **洗点**，清空所有已分配属性并全额返还为可用属性点。每3天仅可使用一次，无任何额外消耗。

### 修为与突破

* **突破**，消耗当前境界所需的标准修为，冲击下一境界。若有同地点的护道者将获得加成；若失败则直接扣除巨量修为。

### 护道与羁绊

* **护道**，护道 [道号]，向境界不高于自己的道友发起护道契约。建立后，可在其突破时提供成功率加成。
* **护道解除**，护道解除 [道号]，主动单方面切断护道契约。腾出自己的护道名额（上限3人）。
* **护道查询**，查看当前正在为谁护道（显示目标及对方所在位置），以及有谁正在为自己护道（显示预估总加成及是否在同地点）。
