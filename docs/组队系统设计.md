# 组队系统设计

## 1. 定位

组队是独立于具体玩法的通用系统，复用于秘境、切磋等多人活动。队伍状态不绑定具体玩法——玩家先组队，再以队伍身份进入秘境或其他多人活动。

| 定位 | 说明 |
|------|------|
| **通用组队** | 与秘境/切磋解耦，队伍可跨玩法保持 |
| **邀请制** | 队长邀请，队员接受，支持多邀请排队 |
| **轻量** | 不绑定实例，离队/解散不影响已进入的秘境实例 |

---

## 2. 数据模型

### 2.1 `team` — 队伍

```sql
CREATE TABLE team (
    id              BIGSERIAL PRIMARY KEY,
    leader_id       BIGINT NOT NULL REFERENCES xt_user(id),
    member_count    INT NOT NULL DEFAULT 1,
    status          VARCHAR(16) NOT NULL DEFAULT 'ACTIVE'
                    CHECK (status IN ('ACTIVE', 'DISBANDED')),
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 2.2 `team_member` — 队伍成员

```sql
CREATE TABLE team_member (
    id              BIGSERIAL PRIMARY KEY,
    team_id         BIGINT NOT NULL REFERENCES team(id),
    user_id         BIGINT NOT NULL REFERENCES xt_user(id),
    joined_at       TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (user_id)  -- 同一时间只能在一支队伍中
);
```

### 2.3 `team_invitation` — 组队邀请

```sql
CREATE TABLE team_invitation (
    id              BIGSERIAL PRIMARY KEY,
    team_id         BIGINT NOT NULL REFERENCES team(id),
    inviter_id      BIGINT NOT NULL REFERENCES xt_user(id),
    invitee_id      BIGINT NOT NULL REFERENCES xt_user(id),
    status          VARCHAR(16) NOT NULL DEFAULT 'PENDING'
                    CHECK (status IN ('PENDING', 'ACCEPTED', 'REJECTED', 'EXPIRED')),
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    expires_at      TIMESTAMP NOT NULL DEFAULT (CURRENT_TIMESTAMP + INTERVAL '5 minutes')
);
```

## 3. 指令

| 指令 | 说明 |
|------|------|
| `组队` | 查看当前队伍信息 + 待处理邀请列表（含编号） |
| `组队邀请 {{道号}}` | 向指定玩家发出组队邀请 |
| `组队接受 [编号]` | 接受组队邀请（不填→最新一条；填编号→指定） |
| `组队拒绝 [编号]` | 拒绝组队邀请（不填→清空全部；填编号→指定） |
| `组队离开` | 离开当前队伍（队长离开→队伍解散） |

## 4. 生命周期

### 4.1 创建与加入

```
队长：组队邀请 张三
   ↓
张三：组队接受
   ↓
张三加入队长的队伍
```

- 被邀请者收到 DM 通知，内容含邀请编号
- 邀请 5 分钟过期，过期后状态变为 EXPIRED
- 同一用户可同时收到多条邀请，通过编号区分

### 4.2 离开与解散

- 队员 `组队离开`：从 `team_member` 中移除，`member_count--`
- 队长 `组队离开`：队伍状态变为 `DISBANDED`，所有成员自动离队
- 队伍为空时自动解散

### 4.3 与秘境交互

- 队伍存在时，队长 `秘境 {{名}}` 自动带入全队
- 队员可随时 `秘境撤退` 独自退出，不影响队伍和其他队友
- 队伍可跨多次秘境保持组队状态

### 4.4 与切磋交互

- 切磋前需先组队（预留）

## 5. 开发流程

1. `domain/team/` — `Team`、`TeamMember`、`TeamInvitation` entity + `TeamStatus` enum
2. `db/migration/` — 建表
3. `infrastructure/mapper/` + `repository/` — Mapper + RepositoryImpl
4. `service/TeamService` — 创建/邀请/接受/拒绝/离开，内部 `(Long userId)` + 公开 `@Authenticated`
5. `handle/command/TeamCommandHandler` — CommandGroup 实现
6. `handle/listener/TeamListener` — 双平台监听器
