# 地图旅行悬赏与历练

## 1. 地图节点 (xt_map_node)

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(128) | 地图名称 |
| `description` | TEXT | 描述 |
| `map_type` | VARCHAR(32) | 类型 (SAFE_TOWN/TRAINING_ZONE/HIDDEN_ZONE) |
| `level_requirement` | INT | 等级要求 |
| `travel_time_minutes` | INT | 默认旅行耗时 |
| `neighbors` | JSONB | 相邻地图 `{"地图名": 耗时分钟}` |
| `specialties` | JSONB | 特产掉落池 (加权列表，含rarity分级) |
| `travel_events` | JSONB | 旅行事件权重 |

### 1.1 特产 (specialties) JSONB

```json
[
  {"name":"铁矿石","weight":50,"templateId":1,"rarity":"common"},
  {"name":"灵草","weight":10,"templateId":3,"rarity":"rare"}
]
```

- `rarity`: `"common"` → 历练掉落, `"rare"` → 悬赏掉落

### 1.2 旅行事件 (travel_events) JSONB

```json
[
  {"eventType":"ambush","weight":40},
  {"eventType":"find_treasure","weight":10}
]
```

## 2. 旅行系统

- `#前往 [地图]` → 进入 RUNNING 状态，记录 startTime
- 到达后 `#完成旅行` 触发 D20 事件 (D20 ≤ 10 触发)
- 约束: 必须 IDLE, 目标相邻, 等级满足

---

## 3. 历练系统 (Training)

AFK 挂机，懒加载结算。

### 3.1 收益

```
经验 = 2 × 分钟数 × (1 + AGI × 0.01)
物品 = (分钟数 / 10) × 效率倍率 次, 从特产(rarity=common)加权随机
```

- 只产出经验和基础材料 (rarity=common), 不含铜币/灵石
- 同名物品合并

### 3.2 状态追踪

- 开始: `#历练` → trainingStartTime, status → EXERCISING
- 结束: `#历练结束` → 计算 `now - startTime`, 发放奖励, status → IDLE

## 4. 悬赏系统

### 4.1 数据表

**xt_bounty** (悬赏配置):

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `map_id` | BIGINT | 所属地图 |
| `name` | VARCHAR(100) | 悬赏名称 |
| `description` | TEXT | 描述 |
| `duration_minutes` | INTEGER | 耗时 |
| `rewards` | JSONB | 奖励池 (加权) |
| `require_level` | INTEGER | 等级要求 |
| `event_weight` | INTEGER | 事件权重 |

**rewards JSONB 格式**:
```json
[
  {"type":"rare_item","weight":50,"count":3},
  {"type":"spirit_stones","weight":30,"amount":100},
  {"type":"beast_egg","weight":20}
]
```

**xt_user_bounty** (玩家悬赏记录):

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGSERIAL | PK |
| `user_id` | BIGINT | 用户 |
| `bounty_id` | BIGINT | 悬赏 |
| `bounty_name` | VARCHAR(100) | 名称快照 |
| `start_time` | TIMESTAMP | 接取时间 |
| `duration_minutes` | INTEGER | 耗时 |
| `rewards` | JSONB | 预确定的奖励物品 |
| `status` | VARCHAR(16) | active/completed/abandoned |

### 4.2 奖励预确定 (种子随机)

接取时用 `seed = userId * 31 + todayEpochDay` 构建 Random，选取奖励并写入 `xt_user_bounty.rewards`。  
完成时直接读取发放，不重新计算——保证同日同用户相同结果，跨日不变。

### 4.3 奖励类型

- `rare_item`: 从地图特产 (rarity=rare) 加权选取
- `spirit_stones`: 固定数量灵石
- `beast_egg`: 从 ItemTemplate 随机选取 BEAST_EGG 类型

### 4.4 流程

```
#悬赏列表       → 查看当前地图可用悬赏
#接悬赏 [ID]    → 验证地图/等级 → 种子预确定奖励 → 写入 DB → status=BOUNTY
   ... 等待 duration_minutes ...
#交悬赏         → 检查时间 → 发放预存奖励 → D20事件 → LLM美化 → status=IDLE
#放弃悬赏       → 标记 abandoned → status=IDLE (无产出)
```

- 约束: 接取时须 IDLE, 交/弃时须 BOUNTY

## 5. 地图类型枚举 (MapType)

| 类型 | 说明 |
|---|---|
| SAFE_TOWN | 安全区域 |
| TRAINING_ZONE | 野外历练区 |
| HIDDEN_ZONE | 秘境 |

## 6. 旅行事件枚举 (TravelEventType)

| 事件 | 说明 | 效果 |
|---|---|---|
| AMBUSH | 伏击 | 扣 10-29 HP |
| FIND_TREASURE | 发现宝藏 | 得 50-149 铜币 |
| WEATHER | 恶劣天气 | 纯叙事 |
| SAFE_PASSAGE | 安全通行 | 无事发生 |
