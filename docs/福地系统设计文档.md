# 福地系统设计文档

## 1. 模块概述

福地系统是仙道游戏的私人领地玩法，包含灵气管理、网格布局、种植养殖、地灵MBTI人格化交互等核心功能。

### 1.1 核心职责

- 福地创建与灵气管理
- 网格布局系统（灵田、兽栏、阵眼）
- 种植收获机制（五行相生相克）
- 兽栏喂养系统
- 地灵MBTI人格化对话
- Function Calling机制
- 天劫渡劫系统

### 1.2 技术栈

- **ORM**：MyBatis-Flex
- **数据库**：PostgreSQL 18 (JSONB存储网格布局)
- **AI集成**：Spring AI 2.x (Function Calling、ChatClient)
- **并发模型**：Java 25 Virtual Threads

## 3. 核心业务规则

### 3.1 灵气管理系统

#### 3.1.1 灵气消耗公式

```
每小时消耗 = 基础消耗(10) + 地块数量 × 2 + 聚灵核心等级 × 5
```

**示例**：

- 初始福地：3个地块，核心等级1
- 每小时消耗 = 10 + 3×2 + 1×5 = 21灵气/小时

- 发展后：9个地块，核心等级5
- 每小时消耗 = 10 + 9×2 + 5×5 = 53灵气/小时

#### 3.1.2 灵气恢复机制

**献祭装备获得灵气**：

```
献祭收益 = 物品基础价值 × 品质乘数 × 等级修正
```

**品质乘数表**：
| 品质 | 乘数 |
|------|------|
| 白色(White) | 0.5x |
| 绿色(Green) | 1.0x |
| 蓝色(Blue) | 2.0x |
| 紫色(Purple) | 5.0x |
| 橙色(Orange) | 10.0x |

**等级修正**：

```
等级修正 = 1 + (物品等级 / 10)
```

**示例**：

- 献祭一件10级蓝色装备，基础价值100
- 献祭收益 = 100 × 2.0 × (1 + 10/10) = 400灵气

#### 3.1.3 灵气溢出保护

```
新灵气值 = min(灵气上限, 当前灵气 + 获得灵气)
```

- 超过上限的部分浪费
- 鼓励玩家及时消耗灵气（升级核心、扩建网格）

### 3.2 网格布局系统

#### 3.2.1 坐标系统

- 使用(x,y)坐标表示地块位置
- 初始网格大小为3x3（9个格子）
- 可通过升级扩展至4x4、5x5等
- 坐标示例："0,0"、"1,1"、"2,2"

#### 3.2.2 地块类型

**灵田(FARM)**：

- 用途：种植灵药
- 五行属性：金木水火土
- 特殊机制：相邻地块五行影响生长速度

**兽栏(PEN)**：

- 用途：饲养灵兽
- 饥饿值系统：0-100，低于30停止产出
- 每次喂食恢复30点饥饿值

**阵眼(NODE)**：

- 用途：提升灵气产出效率
- 五行属性：影响福地整体五行平衡
- 等级系统：高等级阵眼提供更多加成

#### 3.2.3 地块占用检查

```java
boolean isCellEmpty(Fudi fudi, String position) {
    return getCellByPosition(fudi, position) == null;
}
```

**建造前检查**：

- 坐标必须为空
- 坐标不能是焦土
- 坐标必须在网格范围内（0 ≤ x,y < grid_size）

### 3.3 种植收获机制

#### 3.3.1 生长进度计算（懒加载）

系统不主动更新生长进度，而是在查询时即时计算。

**计算公式**：

```
if (当前时间 ≥ 成熟时间):
    生长进度 = 1.0
else:
    总生长秒数 = (成熟时间 - 种植时间) 的秒差
    已过秒数 = (当前时间 - 种植时间) 的秒差
    生长进度 = min(1.0, 已过秒数 / 总生长秒数)
```

**优势**：

- 减少数据库写入频率
- 支持真正的离线生长
- 简化并发控制

#### 3.3.2 实际生长时间

```
实际生长时间 = 基础生长时间 / 生长速度修正系数
```

**生长速度修正系数**：
基于相邻地块的五行相生相克关系计算。

#### 3.3.3 五行相生相克系统

**五行关系**：

- **相生**：木→火→土→金→水→木
- **相克**：木→土→水→火→金→木

**修正系数**：

- 相邻地块五行相生：1.2x 加速
- 相邻地块五行相同：1.0x 无影响
- 相邻地块五行相克：0.8x 减速

**计算公式**：

```
生长速度修正 = Π(相邻地块修正系数)
```

**示例**：

- 在(1,1)种植木属性作物
- 相邻地块：左(0,1)=火、右(2,1)=水、上(1,0)=木、下(1,2)=土
- 修正系数：
    - 左(火)：木生火 → 相生 → 1.2x
    - 右(水)：水生木 → 相生 → 1.2x
    - 上(木)：相同 → 1.0x
    - 下(土)：木克土 → 相克 → 0.8x
- 最终修正 = 1.2 × 1.2 × 1.0 × 0.8 = 1.152x

#### 3.3.4 产量计算

```
基础产量 = 1-3随机
聚灵核心加成 = 核心等级 / 5（向下取整）
最终产量 = 基础产量 + 聚灵核心加成
```

**示例**：

- 核心等级7，基础产量2
- 聚灵核心加成 = 7/5 = 1
- 最终产量 = 2 + 1 = 3

#### 3.3.5 收获约束

- 只有生长进度 ≥ 1.0 的作物才能收获
- 收获消耗地灵精力（每块灵田5点精力）
- 收获后地块变为空地
- 收获的物品添加到玩家背包

### 3.4 兽栏喂养机制

#### 3.4.1 饥饿值系统

- 初始饥饿值：100
- 每次喂食恢复30点饥饿值
- 饥饿值低于30时灵兽停止产出
- 喂养消耗地灵精力（每次3点精力）

#### 3.4.2 喂养流程

1. 检查坐标是否为兽栏
2. 检查玩家是否有饲料物品
3. 恢复饥饿值：`newHunger = min(100, oldHunger + 30)`
4. 扣除地灵精力3点
5. 消耗饲料物品
6. 记录最后喂食时间

#### 3.5.3 情绪状态机

**情绪类型**：

- **平静(Calm)**：默认状态
- **愉悦(Joyful)**：灵气充足、福地繁荣时
- **疲惫(Tired)**：精力值低于30时
- **愤怒(Angry)**：灵气不足、地块损坏时
- **庆祝(Celebrating)**：渡劫成功时

**情绪转换规则**：

```
if (灵气值 > 灵气上限 × 80% && 地块全满):
    情绪 = Joyful
else if (精力值 < 30):
    情绪 = Tired
else if (灵气值 < 灵气上限 × 20% || 有焦土地块):
    情绪 = Angry
else if (刚渡劫成功):
    情绪 = Celebrating
else:
    情绪 = Calm
```

### 3.6 Function Calling机制

#### 3.6.1 工作流程

```
用户输入："@地灵 在0,0种灵芝"
    ↓
LLM接收输入 + 福地状态上下文
    ↓
LLM决定调用 plantCrop 工具
    ↓
Spring AI执行 FudiService.plantCrop(userId, "0,0", 101, "灵芝", WOOD)
    ↓
返回执行结果给LLM
    ↓
LLM生成回复："已在 (0,0) 种植灵芝，预计4小时后成熟～"
    ↓
返回给用户
```

**核心优势**：
- LLM自主决定调用哪个工具，无需中间意图识别层
- Spring AI自动处理工具调用和结果传递
- 代码简洁，维护成本低

#### 3.6.2 工具注册

将后端服务方法注册为Spring Bean，供LLM调用：

```java
@Component
public class SpiritTools {
    
    @Tool(description = "在指定坐标种植作物")
    public String plantCrop(String position, String cropName) {
        // 实现逻辑
    }
    
    @Tool(description = "收获指定坐标的成熟作物")
    public String harvestCrop(String position) {
        // 实现逻辑
    }
    
    // ... 其他工具
}
```

**关键注解**：
- `@Tool`：标记该方法为可用工具
- `description`：向LLM解释工具用途和参数要求
- LLM根据description自主决定是否调用该工具

### 3.7 天劫系统

#### 3.7.1 触发周期

- 每7天触发一次天劫
- 下次天劫时间 = 上次渡劫时间 + 7天

#### 3.7.2 天劫结果

**成功**：

- 地灵等级提升
- 渡劫连胜次数+1
- 获得奖励（灵气、道具等）
- 情绪切换为"庆祝"

**失败**：

- 随机地块变为焦土（无法使用）
- 渡劫连胜次数归零
- 情绪切换为"愤怒"

#### 3.7.3 焦土修复

- 需要消耗特殊道具或等待自动恢复
- 修复期间该坐标无法建造任何地块
- 修复后焦土坐标从scorched_cells列表中移除

public Optional<Fudi> getFudiByUserId(Long userId) {

## 附录A：核心公式汇总

### A.1 灵气消耗

- 每小时消耗：`10 + 地块数量 × 2 + 核心等级 × 5`

### A.2 献祭收益

- 献祭收益：`物品价值 × 品质乘数 × (1 + 物品等级/10)`

### A.3 生长速度

- 实际生长时间：`基础生长时间 / 生长速度修正系数`
- 生长速度修正：`Π(相邻地块修正系数)`
    - 相生：1.2x
    - 相同：1.0x
    - 相克：0.8x

### A.4 产量计算

- 最终产量：`(1-3随机) + (核心等级 / 5)`

### A.5 天劫周期

- 下次天劫：`上次渡劫时间 + 7天`
