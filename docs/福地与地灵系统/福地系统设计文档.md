# 福地系统设计文档

## 1. 模块概述

福地系统是仙道游戏的私人领地玩法，包含灵气管理、网格布局、种植养殖、地灵MBTI人格化交互等核心功能。

### 1.1 核心职责

- 福地创建与灵气管理
- 网格布局系统（灵田、兽栏、阵眼）
- 种植收获机制（五行相生相克）
- 兽栏喂养系统
- 地灵MBTI人格化对话
- Function Calling机制
- 天劫渡劫系统

## 3. 核心业务规则

### 3.1 灵气管理系统

#### 3.1.1 灵气消耗公式

```
每小时消耗 = 基础消耗(10) + 地块数量 × 2 + 聚灵核心等级 × 5
```

**示例**：

- 初始福地：3个地块，核心等级1
- 每小时消耗 = 10 + 3×2 + 1×5 = 21灵气/小时

- 发展后：9个地块，核心等级5
- 每小时消耗 = 10 + 9×2 + 5×5 = 53灵气/小时

#### 3.1.2 灵气恢复机制

**献祭装备获得灵气**：

```
献祭收益 = 物品基础价值 × 品质乘数 × 等级修正
```

**品质乘数表**：
| 品质 | 乘数 |
|------|------|
| 白色(White) | 0.5x |
| 绿色(Green) | 1.0x |
| 蓝色(Blue) | 2.0x |
| 紫色(Purple) | 5.0x |
| 橙色(Orange) | 10.0x |

**等级修正**：

```
等级修正 = 1 + (物品等级 / 10)
```

**示例**：

- 献祭一件10级蓝色装备，基础价值100
- 献祭收益 = 100 × 2.0 × (1 + 10/10) = 400灵气

#### 3.1.3 灵气溢出保护

```
新灵气值 = min(灵气上限, 当前灵气 + 获得灵气)
```

- 超过上限的部分浪费
- 鼓励玩家及时消耗灵气（升级核心、扩建网格）

### 3.2 网格布局系统

#### 3.2.1 坐标系统

- 使用(x,y)坐标表示地块位置
- 初始网格大小为3x3（9个格子）
- 可通过升级扩展至4x4、5x5等
- **聚灵核心**不占用网格位置，作为福地的独立属性存在
- 坐标示例："0,0"、"1,1"、"2,2"
- **聚灵核心**：不占用网格位置，作为福地的独立属性存在，等级决定灵气转化效率和存储上限。

#### 3.2.2 地块类型

**灵田(FARM)**：

- 用途：种植灵药
- 五行属性：金木水火土
- 特殊机制：相邻地块五行影响生长速度

**兽栏(PEN)**：

- 用途：饲养灵兽
- 饥饿值系统：0-100，低于30停止产出
- 每次喂食恢复30点饥饿值

**阵眼(NODE)**：

- 用途：提升灵气产出效率
- 五行属性：影响福地整体五行平衡
- 等级系统：高等级阵眼提供更多加成

#### 3.2.3 地块占用检查

```java
boolean isCellEmpty(Fudi fudi, String position) {
    return getCellByPosition(fudi, position) == null;
}
```

**建造前检查**：

- 坐标必须为空
- 坐标不能是焦土
- 坐标必须在网格范围内（0 ≤ x,y < grid_size）

### 3.3 种植收获机制

#### 3.3.1 生长进度计算（懒加载）

系统不主动更新生长进度，而是在查询时即时计算。

**计算公式**：

```
if (当前时间 ≥ 成熟时间):
    生长进度 = 1.0
else:
    总生长秒数 = (成熟时间 - 种植时间) 的秒差
    已过秒数 = (当前时间 - 种植时间) 的秒差
    生长进度 = min(1.0, 已过秒数 / 总生长秒数)
```

**优势**：

- 减少数据库写入频率
- 支持真正的离线生长
- 简化并发控制

#### 3.3.2 实际生长时间

```
实际生长时间 = 基础生长时间 / 生长速度修正系数
```

**生长速度修正系数**：
基于相邻地块的五行相生相克关系计算。

#### 3.3.3 五行相生相克系统

**五行关系**：

- **相生**：木→火→土→金→水→木
- **相克**：木→土→水→火→金→木

**修正系数**：

- 相邻地块五行相生：1.2x 加速
- 相邻地块五行相同：1.0x 无影响
- 相邻地块五行相克：0.8x 减速

**计算公式**：

```
生长速度修正 = Π(相邻地块修正系数)
```

**示例**：

- 在(1,1)种植木属性作物
- 相邻地块：左(0,1)=火、右(2,1)=水、上(1,0)=木、下(1,2)=土
- 修正系数：
    - 左(火)：木生火 → 相生 → 1.2x
    - 右(水)：水生木 → 相生 → 1.2x
    - 上(木)：相同 → 1.0x
    - 下(土)：木克土 → 相克 → 0.8x
- 最终修正 = 1.2 × 1.2 × 1.0 × 0.8 = 1.152x

#### 3.3.4 产量计算

```
基础产量 = 1-3随机
聚灵核心加成 = 核心等级 / 5（向下取整）
最终产量 = 基础产量 + 聚灵核心加成
```

**示例**：

- 核心等级7，基础产量2
- 聚灵核心加成 = 7/5 = 1
- 最终产量 = 2 + 1 = 3

#### 3.3.5 收获约束

- 只有生长进度 ≥ 1.0 的作物才能收获
- 收获消耗地灵精力（每块灵田5点精力）
- 收获后地块变为空地
- 收获的物品添加到玩家背包

### 3.4 兽栏喂养机制

#### 3.4.1 饥饿值系统

- 初始饥饿值：100
- 每次喂食恢复30点饥饿值
- 饥饿值低于30时灵兽停止产出
- 喂养消耗地灵精力（每次3点精力）

#### 3.4.2 喂养流程

1. 检查坐标是否为兽栏
2. 检查玩家是否有饲料物品
3. 恢复饥饿值：`newHunger = min(100, oldHunger + 30)`
4. 扣除地灵精力3点
5. 消耗饲料物品
6. 记录最后喂食时间

**行动组（SP）**：

|-----|----------|---------------------------------|

#### 3.5.3 情绪状态机

**情绪类型**：

- **平静(Calm)**：默认状态
- **愉悦(Joyful)**：灵气充足、福地繁荣时
- **疲惫(Tired)**：精力值低于30时
- **愤怒(Angry)**：灵气不足、地块损坏时
- **庆祝(Celebrating)**：渡劫成功时

**情绪转换规则**：

```
if (灵气值 > 灵气上限 × 80% && 地块全满):
    情绪 = Joyful
else if (精力值 < 30):
    情绪 = Tired
else if (灵气值 < 灵气上限 × 20% || 有焦土地块):
    情绪 = Angry
else if (刚渡劫成功):
    情绪 = Celebrating
else:
    情绪 = Calm
```

#### 3.5.4 形态演化三阶段

**初创期(Stage 1)**：

- 地灵等级：1-10
- 功能：基础对话，无福地详情感知
- Emoji：基础形态（如⚙️）

**底蕴期(Stage 2)**：

- 地灵等级：11-30
- 功能：可感知福地状态，提供建设建议
- Emoji：进阶形态（添加装饰）

**化形期(Stage 3)**：

- 地灵等级：31+
- 功能：完全人格化，主动管理福地
- Emoji：华丽形态（动态效果）

#### 3.5.5 地灵精力系统

- **精力上限**：100 点，每 24 小时（真实时间）自动恢复至满。
- **精力消耗**：
  - 收割每块灵田：消耗 5 点精力
  - 喂养每只灵兽：消耗 3 点精力
  - 自动补充灵气：消耗 10 点精力/次
- **疲劳状态**：精力值低于 30 时，地灵进入疲劳状态（情绪切换为 Tired），停止所有自动行为，仅响应手动指令。
- **疲劳可视化**：疲劳状态下，地灵 Emoji 添加睡眠符号（如 🔥 → 🔥💤），并提示"地灵已疲劳，需手动指令或等待恢复"。
- **自动行为规则**（需精力充足且自动模式开启）：
  - 收割成熟的灵田
  - 喂养饥饿值低于 30 的灵兽
  - 当灵气 < 50% 时，自动献祭背包中的白色装备补充灵气

### 3.6 Function Calling机制

#### 3.6.1 工作流程

```
用户输入："@地灵 在0,0种灵芝"
    ↓
LLM接收输入 + 福地状态上下文
    ↓
LLM决定调用 plantCrop 工具
    ↓
Spring AI执行 FudiService.plantCrop(userId, "0,0", 101, "灵芝", WOOD)
    ↓
返回执行结果给LLM
    ↓
LLM生成回复："已在 (0,0) 种植灵芝，预计4小时后成熟～"
    ↓
返回给用户
```

**核心优势**：
- LLM自主决定调用哪个工具，无需中间意图识别层
- Spring AI自动处理工具调用和结果传递
- 代码简洁，维护成本低

#### 3.6.2 工具注册

将后端服务方法注册为Spring Bean，供LLM调用：


**关键注解**：
- `@Tool`：标记该方法为可用工具
- `description`：向LLM解释工具用途和参数要求
- LLM根据description自主决定是否调用该工具

### 3.7 天劫系统

#### 3.7.1 触发周期

- 每7天触发一次天劫
- 下次天劫时间 = 上次渡劫时间 + 7天

#### 3.7.2 天劫攻防公式

**天劫攻击力**：
```
天劫攻击力 = 玩家等级 × 100 + 福地规模系数 × 200
```

**福地规模系数**：
| 网格大小 | 规模系数 |
|---------|---------|
| 3x3     | 1       |
| 4x4     | 1.5     |
| 5x5     | 2       |

**福地总防御**：
```
福地总防 = Σ(阵眼耐久度) + Σ(灵兽战力评分) + 玩家力量(STR)评分 × 10 + 地灵等级 × 50
```

**计算示例**：
- 玩家等级 30，3x3 福地，2个阵眼（耐久800,900），1只灵兽（战力评分500），STR评分 45，地灵等级 8
- 福地总防 = (800 + 900) + 500 + 45×10 + 8×50 = 1700 + 500 + 450 + 400 = **3050**
- 天劫攻击力 = 30×100 + 1×200 = **3200**
- 结果：失败（3050 < 3200）

#### 3.7.3 天劫结果

**成功**：

- 地灵等级提升
- 渡劫连胜次数+1
- 获得奖励（灵气、道具等）
- 情绪切换为"庆祝"

**失败**：

- 随机地块变为焦土（无法使用）
- 渡劫连胜次数归零
- 情绪切换为"愤怒"

#### 3.7.4 焦土修复

- 需要消耗特殊道具或等待自动恢复
- 修复期间该坐标无法建造任何地块
- 修复后焦土坐标从scorched_cells列表中移除

### 3.8 好友访问与注灵系统

#### 3.8.1 好友访问

- 玩家可以访问好友福地，查看布局和地灵状态。
- 访问指令：`#福地访问 道号`

#### 3.8.2 注灵机制

- **注入灵气**：消耗自身灵气为好友福地补充灵气，每次最多注入 **50 点**。
- **灵气扣除**：注灵会实时扣除自身福地的灵气存储。
- **多小号机制**：允许开多个小号互相注灵，但每个福地的灵气存储有上限（由聚灵核心等级决定）。

#### 3.8.3 灵气上限公式

```
灵气上限 = 1000 + 聚灵核心等级 × 200
```

### 3.9 蛰伏模式（离线保底）

#### 3.9.1 触发条件

- 玩家下次上线时检查，若连续离线 ≥ 24小时且当前灵气 < 灵气上限的 10%，则触发蛰伏模式。

#### 3.9.2 蛰伏模式效果

- 所有产出停止（灵田暂停生长、兽栏停止产出）。
- 灵气消耗降至 **1 灵气/小时**（最低保护消耗）。
- 确保核心不灭，仙草不枯，这是地灵对主人"最后的守护"。

#### 3.9.3 恢复条件

- 玩家上线后自动解除蛰伏模式，恢复正常运转。
- 可通过 `#唤醒福地` 指令手动解除。

#### 3.9.4 极端情况保护

- 即使离线 7 天，蛰伏模式也能保证福地不被摧毁（7 × 24 × 1 = 168 灵气最低消耗）。

## 4. 服务层设计

### 4.1 FudiService

**职责**：福地核心业务逻辑

**核心方法**：

## 5. 指令系统设计

玩家可以使用指令手动控制福地的各项功能，指令分为**查看类**、**操作类**、**管理类**三大类。

### 5.1 查看类指令

| 指令              | 功能说明                       |
|-----------------|----------------------------|
| `#福地` 或 `#福地状态` | 查看福地整体状态（灵气、地灵等级、网格布局等）    |
| `#福地网格`         | 查看网格布局详情（各地块类型、生长进度等）      |
| `#福地灵气`         | 查看灵气详细信息（当前值、上限、每小时消耗）     |
| `#福地地灵`         | 查看地灵详细信息（MBTI人格、形态阶段、好感度等） |

### 5.2 操作类指令

| 指令      | 功能说明                | 参数说明                        | 示例                          |
|---------|---------------------|-----------------------------|-----------------------------|
| `#种植`   | 在指定坐标种植灵药           | `<坐标> <作物ID或名称>`            | `#种植 0,0 灵芝`                |
| `#收获`   | 收获指定坐标的成熟灵药         | `<坐标>` 或 `all`（全部收获）        | `#收获 0,0` 或 `#收获 all`       |
| `#拆除`   | 拆除指定坐标的地块           | `<坐标>`                      | `#拆除 1,2`                   |
| `#建造`   | 在指定坐标建造地块           | `<坐标> <地块类型>`               | `#建造 1,2 灵田` 或 `#建造 2,0 阵眼` |
| `#献祭`   | 献祭背包中的装备换取灵气        | `<物品ID或名称>` 或 `all`（献祭所有白装） | `#献祭 铁剑` 或 `#献祭 all`        |
| `#喂养`   | 喂养指定兽栏中的灵兽          | `<坐标> <饲料ID或名称>`            | `#喂养 2,0 灵兽粮`               |
| `#福地升级` | 升级聚灵核心（消耗资源）        | 无参数                         | `#福地升级`                     |
| `#福地扩建` | 扩建福地网格（3x3→4x4→5x5） | 无参数                         | `#福地扩建`                     |

#### 5.2.1 种植指令详解

* **可用作物列表**：`#作物列表`
* **五行属性**：每种作物有五行属性，种植时需考虑相邻地块的五行相生相克。
* **生长时间**：根据作物品质和聚灵核心等级决定。
* **示例流程**：
  ```
  玩家：#作物列表
  地灵：【可用作物】
  1. 灵芝 (木系) - 生长时间 4小时 - 产出：回复丹药素材
  2. 人参 (土系) - 生长时间 6小时 - 产出：增益丹药素材
  3. 火莲 (火系) - 生长时间 8小时 - 产出：战斗丹药素材
  
  玩家：#种植 0,0 灵芝
  地灵：已在 (0,0) 种植灵芝，预计 4 小时后成熟。
  ```

#### 5.2.2 建造指令详解

* **可建造地块类型**：
    - `灵田`：种植灵药，产出丹药素材
    - `兽栏`：养殖灵兽，提供战力加成
    - `阵眼`：提供防御，多个相连触发连环阵
* **建造消耗**：根据地块类型和等级消耗不同资源（灵石、材料等）。
* **示例流程**：
  ```
  玩家：#建造 1,2 阵眼
  地灵：已在 (1,2) 建造阵眼（Lv.1），消耗灵石×100。当前防御力 +100。
  ```

#### 5.2.3 献祭指令详解

* **献祭规则**：
    - 只能献祭装备类物品（武器、防具、饰品）。
    - 转化率仅由物品品质和等级决定（详见第3章公式）。
    - 献祭后物品永久消失。
* **批量献祭**：`#献祭 <品质>` 会献祭背包中所有对应品质的装备。
* **示例流程**：
  ```
  玩家：#献祭 铁剑
  地灵：献祭成功！铁剑 (Lv.5 绿色) 转化为 150 灵气。当前灵气：1000/2000。
  
  玩家：#献祭 all
  地灵：批量献祭完成！共献祭 8 件白色装备，获得 320 灵气。当前灵气：1320/2000。
  ```

### 5.3 管理类指令

| 指令      | 功能说明             | 示例                    |
|---------|------------------|-----------------------|
| `#福地自动` | 切换地灵自动管理模式（开/关）  | `#福地自动 开` 或 `#福地自动 关` |
| `#福地注灵` | 为好友福地注入灵气        | `#福地注灵 道号 50`         |
| `#福地访问` | 访问好友福地           | `#福地访问 道号`            |
| `#唤醒福地` | 手动解除蛰伏模式（通常自动解除） | `#唤醒福地`               |

#### 5.3.1 自动管理模式

* **开启自动**：地灵会自动执行以下操作（消耗精力）：
    - 收割成熟的灵田
    - 喂养饥饿的灵兽
    - 用背包中的垃圾装备补充灵气（当灵气 < 50% 时）
* **关闭自动**：所有操作需玩家手动指令。
* **默认状态**：福地创建后自动开启。

### 5.4 指令与 LLM 交互的关系

* **指令优先**：当玩家使用明确的指令（如 `#种植 0,0 灵芝`）时，直接执行指令逻辑，不调用 LLM。
* **自然语言解析**：当玩家使用 `@地灵` + 自然语言时，调用 LLM 解析意图，可能转换为对应的指令执行。
* **混合使用示例**：
  ```
  玩家：@地灵 帮我把中间那颗药收了
  AI地灵：（LLM 解析为"收获坐标 (1,1) 的灵田"）
  地灵：已收获 (1,1) 的灵芝，获得 灵芝×3。精力 -5。
  ```

## 附录A：核心公式汇总

### A.1 灵气消耗

- 每小时消耗：`10 + 地块数量 × 2 + 核心等级 × 5`

### A.2 献祭收益

- 献祭收益：`物品价值 × 品质乘数 × (1 + 物品等级/10)`

### A.3 生长速度

- 实际生长时间：`基础生长时间 / 生长速度修正系数`
- 生长速度修正：`Π(相邻地块修正系数)`
    - 相生：1.2x
    - 相同：1.0x
    - 相克：0.8x

### A.4 产量计算

- 最终产量：`(1-3随机) + (核心等级 / 5)`

### A.5 天劫周期

- 下次天劫：`上次渡劫时间 + 7天`

### A.6 天劫攻防

- 天劫攻击力：`玩家等级 × 100 + 福地规模系数 × 200`
- 规模系数：3x3=1, 4x4=1.5, 5x5=2
- 福地总防：`Σ(阵眼耐久度) + Σ(灵兽战力评分) + 玩家STR×10 + 地灵等级×50`

### A.7 灵气上限

- 灵气上限：`1000 + 聚灵核心等级 × 200`

### A.8 蛰伏模式

- 触发条件：离线 ≥ 24小时 且 当前灵气 < 灵气上限×10%
- 蛰伏消耗：`1 灵气/小时`
