# 地灵 MBTI 人格化对话系统 - 使用指南

## 📋 功能概述

已成功接入 LLM API，实现地灵的 MBTI 人格化对话和智能操作执行。主要特性：

* 16 种 MBTI 人格差异化对话：每种人格有独特的对话风格和语气
* Function Calling 智能决策：LLM 根据福地状态自主决定调用工具
* 完整上下文感知：提供福地详细状态（灵气、网格、情绪等）
* 单次会话无记忆：降低 Token 消耗
* Function Calling：Spring AI 自动处理工具调用
* 形态阶段适：根据地灵阶段提供不同详细程度的上下文

## 🎯 使用方式

### 1. Function Calling 智能交互（推荐）

玩家发送：

```
地灵 帮我把中间那颗药收了
```

系统流程：

1. 构建 Prompt：包含福地完整状态（灵气、网格详情、情绪等）
2. LLM 分析：根据用户输入和福地状态，自主判断需要执行的操作
3. 工具调用：LLM 决定调用 harvestCrop("1,1") 工具
4. 自动执行：Spring AI 自动执行收获操作
5. 人格化回复：LLM 根据执行结果生成个性化回复

**示例回复**：

- **INTJ**: "根据分析，中央灵田已成熟。执行收获...获得灵芝×3，产量符合预期。"
- **ESFP**: "好哒好哒！我已经把中间的灵药收好啦～哇！获得了3份灵芝！超棒的！✨"
- **INFJ**: "我能感受到灵田的成熟气息...已经为你收获了，希望这些灵芝能帮到你 🌙"

### 2. 模糊指令处理

玩家发送：

```
地灵 帮我随便种点东西
```

系统流程：

1. LLM 理解"随便"的含义，自主选择空闲位置（如 "1,1"）
2. LLM 选择默认作物（如 "灵草"）
3. 调用 plantCrop("1,1", "灵草") 工具
4. 执行种植并返回人格化回复

**优势**：无需硬编码规则，LLM 根据常识和福地状态智能推断参数

### 3. 纯对话模式

玩家发送：

```
地灵 今天福地怎么样？
```

地灵会根据当前福地状态和 MBTI 人格回复，不调用任何工具。

## ⚙️ 配置说明

### application.yml 关键配置

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY:sk-placeholder}  # LLM API 密钥
      base-url: ${OPENAI_BASE_URL:https://api.deepseek.com}  # API 地址
      chat:
        options:
          model: ${OPENAI_MODEL:deepseek-chat}  # 模型名称
          temperature: 0.7  # 创造性 (0.0-1.0)
          max-tokens: 1000  # 最大 Token 数

xiantao:
  spirit:
    max-tokens: 800  # 地灵单次会话最大 Token
    enable-fallback: true  # 降级开关（LLM 不可用时回退规则匹配）
```

### 环境变量

使用时需设置：

```bash
export OPENAI_API_KEY="your-api-key-here"
export OPENAI_BASE_URL="https://api.deepseek.com"  # 或其他 OpenAI 兼容 API
export OPENAI_MODEL="deepseek-chat"
```

## 🔧 技术架构

### 核心设计理念

### 核心组件

| 组件                    | 说明                                 |
|-----------------------|------------------------------------|
| SpringAiConfig        | Spring AI ChatClient 配置            |
| SpiritPromptTemplates | MBTI 人格化 Prompt 模板                 |
| SpiritChatService     | 地灵对话核心服务，管理 Function Calling 流程    |
| SpiritTools           | Function Calling 工具函数集合（种植、收获、建造等） |
| UserContext           | ThreadLocal 用户上下文，传递 userId 给工具函数  |
| FudiCommandHandler    | 命令处理器，处理自然语言交互入口                   |

### Function Calling 工作流程

```
玩家消息
  ↓
FudiHandle.handleFudiSpirit()
  ↓
FudiCommandHandler.handleSpiritChat()
  ↓
SpiritChatService.processSpiritInteraction()
  ├─ 1. 设置用户上下文（UserContext.setCurrentUserId）
  │
  ├─ 2. 获取福地完整状态
  │    └─ 灵气、等级、精力、好感度
  │    └─ 网格详细布局（每个地块的类型、作物、生长进度等）
  │    └─ 地灵情绪状态
  │
  ├─ 3. 构建 Function Calling Prompt
  │    └─ MBTI 人格特征和对话风格
  │    └─ 福地完整状态信息
  │    └─ 可用工具列表及说明
  │    └─ 重要规则和示例
  │
  ├─ 4. 调用 LLM（带 tools 参数）
  │    └─ LLM 分析用户输入和福地状态
  │    └─ LLM 决定调用哪个工具（或不调用）
  │    └─ LLM 生成工具调用参数
  │
  ├─ 5. Spring AI 自动执行工具
  │    └─ 从 UserContext 获取 userId
  │    └─ 调用对应的 SpiritTools 方法
  │    └─ 执行业务逻辑（种植、收获等）
  │    └─ 返回执行结果
  │
  ├─ 6. LLM 生成人格化回复
  │    └─ 根据工具执行结果
  │    └─ 结合 MBTI 人格和情绪状态
  │    └─ 生成最终回复
  │
  └─ 7. 清理用户上下文（UserContext.clear）
  ↓
回复玩家
```

### 关键设计要点

**1. 用户上下文传递**

- 使用 ThreadLocal 在请求线程中传递 userId
- 在 processSpiritInteraction 开始时设置
- 在 finally 块中清除，防止内存泄漏
- SpiritTools 中的工具函数通过 UserContext 获取 userId

**2. 福地状态构建**

- 为 LLM 提供完整的网格详情，而非简单摘要
- 包含每个地块的位置、类型、名称
- 包含作物信息（名称、生长进度、是否可收获）
- 包含灵兽信息（名称、饥饿值）
- LLM 基于完整信息做出更准确的决策

**3. Prompt 模板设计**

- 明确列出所有可用工具及其参数
- 提供清晰的使用规则和约束
- 包含典型示例（特别是模糊指令的处理）
- 强调 LLM 应自主推断合理参数

**4. 工具函数设计**

- 使用 @Tool 注解标记可调用函数
- 使用 @ToolParam 描述参数含义
- 使用 @Description 说明工具用途
- 返回结构化的响应对象（Record 类型）
- 包含成功标志、消息、相关数据

### 中期优化

1. **对话历史** - 可选启用短期记忆（最近 3-5 条对话）
2. **情感演化** - 根据地灵好感度调整对话风格
3. **主动建议** - 地灵主动提示福地优化建议（如灵气不足、布局不合理）
4. **多模型支持** - 支持切换不同 LLM 提供商
5. **工具反馈优化** - 提供更详细的工具执行结果，帮助 LLM 生成更好的回复

### 长期优化

1. **地灵立绘** - 阶段三化形后接入图像生成
2. **语音合成** - 不同 MBTI 人格的语音风格
3. **羁绊系统** - 玩家长期互动解锁专属对话
4. **地灵成长** - MBTI 人格微调（非重塑）
5. **多轮对话** - 支持复杂的任务分解和多步操作
