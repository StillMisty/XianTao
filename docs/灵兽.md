# 灵兽系统设计

## 1. 数据架构

### 1.1 单一数据源原则

灵兽属性 **仅存储在 `xt_beast` 表**，兽栏地块 (`FudiCell.config`) 只存兽栏专属数据，通过 `beast_id` 联表引用。

```
xt_beast                   ← 灵兽全部属性（唯一数据源）
  └── id (PK)

FudiCell.config            ← 兽栏配置（PenConfig 记录）
  ├── beast_id → Beast.id     (联表引用)
  ├── template_id             (灵兽卵模板ID，用于产出配置)
  ├── hatch_time              (孵化开始时间)
  ├── mature_time             (孵化完成时间)
  ├── production_stored       (累积产出列表 JSONB)
  └── last_production_time    (产出计时)
```

### 1.2 Beast 表结构 (xt_beast)

| 字段                            | 类型           | 说明                                       |
|-------------------------------|--------------|------------------------------------------|
| `id`                          | BIGSERIAL    | PK                                       |
| `user_id`                     | BIGINT       | 所属用户                                     |
| `fudi_id`                     | BIGINT       | 所属福地                                     |
| `template_id`                 | BIGINT       | 灵兽卵模板ID                                  |
| `beast_name`                  | VARCHAR(128) | 灵兽名称                                     |
| `tier`                        | INT          | 等阶 (1-5)                                 |
| `quality`                     | VARCHAR(32)  | 品质 (mortal/spirit/immortal/saint/divine) |
| `is_mutant`                   | BOOLEAN      | 是否变异                                     |
| `mutation_traits`             | JSONB        | 变异特性列表                                   |
| `level`                       | INT          | 等级                                       |
| `exp`                         | INT          | 经验值                                      |
| `attack`                      | INT          | 攻击力                                      |
| `defense`                     | INT          | 防御力                                      |
| `max_hp`                      | INT          | 最大HP                                     |
| `hp_current`                  | INT          | 当前HP                                     |
| `skills`                      | JSONB        | 技能列表（技能ID 数组，`List<Long>`）              |
| `is_deployed`                 | BOOLEAN      | 是否出战                                     |
| `recovery_until`              | TIMESTAMP    | 恢复截止时间（休养中），可为 null；过期后自动清除           |
| `level_cap`                   | INT          | 等级上限 (= tier × 10 + 10)                  |
| `penned_cell_id`              | INT          | 所在兽栏地块ID（null = 栏外休憩）                    |
| `birth_time`                  | TIMESTAMP    | 出生时间                                     |
| `evolution_count`             | INT          | 进化次数                                     |
| `create_time` / `update_time` | TIMESTAMP    | 时间戳                                      |

### 1.3 与兽栏的关系

- 灵兽卵在兽栏中孵化，孵化完成后生成 `xt_beast` 记录
- 灵兽通过 `penned_cell_id` 关联到具体的兽栏地块
- `penned_cell_id = null` 表示灵兽在栏外休憩（兽栏被拆除时）
- 兽栏地块通过 `config.beast_id` 引用 Beast 实体

## 2. 灵兽品质 (BeastQuality)

| 品质            | 属性倍率 | 产出倍率 | 恢复分钟 | 孵化权重 |
|---------------|------|------|------|------|
| 凡品 (mortal)   | 0.8× | 1.0× | 30   | 900  |
| 灵品 (spirit)   | 1.0× | 1.2× | 60   | 80   |
| 仙品 (immortal) | 1.3× | 1.5× | 120  | 18   |
| 圣品 (saint)    | 1.6× | 2.0× | 240  | 2    |
| 神品 (divine)   | 2.0× | 3.0× | 480  | 0    |

神品孵化权重为 0，仅可通过品质突破（圣品→神品）获得。
恢复分钟：灵兽 HP 归零后自动进入休养状态，该时长后 `recoveryUntil` 过期，可重新出战。

## 3. 核心公式

### 3.1 经验与等级

孵化消耗 = tier × 200 + 200 灵石
孵化时间 = (24 + tier × 8)h / 兽栏等级加速系数
产出间隔 = 4h / 兽栏等级加速系数
每轮产出 = tier × 品质产出倍率 × (1 + rand(0,tier)) / 2 件
HP最大值 = tier × 200 + (level - 1) × tier × 20
恢复消耗 = ceil(缺失HP × 0.1) 灵石

升级所需经验 = 50 × level^1.5
等级上限 = tier × 10 + 10
攻击 = round((10 + (level - 1) × 3 × q) × q)     (q = 品质属性倍率)
防御 = round((8 + (level - 1) × 2 × q) × q)
速度 = level × 2 + 8

### 3.2 属性重算触发时机

| 时机 | 动作                               |
|----|----------------------------------|
| 升级 | 攻防按公式重算                        |
| 升阶 | tier 加 1, levelCap 更新, 属性重算    |
| 升品 | quality 升级, 品质倍率 q 变化, 属性重算 |

## 4. 灵兽状态

### 4.1 状态机

```
孵化中 → 正常（在栏） → 出战 → 战斗受伤 → 回栏休养 → 正常（在栏）
                ↓
            栏外休憩（兽栏拆除）
```

### 4.2 状态说明

| 状态     | 条件                                    | 产出 | 天劫防御 | HP恢复   |
|--------|---------------------------------------|----|------|--------|
| 孵化中    | `cell.matureTime` 在未来                  | ✗  | ✗    | —      |
| 正常（在栏） | HP满，非出战                               | ✓  | ✗    | —      |
| 出战     | `beast.is_deployed = true`            | ✗  | ✓    | —      |
| 休养中    | HP < max，或 `beast.recoveryUntil` 未过期 | ✗  | ✗    | 消耗灵石恢复 |
| 栏外休憩   | `beast.penned_cell_id = null`         | ✗  | ✗    | 可出战    |

### 4.3 出战规则

- **出战上限**：出战上限由玩家等级决定：`min(3, level/5 + 1)`。等级 1 可出战 1 只，等级 5 可出战 2 只，等级 10 及以上可出战 3 只。
- **可出战略**：满足以下全部条件时方可出战：is_deployed 为 true, hp_current 大于 0, 且 recovery_until 为 null 或已过期
- **recovery_until**：可为 null（表示无休养状态），过期后自动视为可出战；检测到已过期时自动将其清为 null
- **HP归零**：灵兽自动回栏，`recoveryUntil` 设置为 now + 品质恢复分钟数
- **恢复**：消耗灵石瞬间回满 HP，同时清除 `recoveryUntil`
- **召回**：手动召回出战灵兽，回栏后立即恢复产出（若 HP 满）

### 4.4 栏外休憩

拆除兽栏时灵兽 **不会消失**，而是退为"栏外休憩"状态（`penned_cell_id = null`）：

- 不可生产，但 **可出战战斗**
- 新建兽栏后可重新入栏
- 栏外休憩的灵兽 **不计入** 出战上限

## 5. 进化与品质突破

### 5.1 前置条件

进化前灵兽必须达到 **等级上限**（`level >= levelCap`），否则无法进化/突破。

### 5.2 进化（升阶）

- 消耗：1 进化石 + `(tier+1) × 200` 灵石 + 8 地灵精力
- 成功率：85% + 好感加成（最高+15%）
- 10%概率连带品质升级
- 失败：进化石+灵石不退
- 效果：tier 加 1, levelCap 更新, 属性重算, HP 回满, 解锁先天技能

### 5.3 品质突破（升品）

- 消耗：2 进化石 + `nextQuality.order × 300` 灵石 + 8 地灵精力
- 成功率：60% + 好感加成（最高+20%）+ 灵悟特性(+10%)
- 失败：进化石+灵石不退
- 效果：quality 升级, 属性重算, HP 回满, 解锁先天技能

## 7. 兽栏产出机制

### 7.1 设计定位

兽栏产出是 **稳定、持续的低/中阶药材来源**，与炼丹系统形成闭环。高阶药材通过灵田种植、地图探索等途径获取，兽栏不越位。

### 7.2 产出物配置

产出物由灵兽卵模板的 `properties.production_items` 决定，不同品种产出不同药材：

| 等阶      | 产出物品阶       | 示例       |
|---------|-------------|----------|
| Tier 1  | 凡阶药材        | 灵草、黄精    |
| Tier 2  | 凡阶药材 + 少量灵阶 | 灵草、紫芝    |
| Tier 3  | 灵阶药材        | 紫芝、碧灵花   |
| Tier 4+ | 灵阶药材        | 碧灵花、九转灵参 |

**灵兽卵模板配置示例**：

```json
{
  "grow_time": 24,
  "production_items": [
    {"template_id": 1, "name": "灵草", "weight": 70},
    {"template_id": 2, "name": "黄精", "weight": 30}
  ],
  "skill_pool": {
    "innate_skills": [
      {"skill_id": 101, "unlock": "birth"},
      {"skill_id": 102, "unlock": "tier_2"},
      {"skill_id": 103, "unlock": "tier_3"},
      {"skill_id": 104, "unlock": "tier_4"},
      {"skill_id": 105, "unlock": "tier_5"},
      {"skill_id": 106, "unlock": "quality_break"}
    ],
    "awakening_skills": [
      {"skill_id": 110, "weight": 50},
      {"skill_id": 111, "weight": 30}
    ]
  }
}
```

每轮产出时，从 `production_items` 中按权重随机选取物品。产出配置通过 `cell.config.template_id` 查找灵兽卵模板获得。

### 7.3 产出流程

1. **累积阶段**：按产出间隔计算本轮产出件数（tier/品质/变异特性由 Beast 实体读取），写入 `FudiCell.config.production_stored`
2. **收取阶段**：玩家执行 `福地收取` 时，将物品实际写入背包（`StackableItem`）
3. **背包满**：产出暂存在 `production_stored`，不消失，下次收取时一并发放
4. **上限**：`production_stored` 最多累积 `tier × 20` 件，超出不再产出

### 7.4 变异特性影响

| 变异 | 代码             | 效果                |
|----|----------------|-------------------|
| 高产 | `high_yield`   | 每轮产出件数 ×1.3       |
| 稀产 | `rare_produce` | 每轮 5% 概率额外产出高品阶物品 |

### 7.5 产出存储格式

`production_stored` 存储为物品列表：

```json
[
  {"template_id": 1, "name": "灵草", "quantity": 5},
  {"template_id": 2, "name": "黄精", "quantity": 2}
]
```

## 8. 等级成长

### 8.1 经验获取

| 来源       | 经验值         | 说明            |
|----------|-------------|---------------|
| 出战战斗（每场） | `怪物等级 × 10` | 鼓励带灵兽历练       |
| 历练结算     | `历练分钟 × 2`  | 挂机也有成长        |
| 灵兽精华喂养   | `精华 × 50`   | 放生灵兽获得精华，主动培养 |

### 8.2 升级公式

升级所需经验 = 50 × level^1.5
等级上限 = tier × 10 + 10

| 等阶     | 等级上限 |
|--------|------|
| Tier 1 | 20   |
| Tier 2 | 30   |
| Tier 3 | 40   |
| Tier 4 | 50   |
| Tier 5 | 60   |

### 8.3 属性成长

攻击 = round((10 + (level - 1) × 3 × q) × q)
防御 = round((8 + (level - 1) × 2 × q) × q)
HP最大值 = tier × 200 + (level - 1) × tier × 20
速度 = level × 2 + 8

其中 `q = 品质属性倍率`（凡品 0.8, 灵品 1.0, 仙品 1.3, 圣品 1.6, 神品 2.0）。

每升一级：攻防按公式增长，HP +tier×20（线性）。

### 8.4 进化与等级

**进化保留等级**，进化本身提供额外的 tier 跃升加成：

- 升阶后等级不变，等级上限随 tier 提升
- 升品后等级不变，属性倍率 q 提升
- 属性按新 tier/q 重新计算
- 若当前等级超过新上限，等级不变但无法继续获取经验

## 9. 技能系统

### 9.1 设计理念

每个灵兽品种有自己的 **技能池**，技能与品种主题一致（火系灵兽配火系技能）。技能通过 **进化解锁** 和 **战斗觉醒** 获取，不使用道具学习。

灵兽实体中的技能字段 `skills` 存储为技能 ID 列表（`List<Long>`），组建战斗队伍时通过技能仓库按 ID 批量加载技能完整数据。

### 9.2 技能池配置

每个灵兽模板配置两组技能池：

**先天技池** — 进化解锁，稳定获取：

```json
{
  "innate_skills": [
    {"skill_id": 101, "unlock": "birth"},
    {"skill_id": 102, "unlock": "tier_2"},
    {"skill_id": 103, "unlock": "tier_3"},
    {"skill_id": 104, "unlock": "tier_4"},
    {"skill_id": 105, "unlock": "tier_5"},
    {"skill_id": 106, "unlock": "quality_break"}
  ]
}
```

**后天悟池** — 战斗觉醒，随机获取：

```json
{
  "awakening_skills": [
    {"skill_id": 110, "weight": 50},
    {"skill_id": 111, "weight": 30},
    {"skill_id": 112, "weight": 20}
  ]
}
```

### 9.3 先天技 — 进化解锁

灵兽进化时自动解锁对应技能，无需概率判定：

| unlock 条件       | 含义            |
|-----------------|---------------|
| `birth`         | 孵化即拥有         |
| `tier_2`        | 升阶到 Tier 2 解锁 |
| `tier_3`        | 升阶到 Tier 3 解锁 |
| `tier_4`        | 升阶到 Tier 4 解锁 |
| `tier_5`        | 升阶到 Tier 5 解锁 |
| `quality_break` | 品质突破时解锁       |

当前服务端仅在孵化（birth）、升阶（tier_2 / tier_3）和品质突破（quality_break）时触发解锁调用；tier_4 和 tier_5 的 unlock 条件为配置预留，可在模板中预定义，待后续进化链扩展时启用。

### 9.4 后天悟 — 战斗觉醒

在战斗高光时刻，灵兽有概率从后天悟池中随机领悟一个技能。

**触发条件**（满足其一）：

- 灵兽在战斗中击杀最后一击的怪物
- 灵兽 HP 降至 0 但战斗最终胜利
- 战斗被标记为高光战斗（势均力敌、回合数多）

**觉醒概率**：15%（每场战斗最多判定一次）

**觉醒流程**：按权重从后天悟池中随机选取一个未拥有的技能，技能上限满 4 个时不再觉醒。

### 9.5 技能上限与替换

- 灵兽技能上限：**4 个**
- 满 4 个时无法再解锁新技能（包括进化解锁和战斗觉醒）
- 放生灵兽可获得灵兽精华，用于喂养其他灵兽

### 9.6 技能成长路线

孵化 → 先天技 ×1 (birth)
  │
  ├─ 升阶 Tier 2 → 先天技 ×1
  ├─ 升阶 Tier 3 → 先天技 ×1
  ├─ 升阶 Tier 4 → 先天技 ×1
  ├─ 升阶 Tier 5 → 先天技 ×1
  │
  ├─ 战斗觉醒 → 后天悟 ×0~1 (概率)
  │
  └─ 品质突破 → 先天技 ×1

一只完全体灵兽最多可通过进化获得 **6 次先天技解锁机会**，但技能上限为 4 个，实际承载技能数受上限约束。后天悟最多 1 个。
