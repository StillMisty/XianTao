# 宗门系统设计

## 1. 概述

宗门是玩家可加入的持久性组织。**宗灵**是宗门本身的灵性化身，LLM 驱动，玩家通过自然语言与宗灵对话来执行所有宗门操作。师徒关系必须绑定在同一宗门内。

### 设计原则

- **文字挂机 MUD**：所有交互异步，无实时通信、无定时任务
- **宗灵统一入口**：除创建外，所有宗门事务通过 `宗灵 [自然语言]` 完成
- **零玩家间交易**：宗门资金/贡献均为玩家↔系统流转
- **师徒同宗**：双方必须在同一宗门（或同为散修），师傅宗门状态变化自动同步徒弟
- **退宗遗忘**：从宗门习得的共享功法，退宗后自动遗忘，防止功法外流

---

## 2. 宗灵

### 2.1 定义

宗灵不是长老、不是 NPC，**是宗门本身的意志体现**。创建宗门时 LLM 赋予其灵性——剑宗的宗灵可能是一柄傲雪古剑的剑意，丹宗的宗灵可能是一缕药香凝成的老叟。玩家跟宗灵说话，就是在跟宗门本身交流。

### 2.2 宗灵 vs 地灵

| | 地灵 | 宗灵 |
|---|---|---|
| 身份 | 个人福地守护灵 | 宗门意志化身 |
| 范围 | 单人 | 全宗共享 |
| 情绪 | MBTI + 好感度驱动 | 无个体情绪，反映宗门状态 |
| 工具 | 操作福地（种植/建造/收获） | 管理宗门事务 |
| 记忆 | 与单人的对话历史 | 统一 `xt_chat_history` 表，按成员分别记录（ChatType=SECT，窗口10条） |

### 2.3 宗灵操作范围

宗门概况（名称、等级、资金、人数、事件、公告）与个人贡献值在对话时**注入 system prompt**，不走 tool。

其余操作通过 Tool Calling（Spring AI `@Tool`），根据调用者职位**选择性注册**：

| 权限 | 工具 | 说明 |
|------|------|------|
| 全员 | `queryShop` | 查看贡献商店商品列表 |
| 全员 | `exchangeShopItem` | 用贡献兑换商品 |
| 全员 | `querySharedSkills` | 查看共享功法列表 |
| 全员 | `learnSharedSkill` | 消耗贡献学习共享功法 |
| 全员 | `submitSkillJade` | 提交背包中的功法玉简到宗门，获 300 贡献 |
| 全员 | `donateStones` | 捐献灵石 → 贡献（灵石 × 0.1）+ 宗门资金等额 |
| 全员 | `queryTasks` | 查看当前宗门事件任务 |
| 长老+ | `inviteMember` | 邀请散修玩家入宗 |
| 长老+ | `kickMember` | 踢出弟子 |
| 长老+ | `postNotice` | 发布/更新宗门公告 |
| 长老+ | `removeSharedSkill` | 下架共享功法（LISTED → PENDING，已学者不受影响） |
| 长老+ | `listSharedSkill` | 上架共享功法（PENDING → LISTED） |
| 宗主 | `appointMember` | 任命/罢免长老（含宗主传承） |
| 宗主 | `upgradeSect` | 消耗宗门资金提升宗门等级 |
| 宗主 | `expandMembers` | 消耗宗门资金扩充成员上限（+5） |
| 宗主 | `queryBuildings` | 查看建筑状态 |
| 宗主 | `buildStructure` | 消耗宗门资金建造建筑 |
| 宗主 | `upgradeBuilding` | 消耗宗门资金升级建筑 |
| — | `dismissSect` | 解散宗门（保留为命令，不走宗灵） |

### 2.4 保留的命令

| 指令 | 原因 |
|------|------|
| `宗门创建 [名称] [道统描述?]` | 无宗门无宗灵 |
| `宗门退出` | 急迫操作不宜对话 |
| `宗门解散` | 毁灭性操作需明确指令 |

玩家只能被邀请加入宗门，不可自行申请。

## 3. 宗门创建（LLM 注入）

### 3.1 创建方式

| 类型 | 说明 |
|------|------|
| **玩家创建** | 达到金丹期 + 消耗 5000 灵石 + 可选道统描述 |

### 3.2 创建流程

```
玩家：宗门创建 青云宗 以剑入道，追求一击必杀
  ↓
① 验证境界 ≥ 金丹期、未在宗门、名称唯一
② 扣减 5000 灵石
③ LLM 根据道统描述生成宗灵身份：
   - 宗门诗号（4-7 言对仗句）
   - 道统简述（100 字内）
   - 宗灵人格种子（≤50 字）
④ 若无道统描述，LLM 据宗门名生成默认
⑤ 创建宗门（Lv.1，成员上限 10），玩家任宗主
⑥ 初始资金池 = 2000 灵石
⑦ 返回诗号 + 创建成功信息
```

## 4. 层级结构

宗门设三级职位，由上至下：

| 职位 | 数量 | 权限 |
|------|------|------|
| **宗主** | 1 人 | 全部权限 |
| **长老** | 不限制 | 邀请/踢出成员、发布公告、下架功法 |
| **弟子** | — | 贡献商店兑换、学习/提交功法 |

### 4.1 权限矩阵

通用规则：上任可管理下任（只能处置比自己职位低的成员）。

| 操作 | 宗主 | 长老 | 弟子 |
|------|:----:|:----:|:----:|
| 任命/罢免（含传承） | ✅ | — | — |
| 解散宗门 | ✅ | — | — |
| 宗门升级 | ✅ | — | — |
| 宗门扩容 | ✅ | — | — |
| 建造/升级建筑 | ✅ | — | — |
| 发布公告 | ✅ | ✅ | — |
| 踢出成员 | 下任 | 下任 | — |
| 邀请成员 | ✅ | ✅ | — |
| 下架功法 | ✅ | ✅ | — |
| 上架功法 | ✅ | ✅ | — |
| 贡献商店 | ✅ | ✅ | ✅ |
| 共享功法 | ✅ | ✅ | ✅ |

> "下任" = 可管理自己以下职位的成员，不可管理同职位或上级。宗主不可被踢出。

## 5. 成员管理

### 5.1 成员上限

- **初始上限**：10 人
- **升级扩容**：每次升级自动 +5
- **独立扩容**：消耗 `5 × 500 = 2500` 灵石增加 5 个槽位（通过宗灵）

### 5.2 加入方式

| 方式 | 说明 | 入口 |
|------|------|:----:|
| 受邀加入 | 有邀请权限的成员通过宗灵邀请 | 宗灵 |

玩家不可自行申请加入宗门。

### 5.3 师徒绑定规则

| 场景 | 规则 |
|------|------|
| **建立师徒** | 双方必须在同一宗门，或同为散修 |
| **师傅加入宗门** | 所有徒弟自动跟随加入 |
| **师傅退出/被踢/解散** | 所有徒弟自动退出宗门（视为叛师），受 24h 冷却 |

### 5.4 主动退出

- 可通过 `宗门退出` 命令或宗灵对话退出
- **惩罚**：贡献清零，24h 冷却，**已学共享功法自动遗忘**

### 5.5 踢出成员

- 宗主可踢出长老和弟子，长老可踢出弟子
- 不可踢出同职位或上级（宗主不可被踢）
- 被踢出者贡献清零，24h 冷却，**已学共享功法自动遗忘**

## 6. 贡献值体系

### 6.1 个人贡献值

| 获取途径 | 贡献值 | 说明 |
|---------|--------|------|
| 捐献灵石 | 捐献量 × 0.1 | 同时增加宗门资金等额 |
| 提交功法玉简 | 300/份 | 一次性奖励，消耗玉简 |
| 完成宗门事件任务 | 按任务奖励 | 事件触发后自动生成的任务 |

### 6.2 宗门资金池

- 来源：成员捐献
- 用途：宗门升级、成员扩容、**宗门建筑建造/升级**

## 7. 宗门建筑

宗门资金的核心消耗渠道。每栋建筑可独立建造和升级。

### 7.1 建筑列表

| 建筑 | 代码 | 效果（每级） | 建造 | 满级 | 满级总耗 | 联动系统 |
|------|------|------|-----:|:----:|------:|------|
| **藏经阁** | SCRIPTURE_PAVILION | +3 共享功法位 | 3,000 | 5 | 15,000 | 法决 |
| **练功房** | TRAINING_ROOM | 修炼经验 +3% | 2,000 | 5 | 10,000 | 历练 |
| **炼丹房** | ALCHEMY_CHAMBER | 炼丹品质评分 +5% | 2,000 | 3 | 6,000 | 炼丹 |
| **灵脉** | SPIRIT_VEIN | 日产灵石 100 注入宗门资金 | 5,000 | 3 | 15,000 | 经济 |
| **锻造坊** | FORGE_WORKSHOP | 强化费用 -5% | 2,000 | 3 | 6,000 | 装备强化 |
| **护阵** | GUARD_ARRAY | 受伤 -3% | 4,000 | 3 | 12,000 | 战斗 |
| **药园** | HERB_GARDEN | 解锁宗门专属灵药（商店上架） | 3,000 | 3 | 9,000 | 丹药/福地 |

### 7.2 建筑规则

- 建筑等级独立于宗门等级，未建造时表中无行
- 宗主消耗宗门资金建造/升级（每次升级费用 = 建造费用）
- 效果对所有宗门成员自动生效，实时读取
- 建筑总费用为宗门资金提供长期消耗目标

### 7.3 特殊建筑规则

**灵脉**：
- 惰性结算，查看宗门资金时计算距上次结算时间的产出（`小时 × 等级 × 100 / 24`）
- 需在 `xt_sect` 新加 `last_vein_payout` 字段记录上次结算时间

**药园**：
- 建造时宗主选择灵药路线（疗伤系/修炼系/突破系/毒系），对应商品上架到贡献商店
- 后续可消耗贡献转系
- 升级自动追加更高阶商品

**炼丹房**：
- 品质加成作用于炼丹评分（`baseScore × (1 + 等级 × 0.05)`），提高产出上成品概率

## 8. 功法传授

### 8.1 流程

```
玩家 A：宗灵 我要提交功法
  → 消耗背包中的功法玉简
  → 功法入库（状态 = PENDING），玩家 A 获得 300 贡献

长老/宗主：宗灵 上架烈焰诀
  → 功法状态 PENDING → LISTED，全员可学

玩家 B：宗灵 我想学烈焰诀
  → 扣减玩家 B 贡献（贡献 = 功法 levelRequirement × 50）
  → 传授功法

长老/宗主：宗灵 下架烈焰诀
  → 功法状态 LISTED → PENDING，已学者不受影响

玩家 B 退出宗门
  → 系统自动删除玩家 B 已学的所有宗门共享功法
  → 功法原样留在宗门，下个弟子接着学
```

### 8.2 防泛滥规则

| 规则 | 说明 |
|------|------|
| **退宗遗忘** | 退宗/被踢后，该玩家所有共享功法自动删除 |
| **同一功法不可重复提交** | 已存在的功法不能再次提交（UIQUE sect+skill） |
| **宗门功法位上限** | 受藏经阁建筑等级控制（基础 3 + 每级 +3） |
| **消耗品提交** | 提交时消耗功法玉简，不是复制 |

## 9. 宗门事件与宗门任务

### 9.1 事件机制

复用 FudiEventGenerator 模式：时间门控（4-8h），在宗灵对话时惰性检查并生成。事件存储于 `xt_sect`（`last_event_type`、`last_event_text`、`last_event_time`、`event_expires_at`）。

### 9.2 事件类型

| 事件 | 效果 | 持续 |
|------|------|------|
| 灵脉喷涌 | 贡献获取 +50% | 到下次事件 |
| 妖兽围山 | 全员战斗属性 +5% | 2h |
| 秘境裂隙 | 秘境次数 +1（当日） | 当日 |
| 仙人授法 | 随机一成员免费学一功法 | 一次性 |

### 9.3 宗门任务

事件触发时自动生成一批任务，全员可接、独立完成。

| 任务类型 | 完成条件 | 奖励 |
|----------|----------|------|
| HUNT（狩猎） | 在对应区域挂机历练，自动统计击杀数 | 贡献值 |
| DONATE（捐献） | 上交指定物品到宗门 | 贡献值 |

**机制**：
- 每次新事件生成时，删除旧事件的所有任务及进度
- 每个成员独立进度（`xt_sect_task_progress`），先完成先得奖励
- 奖励贡献由系统产出，不扣宗门资金
- 成员通过宗灵 `queryTasks` 查看任务列表

### 9.4 展示

- 宗灵对话时以 system prompt 注入事件上下文
- 宗门概况（prompt 注入）中显示当前事件

## 10. 宗门升级

宗主消耗宗门资金提升宗门等级：

| 目标等级 | 消耗资金 | 效果 |
|:------:|---------|------|
| Lv.1 → Lv.2 | 5,000 | 成员上限 +5 |
| Lv.2 → Lv.3 | 15,000 | 成员上限 +5 |
| Lv.3 → Lv.4 | 30,000 | 成员上限 +5 |
| Lv.4 → Lv.5 | 50,000 | 成员上限 +5 |

等级上限 Lv.5。
