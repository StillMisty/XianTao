# 宗门系统设计

## 1. 概述

宗门是玩家可加入的持久性组织，提供贡献商店、宗门功法等核心收益。师徒关系必须绑定在同一宗门内，师傅的宗门归属决定徒弟的宗门归属。

### 设计原则

- **文字挂机 MUD**：所有交互异步，无实时通信、无定时任务
- **轻社交**：宗门是单人游戏的社交调味，非强制内容
- **零玩家间交易**：宗门资金/贡献均为玩家↔系统流转，不开启玩家间物品转移
- **师徒同宗**：建立师徒关系前双方必须在同一宗门（或同为散修），后续师傅的宗门状态变化自动同步影响徒弟

## 2. 宗门创建

### 2.1 创建方式

| 类型 | 说明 |
|------|------|
| **玩家创建** | 达到金丹期 + 消耗 5000 灵石 |

NPC 宗门为规划中功能。

### 2.2 创建流程

```
玩家：宗门创建 青云宗
  ↓
① 验证玩家境界 ≥ 金丹期
② 验证玩家未在宗门中、宗门名未被占用
③ 扣减 5000 灵石
④ 创建宗门（Lv.1，成员上限 10），玩家任宗主
⑤ 初始资金池 = 2000 灵石
⑥ 返回成功信息
```

## 3. 层级结构

宗门设五级职位，由上至下：

| 职位 | 数量 | 权限 |
|------|------|------|
| **宗主** | 1 人 | 全部权限：任命/罢免（含传承）、宗门升级、解散宗门、资金管理、发布公告、扩容 |
| **副宗主** | 最多 2 人 | 邀请/踢出成员、发布公告 |
| **长老** | 最多 5 人 | 邀请成员 |
| **精英** | 不限制 | 预留（可发布宗门任务） |
| **普通成员** | — | 贡献商店兑换、学习功法 |

## 4. 成员管理

### 4.1 成员上限

- **初始上限**：10 人
- **升级扩容**：每次升级自动 +5
- **独立扩容**：`宗门扩充` 命令，消耗 `5 × 500 = 2500` 灵石增加 5 个槽位

### 4.2 加入方式

| 方式 | 说明 | 状态 |
|------|------|:----:|
| 受邀加入 | 有邀请权限的成员发送邀请，目标直接加入 | ✅ |
| 申请加入 | 玩家主动申请，直接加入（无审核） | ✅ |
| NPC 宗门 | 规划中 | 📋 |

### 4.3 师徒绑定规则

师徒关系与宗门归属紧密绑定，规则如下：

| 场景 | 规则 |
|------|------|
| **建立师徒** | 双方必须在同一宗门，或同为散修（均不在任何宗门） |
| **师傅加入宗门** | 所有徒弟自动跟随加入同一宗门（忽略徒弟个人意愿） |
| **师傅退出宗门** | 所有徒弟自动退出宗门（视为叛师） |
| **师傅被踢出宗门** | 所有徒弟自动退出宗门（视为叛师），受 24 小时冷却惩罚 |
| **师傅解散宗门** | 所有成员（含师傅本人和所有徒弟）退出宗门，师徒关系受影响（视为叛师） |

### 4.4 主动退出

- 玩家可随时主动退出宗门
- **惩罚**：个人贡献值清零，24 小时内不可加入新宗门；若该玩家是师傅，其所有徒弟自动叛师并退出宗门，同步受 24 小时冷却惩罚
- 退出后宗门功法自动失效

### 4.5 踢出成员

- 宗主/副宗主可踢出成员（不可踢出宗主）
- 被踢出者贡献值清零，**24 小时内不可加入新宗门**
- 被踢出者若有徒弟，所有徒弟自动叛师并退出宗门

## 5. 贡献值体系

### 5.1 个人贡献值

个人贡献是成员在宗门内的唯一货币，不可转让、不可交易。

| 获取途径 | 贡献值 | 说明 |
|---------|--------|------|
| 捐献灵石 | 捐献量 × 0.1 | 同时增加宗门资金等额 |
| 宗门任务 | 10~300/次 | 规划中 |

### 5.2 宗门资金池

- 独立于个人贡献的宗门共有资产
- 来源：成员捐献
- 用途：宗门升级、成员上限扩容

## 6. 核心收益

### 6.1 宗门任务

### 6.2 贡献商店

成员消耗个人贡献值兑换物品的专属商店。

| 商品类型 | 贡献值 | 说明 |
|---------|--------|------|
| 各类可堆叠物品 | 按模板配置 | 物品直接发放到背包 |

**兑换流程**：扣减贡献 → 扣减库存 → 发放物品到背包（物品模板名 + 类型 + 数量 1）。

商城刷新为规划中功能（`宗门刷新商店` 预留）。

### 6.3 宗门功法

|------|------|:----------:|

| 聚灵诀 | 历练效率 +5% | Lv.2 |
| 通脉诀 | 突破成功率 +3% | Lv.3 |
| 护体诀 | 战斗受伤 -5% | Lv.4 |

## 7. 宗门升级

宗主消耗宗门资金提升宗门等级：

| 目标等级 | 消耗资金 | 效果 |
|:------:|---------|------|
| Lv.1 → Lv.2 | 5,000 | 成员上限 +5，解锁聚灵诀 |
| Lv.2 → Lv.3 | 15,000 | 成员上限 +5，解锁通脉诀 |
| Lv.3 → Lv.4 | 30,000 | 成员上限 +5，解锁护体诀 |
| Lv.4 → Lv.5 | 50,000 | 成员上限 +5，解锁天罡诀 |

等级上限为 Lv.5。

## 8. 宗主传承

- 宗主可使用 `宗门任命 [道号] LEADER` 将宗主之位传给任意成员
- 原宗主自动降为副宗主，新宗主获得全部权限
- 若副宗主已达上限（2 人），传承仍可执行

## 10. 数据模型

### 10.1 `xt_sect` — 宗门

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `name` | VARCHAR(32) | 宗门名（UNIQUE） |
| `leader_id` | BIGINT | 宗主用户 ID |
| `level` | INT | 宗门等级（1~5，默认 1） |
| `funds` | BIGINT | 宗门资金池 |
| `max_members` | INT | 成员上限（默认 10） |
| `description` | TEXT | 宗门简介 |
| `notice` | TEXT | 宗门公告 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

### 10.2 `xt_sect_member` — 宗门成员

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `sect_id` | BIGINT | FK → xt_sect（可 NULL，退宗冷却记录用） |
| `user_id` | BIGINT | FK → xt_user |
| `position` | VARCHAR(16) | 职位：LEADER/VICE_LEADER/ELDER/ELITE/MEMBER |
| `contribution` | INT | 个人贡献值（默认 0） |
| `joined_at` | TIMESTAMP | 加入时间 |
| `cooldown_until` | TIMESTAMP | 退宗冷却截止时间（NULL=无冷却） |
| `UNIQUE` | (user_id) | 每人同时只在一个宗门 |

> **冷却机制**：退宗/被踢后写入一条 `sect_id = NULL` 的记录，`cooldown_until` 设置为当前时间 + 24h。下次加入时检查此记录。

### 10.3 `xt_sect_skill` — 宗门功法（已解锁）

| 字段 | 类型 | 说明 |
|------|------|------|
| `sect_id` | BIGINT | FK → xt_sect |
| `skill_id` | BIGINT | FK → xt_skill |
| `unlocked_at` | TIMESTAMP | 解锁时间 |
| `UNIQUE` | (sect_id, skill_id) |

### 10.4 `xt_sect_shop_item` — 宗门商店商品

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | PK |
| `sect_id` | BIGINT | FK → xt_sect |
| `item_template_id` | BIGINT | FK → xt_item_template |
| `price_contribution` | INT | 贡献价格 |
| `stock` | INT | 库存（-1 为无限） |
| `last_refresh` | TIMESTAMP | 上次刷新时间 |
| `UNIQUE` | (sect_id, item_template_id) |

## 11. 玩家指令

| 指令 | 说明 | 要求 |
|------|------|------|
| `宗门` | 查看宗门总览（名称、等级、成员、资金、公告） | 已加入宗门 |
| `宗门创建 [名称]` | 创建宗门 | 金丹期 + 5000 灵石 + 无宗门 |
| `宗门邀请 [道号]` | 邀请玩家加入 | 有邀请权限 |
| `宗门申请 [宗门名]` | 申请加入宗门（直接加入） | 无宗门 |
| `宗门踢出 [道号]` | 踢出成员 | 有踢出权限 |
| `宗门退出` | 退出宗门（贡献清零，24h 冷却） | 已加入宗门 |
| `宗门任命 [道号] [职位]` | 任命职位 / 传承宗主（LEADER） | 宗主 |
| `宗门解散` | 解散宗门（所有成员退出） | 宗主 |
| `宗门公告 [内容]` | 发布公告 | 有发布权限 |
| `宗门捐献 [灵石数量]` | 捐献灵石（得 10% 贡献） | 已加入宗门 |
| `宗门升级` | 消耗资金提升宗门等级 | 宗主 |
| `宗门扩充` | 消耗资金扩充成员上限 +5 | 宗主 |
| `宗门刷新商店` | 刷新贡献商店（规划中） | 宗主 |
| `宗门商店` | 查看贡献商店 | 已加入宗门 |
| `宗门兑换 [商品编号]` | 用贡献兑换物品（物品直接入包） | 已加入宗门 |
| `宗门功法` | 查看宗门功法列表 | 已加入宗门 |

## 12. 开发流程

1. `domain/sect/` — Sect、SectMember entity + enums（SectPosition）
2. `db/migration/` — 建表语句（V1.0.55~58）
3. `infrastructure/mapper/` + `repository/` — Mapper + RepositoryImpl
4. `service/SectService` — 创建/加入/退出/商店/功法/升级/扩容/传承
5. `handle/command/SectCommandHandler` — CommandGroup 实现
6. `handle/listener/SectListener` — 双平台监听器
