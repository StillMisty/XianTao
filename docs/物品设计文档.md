### 物品设计文档：模板 (Template) vs 实例 (Instance)

在数据库/代码设计中，必须将物品分为两类处理：

1. **标品（可堆叠物品）**：材料、消耗品、种子。玩家背包里只记录 `[物品ID, 数量]`。
2. **非标品（波动装备）**：武器、防具。玩家背包里记录的是系统根据【装备模板】Roll出来的【装备实例UUID】，具有唯一性。

---

### 二、 装备波动系统设计 (RNG System)

装备的最终属性由三个维度随机生成：**基础波动率**、**品质前缀**、**随机词条（四维属性加成）**。

#### 1. 品质与基础波动公式

当一件装备（如：铁剑，模板Base_ATK = 20）掉落时，系统先根据挂机地图或怪物掉率，判定其**品质（Rarity）**：

| 品质颜色          | 品质系数 (RNG区间)  | 附加词条数 | 装备前缀示例  |
|:--------------|:--------------|:------|:--------|
| ⚪ **破旧 (白)**  | `0.8 - 0.99`  | 0 条   | 破烂的、生锈的 |
| 🟢 **普通 (绿)** | `1.0 - 1.15`  | 1 条   | 普通的、结实的 |
| 🔵 **稀有 (蓝)** | `1.16 - 1.35` | 1-2 条 | 锋利的、精工的 |
| 🟣 **史诗 (紫)** | `1.36 - 1.60` | 2-3 条 | 卓越的、附魔的 |
| 🟡 **传说 (金)** | `1.61 - 2.00` | 3-4 条 | 传说的、完美的 |

*计算公式*：`最终攻击力 = 模板基础攻击力 × 品质系数 (取整)`
*举例*：一把紫色的【铁剑】，系统Roll到的系数是 1.5，那么它的最终攻击力就是 $20 \times 1.5 = 30$。系统为其冠名：**【卓越的铁剑】
**。

#### 2. 随机词条池 (Affixes)

除了基础攻防的波动，装备还会附带直接加成“四维属性”的词条。词条数值根据装备等级区间（Lv1-10, Lv11-20...）进行随机。

* **[蛮力]**：力量 (STR) + `X`
* **[坚韧]**：体质 (CON) + `X`
* **[轻灵]**：敏捷 (AGI) + `X`
* **[启迪]**：智慧 (WIS) + `X`
* *特殊稀有词条*（仅金装可能出现）：**[吸血]** 伤害的5%转化为HP / **[寻宝]** 挂机极品掉率+5%

### 三、 物品数据结构设计 (JSON示例)

#### 1. 装备模板 (配置表字典中)

```json
{
  "item_id": "WPN_001",
  "name": "铁剑",
  "type": "weapon",
  "equip_level": 10,
  "base_stats": {
    "atk": 20
  },
  "drop_weight": {
    "white": 50,
    "green": 30,
    "blue": 15,
    "purple": 4,
    "gold": 1
  }
}
```

#### 2. 装备实例 (玩家背包中 / 数据库中)

这就是那把掉落后，经过波动系统生成的唯一装备。

```json
{
  "uuid": "eq_5f8a9b2c",
  "template_id": "WPN_001",
  "display_name": "锋利的铁剑(蓝)",
  "quality": "blue",
  "stats": {
    "atk": 26
    // 20 * 1.30 (波动系数)
  },
  "affixes": {
    "STR": 3,
    // 随机生成的词条1
    "AGI": 2
    // 随机生成的词条2
  },
  "forge_level": 0
  // 锻造强化等级
}
```

### 四、 全物品分类及属性设计规范

除了装备，其他物品也需要标准化分类，以便于炼药、锻造、地灵AI和NPC交互系统调用。

#### 1. 装备类 (Equipment) - `type: "weapon" | "armor"`

* **作用**：穿戴后提供ATK/DEF和四维加成。
* **处理机制**：必须进行波动生成，丢弃/出售时销毁UUID。不叠加。

#### 2. 材料类 (Material) - `type: "material"`

* **作用**：用于锻造升级、福地布阵。
* **字段设计**：需要包含 `tags` 字段（供NPC识别喜好或地灵AI识别用途）。
* **示例**：
    * 【黑铁矿石】：`tags: ["ore", "metal", "forge_base"]`
    * 【破损的阵法残页】：`tags: ["array", "magic", "fudi_def"]`

#### 3. 草药与炼药产物 (Alchemy) - `type: "herb" | "potion"`

* **作用**：草药用于智慧加成的 `#炼制` 指令，产物用于恢复或Buff。
* **示例**：
    * 【毒龙草】(herb)：炼制毒药或高级金创药的材料。
    * 【大还丹】(potion)：消耗品，恢复 100% HP，战斗外使用。
    * 【敏捷药剂】(potion)：挂机增益，持续2小时内，敏捷+50（极大提升挂机杀怪效率）。

#### 4. 福地专供类 (Fudi Core) - `type: "seed" | "egg"`

* **作用**：供地灵AI读取并执行挂机种植/养殖。
* **特有字段**：
    * `grow_time`：成熟所需的真实时间（如：24小时）。
    * `yield_id`：成熟后产出的物品ID（如：种下【赤火莲子】，产出【赤火莲】）。
    * `survive_rate`：存活率，受福地灵气浓郁度和玩家智慧影响。
* **示例**：
    * 【赤火莲子】：火属性种子，成熟需48小时，`tags: ["seed", "fire", "rare"]`。
    * 【玄冰龟蛋】：灵蛋，孵化需72小时，孵化后转化为永久战宠或极品材料，`tags:["egg", "water"]`。

#### 5. 交互/珍宝类 (Treasure & Gift) - `type: "gift"`

* **作用**：主要用于给酒馆导入的虚拟NPC送礼，提升羁绊好感度；或高价卖给系统黑市。
* **示例**：
    * 【星尘水晶】：无实际战斗作用。`tags: ["gem", "shiny", "magic"]`。（如果送给喜欢 `shiny` 标签的傲娇法师NPC，好感度+15；如果送给喜欢
      `muscle` 的狂战士，好感度-5并被骂）。

### 五、 物品与各大系统的联动说明

1. **【背包展示防刷屏】**：
    * 对于非标装备，由于名字很长且后缀多，玩家输入 `#背包` 时，建议只按品质折叠显示，例如：“装备：紫装(2件)，蓝装(5件)”。
    * 玩家想看具体属性，需输入 `#背包 装备` 或 `#查看 [装备UUID]`。
2. **【锻造系统 (力量主导)】**：
    * `#锻造` 的逻辑是在**保留波动的最终基础属性和词条**上，额外增加固定数值。
    * 例如：【锋利的铁剑(蓝)】ATK为26。通过消耗黑铁矿石进行 `#锻造`，变为【锋利的铁剑(蓝) +1】，ATK变为 $26 + 5 = 31$
      。这保证了底层胚子（波动初值）极品的重要性。
3. **【地灵AI与物品Tags的联动】**：
    * 当地灵接收到自然语言指令：`“@地灵 帮我把包里那些火属性的种子全种了”`。
    * LLM解析出意图 `action: "plant"`, `target_tags: ["seed", "fire"]`。
    * 你的后端代码会遍历玩家背包，找到所有带有这两个tag的物品并执行扣除/种下逻辑。

这样的物品架构，不仅满足了你想要的**刷子游戏**的乐趣（即每次掉落的装备都不一样，追求极品词条组合），同时也做到了**数据结构高度可复用
**，能完美无缝对接到你的地灵管家和好感度NPC系统之中。
